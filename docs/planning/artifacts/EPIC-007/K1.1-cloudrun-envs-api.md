# K1.1 — Inventário de Envs (Cloud Run · API)

## Objetivo

Consolidar **apenas metadados** das variáveis e referências a secrets do serviço de **API no Cloud Run** impactado pela migração, para garantir bootstrap/deploy/rollback com segurança.

- **Proibido**: registrar valores, outputs de `gcloud` com valores, ou qualquer conteúdo sensível.
- **Permitido**: registrar **nome**, **tipo (secret/não-secret)**, **origem**, **escopo**, e observações.

## Instruções de preenchimento

- Preencha uma linha por variável/config.
- Quando a variável vier de Secret Manager, registre **somente**:
  - `secret_name` (nome do secret)
  - `secret_version` (ex: `latest` ou número), se aplicável
  - **nunca** o valor.
- Em **Origem**, use um dos padrões:
  - `cloudrun/env`
  - `gcp/secret-manager`
  - `external/<provider>` (ex: `external/neon`, `external/upstash`, `external/stripe`)
  - `manual`
- Em **Escopo**, use:
  - `service` (injeção no serviço Cloud Run)
  - `revision` (se variar por revision/rollout)
  - `runtime` (apenas em runtime)
  - `build` (se houver etapa build relevante)

## Observações críticas (ação)

- **Legado Paddle:** há `PADDLE_*`. Confirmar se estão mortos (não remover sem evidência/rollback).
- **Chaves de assinatura:** `SIGNING_PRIVATE_KEY` exige controle/rotação; mapear onde vive e quem tem acesso.
- **Stripe prices:** os IDs de price existem no env; na migração K1.12 vamos precisar do price de teste (R$1) separado.
