# K1.1 — Inventário de Envs (Vercel · Admin)

## Objetivo

Consolidar **apenas metadados** das variáveis de ambiente do projeto Vercel **admin** (Production / Preview / Development) para permitir bootstrap, deploy e rollback com segurança.

- **Proibido**: registrar valores de envs/secrets, outputs de CLI, prints com valores, ou colar `.env`.
- **Permitido**: registrar **nome**, **ambiente**, **tipo (secret/não-secret)**, **origem**, e observações de uso/risco.

## Instruções de preenchimento

- Preencha uma linha por variável.
- Se a variável existir em mais de um ambiente (prod/preview/dev), registre **uma linha por ambiente** (para capturar diferenças).
- Em **Origem**, use um dos padrões:
  - `vercel/env` (configurada como env no projeto)
  - `vercel/secret` (referência a secret gerenciado pela Vercel)
  - `gcp/secret-manager` (referência indireta; registrar apenas o nome/identificador do secret, sem valor)
  - `external/<provider>` (ex: `external/stripe`, `external/github`, `external/resend`)
  - `manual` (aplicada manualmente; evitar)
- Em **Notes**, descreva impacto/criticidade, dependência, e plano de rotação quando aplicável.

## Variáveis críticas (foco)

Preencher e **destacar** (ex: com `**` no Nome) quando aplicável:

- `API_URL`
- `NEXTAUTH_URL`
- `NEXTAUTH_SECRET`
- `DATABASE_URL`
- `STRIPE_*`
- `GITHUB_*`
- `RESEND_*`
- `JWT_SECRET`

## Evidências

- [ ] Inventário preenchido **sem valores** (apenas metadados)
- [ ] Revisão 4-eyes registrada (owner + reviewer)
- [ ] Timestamp preenchido (UTC)
- [ ] Variáveis críticas identificadas e destacadas
- [ ] Divergências por ambiente (prod vs preview vs dev) registradas **somente por nome**
- [ ] Links/refs para origem (dashboard/CLI) anotados **sem outputs com valores**
