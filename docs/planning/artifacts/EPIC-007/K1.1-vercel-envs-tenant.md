# K1.1 — Inventário de Envs (Vercel · Tenant)

## Objetivo

Consolidar **apenas metadados** das variáveis de ambiente do projeto Vercel **tenant** (Production / Preview / Development) para permitir bootstrap, deploy e rollback com segurança.

- **Proibido**: registrar valores de envs/secrets, outputs de CLI, prints com valores, ou colar `.env`.
- **Permitido**: registrar **nome**, **ambiente**, **tipo (secret/não-secret)**, **origem**, e observações de uso/risco.

## Instruções de preenchimento

- Preencha uma linha por variável.
- Se a variável existir em mais de um ambiente (prod/preview/dev), registre **uma linha por ambiente**.
- Em **Origem**, use um dos padrões:
  - `vercel/env`
  - `vercel/secret`
  - `gcp/secret-manager` (somente referência/identificador; **sem valor**)
  - `external/<provider>` (ex: `external/stripe`, `external/github`, `external/resend`, `external/oauth`)
  - `manual`
- Em **Notes**, registre:
  - se é usada em runtime/build
  - se afeta autenticação, billing, email, ou acesso a banco/cache
  - plano de rotação quando aplicável
