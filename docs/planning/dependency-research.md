
### 2.3 Compat Layer Pattern

The compat components are **full custom implementations** using Tailwind CSS + class-variance-authority (CVA), NOT simple re-exports from Radix/Shadcn. Example from `compat/alert.tsx`:

```typescript
import { cva, type VariantProps } from 'class-variance-authority';
import { cn } from '../patterns/utils';

const alertVariants = cva('...base styles...', {
  variants: { variant: {...}, severity: {...} },
  compoundVariants: [...],
  defaultVariants: { variant: 'standard', severity: 'info' },
});

const Alert = React.forwardRef<HTMLDivElement, AlertProps>(
  ({ className, variant, severity, ...props }, ref) => (
    <div ref={ref} className={cn(alertVariants({ variant, severity }), className)} {...props} />
  )
);
```

**Key Pattern:** All compat components follow this structure:
1. CVA for variant management
2. `cn()` utility for class merging
3. `React.forwardRef` for ref forwarding
4. Full props interfaces with JSDoc

### 2.4 Risk Assessment

- The compat layer is **substantial** (~40 components). Any new components would need to follow this same CVA + Tailwind pattern.
- Components like `RadioGroupItem` exist in `compat/radio-group.tsx` and are properly exported via `index.ts`.
- The only confirmed missing export is `SpotlightCard`.

## 3. Usage Tracking Dependencies

### 3.1 UsageRecord Model (Schema)

The `UsageRecord` model is **fully defined** in `schema.base.prisma`:

```prisma
model UsageRecord {
  id           String   @id @default(uuid())
  tenantId     String   @map("tenant_id")
  featureId    String   @map("feature_id")
  currentUsage Int      @default(0) @map("current_usage")
  periodStart  DateTime @map("period_start")
  periodEnd    DateTime @map("period_end")
  lastReset    DateTime @default(now()) @map("last_reset")
  updatedAt    DateTime @updatedAt @map("updated_at")

  tenant  Tenant  @relation(...)
  feature Feature @relation(...)

  @@unique([tenantId, featureId, periodStart])
  @@map("usage_records")
}
```

## 4. File Storage Dependencies

### 4.1 Current Implementation

The file service (`apps/api/src/modules/files/services/file.service.ts`) uses **local filesystem storage**:

```typescript
const UPLOAD_DIR = path.join(process.cwd(), 'uploads');
const MAX_FILE_SIZE = 10 * 1024 * 1024; // 10MB
```

- Writes to local `uploads/` directory
- Uses `crypto.randomUUID()` for unique filenames

## 5. Existing Patterns to Reuse

### 5.1 Soft Delete Pattern

**File:** `apps/api/src/middleware/prisma-soft-delete.ts`

A Prisma middleware (`$use`) that:
1. Automatically adds `deletedAt: null` to all read queries (findMany, findFirst, findUnique, count, aggregate)
2. Also filters write operations (update, updateMany, delete, deleteMany)
3. Supports `includeDeleted` flag to bypass the filter
4. Has a **whitelist** for models without `deletedAt` (Tenant, SecurityAuditLog, ImpersonationSession, AuditLog)
5. Exports utility functions: `softDelete()`, `restore()`, `hardDelete()`

**Reusability:** HIGH — any new model with a `deletedAt` field automatically gets soft delete support.

### 5.2 Bulk Operations Pattern

Only one file references bulk/batch operations: `apps/api/src/lib/email/providers/ses.provider.ts` (for SES batch sending).

No generic bulk operation utility exists. The codebase uses `$transaction` with `createMany`/`deleteMany` for batch operations (e.g., in `RoleService.createRole()` and `updateRole()`).

**Pattern used across the codebase for bulk writes:**
```typescript
return prisma.$transaction(async (tx) => {
  const parent = await tx.model.create({ data: {...} });
  await tx.related.createMany({
    data: items.map(item => ({ parentId: parent.id, ...item }))
  });
  return parent;
});
```

### 5.3 Design System Adapter Pattern

**File:** `apps/admin/lib/design-system/provider/index.tsx`

A React Context-based design system provider that:
1. **Adapter Registry** — maps `DesignSystemType` (MUI, HIG, FLUENT, SHADCN) to adapter implementations
2. **Each adapter** provides: `toSemanticTokens()`, `getDefaultTokens()`, `validateCustomization()`
3. **CSS Variable Injection** — converts semantic tokens to CSS custom properties on `document.documentElement`
4. **Persistence layer** — `syncCustomization()` reads from DB/cache, `saveAndSync()` writes back
5. **State management** — `designSystem`, `tokens`, `mode`, `customization` in React state
6. **Public API** — `useDesignSystem()` hook exposes: `setDesignSystem()`, `toggleMode()`, `updateCustomization()`, `resetCustomization()`

**Reusability:** This pattern could be extended for tenant-level theming (currently user-level via `DesignSystemCustomization` model). For per-tenant themes, the `PlatformConfig` model already has `primaryColor` and branding fields.

**Current gap:** FLUENT and SHADCN adapters are stubbed (fall back to MUI adapter with `// TODO` comments).
