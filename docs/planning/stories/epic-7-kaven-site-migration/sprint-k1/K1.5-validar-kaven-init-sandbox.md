# K1.5 — Validar kaven init em sandbox limpo (pnpm install + pnpm build)

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @qa (Shield)  
> **Priority:** P0 — Gate: se init falhar no sandbox, é proibido avançar para o repo real.  
> **Depends on:** K1.4

---

## Contexto

Antes de bootstrapar o produto real, precisamos provar que `kaven init` funciona em ambiente limpo, gerando um projeto compilável.

---

## User Story

**Como** QA do Kaven,  
**Quero** validar que `kaven init` gera um projeto saudável do zero,  
**Para que** o bootstrap do `kaven-site` não seja uma tentativa em produção.

---

## Acceptance Criteria

- [ ] Rodar `kaven init kaven-site-test` em `/tmp/` (ou workspace temporário limpo).
- [ ] Estrutura obrigatória existe: `apps/api`, `apps/admin`, `apps/tenant`, `packages/database`, `packages/ui`.
- [ ] `pnpm install` executa sem falhas.
- [ ] `pnpm build` executa sem falhas.
- [ ] Se houver erro: abrir item P0 e corrigir no `kaven-framework`/template antes de seguir.

---

## Technical Notes

### Evidência mínima
- log do `kaven init`
