# K1.6 — Bootstrap do kaven-site_ via kaven init + instalação controlada de módulos

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @dev (Forge/Bolt)  
> **Priority:** P0 — Este é o nascimento do produto real pelo mesmo caminho do cliente.  
> **Depends on:** K1.3, K1.4, K1.5

---

## Contexto

O `kaven-site_` deve ser recriado via `kaven init` no branch limpo, com módulos instalados apenas se houver artefato real (K1.4).

---

## User Story

**Como** executor do bootstrap,  
**Quero** gerar o `kaven-site` via CLI e instalar somente módulos válidos,  
**Para que** o produto em produção seja dogfooding real e suportável por `kaven upgrade`.

---

## Acceptance Criteria

- [ ] `kaven init kaven-site` executado no branch limpo do `kaven-site_`.
- [ ] Estrutura final corresponde ao template (apps + packages).
- [ ] Módulos “instalar agora” (K1.4) instalados com versões registradas.
- [ ] `pnpm install` executa sem falha no repo.
- [ ] Registro de baseline (commit + lista de módulos) criado.
