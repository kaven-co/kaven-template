# K1.1 — Auditoria Consolidada de Envs e Secrets (Vercel + Cloud Run)

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @devops (Deploy)  
> **Priority:** P0 — Gate de segurança/execução. Sem isso, deploy é proibido.  
> **Depends on:** Nenhum

---

## Contexto

A migração do `kaven-site_` para ser o produto real em produção depende de replicar/configurar corretamente a infraestrutura **já existente** (Vercel + Cloud Run + Neon + Upstash + Stripe + Resend + OAuth). O risco #1 é “migrar no escuro”: envs faltantes ou secrets errados geram falhas silenciosas que só aparecem após o corte.

Esta story cria o inventário verificável e auditável (sem expor valores sensíveis) para:
- Vercel: projetos `admin` e `tenant`
- Cloud Run: API (serviço atual que será substituído pela imagem do `kaven-site`)
- Dependências externas: referências de Neon/Upstash/Stripe/Resend/OAuth

---

## User Story

**Como** operador de produção do Kaven,  
**Quero** mapear todos os envs/secrets e suas fontes,  
**Para que** o bootstrap e o deploy do `kaven-site` sejam executados com precisão, segurança e rollback viável.

---

## Acceptance Criteria

### Inventário Vercel (Admin + Tenant)
- [ ] Lista completa de envs do projeto Vercel **admin** (production/preview/development).
- [ ] Lista completa de envs do projeto Vercel **tenant** (production/preview/development).
- [ ] Para cada env: **nome**, **ambiente**, **tipo** (secreto/não-secreto), **origem** (Vercel env / secret manager / manual).
- [ ] Variáveis críticas identificadas e destacadas: `API_URL`, `NEXTAUTH_URL`, `NEXTAUTH_SECRET`, `DATABASE_URL`, `STRIPE_*`, `GITHUB_*`, `RESEND_*`, `JWT_SECRET`.

### Inventário Cloud Run (API)
- [ ] Lista completa de envs do Cloud Run do serviço de API impactado (incluindo referências de Secret Manager).
- [ ] Para cada env: **nome**, **tipo**, **origem**, **escopo**.
- [ ] Variáveis críticas destacadas: `DATABASE_URL`, `REDIS_URL`, `JWT_SECRET`, `STRIPE_*`, `GITHUB_*`, `RESEND_*`.

### Governança de secrets (sem vazamento)
- [ ] Nenhum valor sensível é escrito em arquivo versionado.
- [ ] Inventário registra **onde** o secret vive (Vercel/Cloud Run/Secret Manager/.env local) sem registrar o valor.
- [ ] Plano de rotação para `JWT_SECRET`, `NEXTAUTH_SECRET`, `STRIPE_WEBHOOK_SECRET`, `GITHUB_CLIENT_SECRET` (cutover + rollback).
