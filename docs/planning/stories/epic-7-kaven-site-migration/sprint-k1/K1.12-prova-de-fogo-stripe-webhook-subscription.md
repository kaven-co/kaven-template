# K1.12 — Prova de fogo backend II (Stripe R$1 + webhook + subscription persistida)

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @qa (Shield)  
> **Priority:** P0 — Prova comercial. Sem webhook 200 + persistência, não existe produto vendável.  
> **Depends on:** K1.11

---

## Contexto

Existe um produto de teste de R$1,00 no Stripe para validar o pipeline completo: checkout -> pagamento -> webhook -> banco -> acesso no tenant.

---

## User Story

**Como** QA do produto,  
**Quero** executar o fluxo Stripe ponta-a-ponta com evidência no Stripe e no banco,  
**Para que** possamos afirmar que o produto é vendável e auditável.

---

## Acceptance Criteria

### Setup
- [ ] Price ID do produto de teste confirmado e registrado.
- [ ] Plano “R$1 teste” mapeado no frontend de forma temporária (não público).

### Checkout
- [ ] Usuário logado inicia checkout e é redirecionado ao Stripe corretamente.
- [ ] Pagamento teste finaliza (cartão `4242...`).
- [ ] Redirect para success URL retorna ao site.

### Webhook + Banco
- [ ] Stripe dashboard mostra evento entregue com status 200.
- [ ] API processa evento com idempotência (sem duplicar subscription).
- [ ] Banco possui `Subscription` criada/atualizada com status coerente.
- [ ] Tenant reflete assinatura ativa (UI ou endpoint).

### Pós-teste
- [ ] Remover plano de teste do frontend.
- [ ] Cancelar assinatura de teste se necessário.
- [ ] Verificar que não ficou sujeira de dados de teste (ou registrar o que ficou e por quê).

---

## Evidence

- [ ] Screenshot/registro do evento no Stripe (status 200).
- [ ] Registro do `Subscription` no banco (query/visualização).
- [ ] Logs da API do processamento do webhook (sem secrets).

---

## Definition of Done

- [ ] Fluxo Stripe validado ou P0 registrado para correção via framework.

---

## Time Estimate

- Setup + deploy temporário: 2h
- Fluxo checkout: 2h
- Webhook + banco: 2h
- Limpeza pós-teste + relatório: 2h
- **Total: 8h**
