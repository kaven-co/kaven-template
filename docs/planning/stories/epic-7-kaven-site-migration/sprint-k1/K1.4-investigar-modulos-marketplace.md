# K1.4 — Investigação Real de Módulos no Marketplace (artefato real vs placeholder)

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @dev (Bolt/Trace)  
> **Priority:** P0 — Não assumir lista; módulo “público” sem release quebra bootstrap.  
> **Depends on:** Nenhum

---

## Contexto

O `kaven-site_` será bootstrapado via `kaven init` e poderá instalar módulos do marketplace. O plano exige **evidência real** do que existe (módulo público + release + artefato baixável), antes de decidir instalar.

---

## User Story

**Como** executor do bootstrap,  
**Quero** validar quais módulos são realmente instaláveis agora,  
**Para que** o `kaven init` do produto não falhe por dependências fictícias.

---

## Acceptance Criteria

- [ ] Rodar `kaven marketplace browse` e registrar output (sem segredos).
- [ ] Confirmar via API do marketplace:
  - categorias disponíveis
  - módulos com `isPublic: true`
  - releases com artefato existente
- [ ] Para cada módulo candidato: validar “download possível” (token + S3 + assinatura, se aplicável).
- [ ] Produzir lista final:
  - **Instalar no bootstrap** (agora)
  - **Pular por agora** (motivo e próximo passo)
