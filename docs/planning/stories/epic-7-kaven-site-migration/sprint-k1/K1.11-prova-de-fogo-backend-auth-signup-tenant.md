## Contexto

O foco aqui é backend “funcionar”, mesmo com UI imperfeita. Precisamos provar:
- signup e confirmação mínima por email
- auth social (Google/GitHub) ponta-a-ponta
- tenant mínimo criado e acessível
- isolamento básico A/B

Qualquer falha aqui é P0.

---

## User Story

**Como** QA,  
**Quero** validar os fluxos de identidade e tenant,  
**Para que** o produto esteja pronto para prova de pagamento (Stripe).

---

## Acceptance Criteria

### Auth Social
- [ ] Google: botão/fluxo -> redirect -> callback -> sessão -> acesso tenant ✅
- [ ] GitHub: botão/fluxo -> redirect -> callback -> sessão -> acesso tenant ✅
- [ ] Banco: usuário criado com `googleId` ou `githubId` (ou equivalente).

### Signup email/password
- [ ] Signup cria usuário no banco.
- [ ] Email transacional enviado (Resend) quando aplicável.
- [ ] Link de confirmação (se existir) funciona e ativa conta.
- [ ] Login pós-confirmação funciona.

### Tenant mínimo
- [ ] Após login, usuário acessa tenant page sem erro 5xx.
- [ ] Tenant criado/associado no banco.

### Isolamento multi-tenant (mínimo)
- [ ] Criar usuário/tenant B e tentar acessar dados do A (deve falhar).
- [ ] Logs/queries indicam `tenantId` aplicado consistentemente.

### Triage
- [ ] Todos os achados classificados: P0 / P1 / P2.
- [ ] P0 cria entrada imediata para loop de correção (K1.13).
