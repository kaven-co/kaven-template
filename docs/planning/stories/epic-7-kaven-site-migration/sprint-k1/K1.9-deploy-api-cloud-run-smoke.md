# K1.9 — Deploy da API do kaven-site no Cloud Run + smoke test de saúde

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @devops (Deploy)  
> **Priority:** P0 — Troca de backend em produção. Go/No-Go rígido com rollback automático.  
> **Depends on:** K1.7, K1.8

---

## Contexto

Este passo substitui a API em produção (mesma URL do Cloud Run). Qualquer falha bloqueia o fluxo do produto inteiro.

---

## User Story

**Como** operador do deploy,  
**Quero** publicar a nova API com validação imediata e rollback,  
**Para que** o corte seja seguro e mensurável.

---

## Acceptance Criteria

### Pré-deploy (No-Go se falhar)
- [ ] Serviço Cloud Run alvo identificado sem ambiguidade.
- [ ] Imagem Docker buildada com digest imutável (não usar `latest`).
- [ ] Baseline pré-deploy coletado (health e métricas mínimas).

### Deploy
- [ ] Deploy no Cloud Run executado com envs do K1.7.
- [ ] `prisma migrate deploy` executado **somente** se necessário e com plano de rollback (expand/contract quando aplicável).

### Smoke (até +5 min)
- [ ] `GET /health` retorna 200 (3x consecutivas).
- [ ] Endpoint de auth básico responde sem 5xx.
- [ ] 5xx < 1% nos primeiros 5 min.

### Rollback Trigger
- [ ] Se health falhar >5 min: rollback imediatamente para revision anterior.

---

## Evidence

- [ ] Revision/digest do deploy registrado.
- [ ] Logs de health e taxa de erro registrados.
- [ ] Decisão Go/No-Go registrada (com timestamp).

---

## Definition of Done

- [ ] API saudável em produção com smoke aprovado.
- [ ] Rollback preparado e validado (mesmo que não executado).

---

## Time Estimate

- Build/push imagem: 2h
- Deploy + migrações: 2h
- Smoke + evidência: 2h
- **Total: 6h**
