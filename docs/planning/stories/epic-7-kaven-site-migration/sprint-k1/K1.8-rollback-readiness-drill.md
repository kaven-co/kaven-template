# K1.8 — Rollback readiness (<30 min) com drill verificável

> **Epic:** 7 (Kaven Site Production Migration) | **Sprint:** K1 | **Agent:** @devops (Deploy)  
> **Priority:** P0 — Sem rollback ensaiado, o cutover é proibido.  
> **Depends on:** K1.1, K1.2

---

## Contexto

O plano substitui produção (API Cloud Run + Vercel admin/tenant). Se algo falhar, precisamos voltar ao estado anterior em <30 min, com verificação pós-rollback.

---

## User Story

**Como** operador de produção,  
**Quero** ter rollback ensaiado e verificável,  
**Para que** eu consiga desfazer o corte sem improviso durante incidente.

---

## Acceptance Criteria

- [ ] Snapshot do estado atual do Cloud Run exportado (config completa).
- [ ] Backup dos envs do Vercel admin e tenant exportado (para restauração).
- [ ] Revision anterior do Cloud Run identificada e anotada.
- [ ] Deployments anteriores no Vercel identificados (admin + tenant).
- [ ] Drill completo executado (simulação controlada) e tempo registrado.
- [ ] Checklist de “pós-rollback” definido: health + login + webhook.
