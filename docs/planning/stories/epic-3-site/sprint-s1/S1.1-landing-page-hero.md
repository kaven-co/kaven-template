# S1.1 — Landing Page Hero

> **Epic:** 3 (Site — Landing Page MVP) | **Sprint:** S1 | **Agent:** @dev
> **Priority:** P0
> **Depends on:** None

---

## Contexto

O kaven-site é o ponto de entrada para todos os potenciais clientes do Kaven. A landing page precisa comunicar valor em 5 segundos: "SaaS boilerplate enterprise-grade que comprime 3-6 meses de setup para dias". O hero section é o primeiro contato visual e precisa ter performance excelente (LCP < 2.5s), ser responsivo, e converter visitantes em cliques no pricing ou docs.

O projeto kaven-site já existe como scaffold (2 commits) usando Next.js App Router. Esta story configura a base do projeto e implementa o hero section completo.

---

## User Story

**Como** visitante do kaven.site,
**Quero** ver imediatamente o que o Kaven faz e qual o valor que oferece,
**Para que** eu decida em segundos se vale explorar mais (pricing, docs, GitHub).

## Technical Notes

- **Framework**: Next.js 14+ with App Router (`app/` directory)
- **Styling**: Tailwind CSS v3.4+ with `tailwind.config.ts` defining Kaven design tokens
- **Dark mode**: Tailwind `darkMode: 'class'` strategy with theme provider component
- **Font**: Use `next/font` to load Inter (body) and JetBrains Mono (code) — zero layout shift
- **Icons**: Lucide React for UI icons, custom SVGs for tech badges
- **Layout structure**:
  ```
  app/
  ├── layout.tsx          # Root layout with ThemeProvider, fonts, metadata
  ├── page.tsx            # Landing page (hero + sections below fold)
  ├── globals.css         # Tailwind directives + custom properties
  components/
  ├── layout/
  │   ├── Header.tsx      # Navbar with mobile hamburger
  │   └── Footer.tsx
  ├── landing/
  │   ├── HeroSection.tsx # Main hero with CTAs
  │   └── TechBadges.tsx  # Technology badges row
  └── ui/
      ├── Button.tsx      # Reusable button variants
      └── ThemeToggle.tsx # Dark/light mode switch
  ```
- **Performance**: No client-side JS above the fold except theme toggle. Hero text is server-rendered.
- **OG Image**: Static for now (designed in Figma or placeholder), dynamic generation in S1.5.
- **Env vars needed**: `NEXT_PUBLIC_SITE_URL`, `NEXT_PUBLIC_GITHUB_REPO_URL`
