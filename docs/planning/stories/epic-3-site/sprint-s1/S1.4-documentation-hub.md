# S1.4 — Documentation Hub

> **Epic:** 3 (Site — Landing Page MVP) | **Sprint:** S1 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** None

---

## Contexto

A documentacao e o segundo fator mais importante na decisao de compra de um framework (depois do preco). Desenvolvedores querem ver docs claras, completas e com exemplos de codigo antes de investir. O Kaven precisa de um doc hub com duas camadas: docs publicas (Getting Started, Architecture, CLI) que servem como marketing tecnico, e docs gateadas (API Reference, Advanced Config, Deployment) que incentivam o cadastro/compra.

A escolha de Fumadocs (ou Nextra) como engine de MDX garante que os docs sao escritos em Markdown, versionados no git, e renderizados como paginas Next.js com search, sidebar, e syntax highlighting sem infra adicional.

---

## User Story

**Como** desenvolvedor considerando o Kaven,
**Quero** acessar documentacao tecnica completa com exemplos de codigo e busca,
**Para que** eu avalie a qualidade do framework e saiba como usa-lo antes e depois da compra.

## Technical Notes

- **Docs engine**: Fumadocs (`fumadocs-ui` + `fumadocs-core` + `fumadocs-mdx`) — integrates natively with Next.js App Router and supports content collections, search, and i18n out of the box.
  - Alternative: Nextra v3 (also App Router compatible)
- **Content location**: `content/docs/` directory with MDX files organized by section
  ```
  content/docs/
  ├── getting-started/
  │   ├── introduction.mdx
  │   ├── quick-start.mdx
  │   ├── project-structure.mdx
  │   └── configuration.mdx
  ├── architecture/
  │   ├── overview.mdx
  │   ├── multi-tenancy.mdx
  │   ├── module-system.mdx
  │   └── ...
  ├── cli/
  │   ├── installation.mdx
  │   ├── commands.mdx
  │   └── configuration.mdx
  └── meta.json  # Navigation structure
  ```
- **Gating mechanism**: Middleware checks auth cookie. If not authenticated, render page with CSS blur + overlay prompt. Content is still server-rendered (SEO benefit) but visually gated.
- **Search**: Fumadocs built-in search (static JSON index) for MVP. Algolia DocSearch integration as enhancement later.
- **Syntax highlighting**: Shiki is built into Fumadocs. Configure with themes matching Kaven dark/light mode.
- **File structure**:
  ```
  app/docs/
  ├── layout.tsx           # Docs layout with sidebar
  ├── [[...slug]]/
  │   └── page.tsx         # Dynamic doc page renderer
  components/docs/
  ├── GatedOverlay.tsx     # Auth gate overlay for premium docs
  ├── CopyButton.tsx       # Code block copy button
  └── DocSearch.tsx        # Search component
  lib/
  └── docs-config.ts       # Navigation structure, gated sections config
  ```
- **Env vars**: `NEXT_PUBLIC_GITHUB_DOCS_URL` (base URL for "Edit on GitHub" links)
