> **Epic:** 3 (Site — Landing Page MVP) | **Sprint:** S1 | **Agent:** @dev
> **Priority:** P0
> **Depends on:** None

---

## Contexto

O pricing page e a pagina mais critica do funil de conversao. Precisa apresentar os 3 tiers do Kaven (Starter, Complete, Pro) com precos de lancamento, comparacao de features, e CTAs que levam direto ao Paddle checkout. A estrategia de pricing ja esta definida: precos de lancamento com strikethrough no preco regular para criar urgencia, toggle mensal/anual com desconto de 2 meses gratis no anual, e um tier Enterprise com CTA de contato.

O kaven-marketplace tera o endpoint POST /checkout/session que cria a sessao Paddle. Esta pagina precisa consumir esse endpoint via fetch client-side.

---

## User Story

**Como** potencial cliente do Kaven,
**Quero** comparar os planos disponlveis com precos e features claramente apresentados,
**Para que** eu escolha o plano adequado e inicie a compra sem friccao.

## Technical Notes

- **Route**: `app/pricing/page.tsx` — Server Component with client interactive parts
- **Pricing data**: Define in `lib/pricing-data.ts` as typed constants (easy to update)
  ```typescript
  interface PricingTier {
    id: string;
    name: string;
    monthlyPrice: number;
    launchMonthlyPrice: number;
    yearlyPrice: number;
    launchYearlyPrice: number;
    description: string;
    features: string[];
    highlighted: boolean;
    ctaText: string;
    paddlePriceId: { monthly: string; yearly: string };
  }
  ```
- **Paddle integration**: Client-side fetch to marketplace API. Paddle.js loaded via `next/script` with `strategy="lazyOnload"`.
- **Feature matrix**: Data-driven from `lib/feature-matrix.ts` — array of categories with features and tier availability booleans.
- **Billing toggle**: Client component `PricingToggle.tsx` using React state. Prices animate on switch (CSS transition).
- **FAQ**: Use `<details>/<summary>` for progressive enhancement, enhanced with JS for animation.
- **Env vars**: `NEXT_PUBLIC_MARKETPLACE_API_URL`, `NEXT_PUBLIC_PADDLE_SELLER_ID`
- **File structure**:
  ```
  app/pricing/
  └── page.tsx
  components/pricing/
  ├── PricingCard.tsx         # Individual tier card
  ├── PricingToggle.tsx       # Monthly/Yearly switch (client component)
  ├── FeatureMatrix.tsx       # Comparison table
  ├── EnterpriseCTA.tsx       # Enterprise contact section
  └── PricingFAQ.tsx          # FAQ accordion
  lib/
  ├── pricing-data.ts         # Tier definitions
  └── feature-matrix.ts       # Feature comparison data
  ```
- **Accessibility**: Price changes announced to screen readers via `aria-live` region. FAQ uses proper `aria-expanded` attributes.
