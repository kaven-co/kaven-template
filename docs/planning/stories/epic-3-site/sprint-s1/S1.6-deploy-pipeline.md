# S1.6 — Deploy Pipeline

> **Epic:** 3 (Site — Landing Page MVP) | **Sprint:** S1 | **Agent:** @devops
> **Priority:** P0
> **Depends on:** S1.1

---

## Contexto

O kaven-site precisa de um pipeline de deploy confiavel e automatizado antes do launch. Vercel e a escolha natural para Next.js (mesma empresa, otimizacao nativa, edge network global). O pipeline precisa suportar dois ambientes (preview para PRs, production para main), custom domain, e injecao segura de environment variables para Paddle, analytics, e API URL do marketplace.

A configuracao e direta mas critica: um erro aqui bloqueia todo o time. Precisa ser configurada cedo (P0) para que todas as outras stories possam ser testadas em preview deployments.

---

## User Story

**Como** desenvolvedor do Kaven,
**Quero** que o site faca deploy automatico em cada push/PR com preview environments,
**Para que** eu possa iterar rapidamente, testar mudancas em preview URLs, e ter confianca no deploy de producao.

## Acceptance Criteria

### Vercel Configuration
- [ ] Vercel project created and linked to kaven-site GitHub repository
- [ ] Framework preset: Next.js (auto-detected)
- [ ] Build command: `npm run build`
- [ ] Output directory: `.next` (auto-detected)
- [ ] Node.js version: 20.x (LTS)
- [ ] Install command: `npm ci` (clean install for reproducibility)

### Environments
- [ ] **Production**: auto-deploys on push to `main` branch
- [ ] **Preview**: auto-deploys on every PR branch push, unique URL per deployment
- [ ] Vercel bot comments on PRs with preview URL, build status, and Lighthouse scores
- [ ] Preview deployments are automatically deleted when PR is closed/merged

### Custom Domain
- [ ] Custom domain configured: `kaven.sh` (primary) or `kaven.dev` (alternative)
- [ ] `www.kaven.sh` redirects to `kaven.sh` (non-www preferred)
- [ ] SSL certificate auto-provisioned and auto-renewed by Vercel
- [ ] DNS records documented in README or deployment docs

### Environment Variables
- [ ] Production environment variables set in Vercel dashboard (not in code):
  - `NEXT_PUBLIC_SITE_URL` = `https://kaven.sh`
  - `NEXT_PUBLIC_MARKETPLACE_API_URL` = marketplace production URL
  - `NEXT_PUBLIC_PADDLE_SELLER_ID` = Paddle seller ID
  - `NEXT_PUBLIC_PADDLE_ENVIRONMENT` = `production`
  - `NEXT_PUBLIC_GITHUB_REPO_URL` = framework repo URL
  - `NEXT_PUBLIC_GITHUB_DOCS_URL` = docs source URL
  - `ANALYTICS_ID` = analytics tracking ID (server-only)
- [ ] Preview environment variables override:
  - `NEXT_PUBLIC_SITE_URL` = Vercel preview URL (auto)
  - `NEXT_PUBLIC_PADDLE_ENVIRONMENT` = `sandbox`
  - `NEXT_PUBLIC_MARKETPLACE_API_URL` = marketplace staging URL
- [ ] Sensitive env vars marked as "Sensitive" in Vercel (hidden from logs)

### GitHub Actions CI
- [ ] `.github/workflows/ci.yml` — runs on PR and push to main:
  - `npm ci` — install dependencies
  - `npm run lint` — ESLint check
  - `npm run typecheck` — TypeScript type checking (if configured)
  - `npm run build` — verify build succeeds
- [ ] CI must pass before PR can be merged (branch protection rule)
- [ ] CI runs in parallel with Vercel preview deployment

### Performance & Caching
- [ ] Vercel edge caching enabled for static assets (default)
- [ ] Cache-Control headers configured for static assets: `public, max-age=31536000, immutable`
- [ ] Next.js ISR (Incremental Static Regeneration) configured for docs pages if applicable
- [ ] Vercel Analytics enabled (Web Vitals tracking)
- [ ] Vercel Speed Insights enabled

### Documentation
- [ ] `DEPLOYMENT.md` in kaven-site repo root with:
  - How to set up Vercel project from scratch
  - Required environment variables list with descriptions
  - DNS configuration instructions
  - How to trigger manual redeploy
  - How to rollback to previous deployment
  - Troubleshooting common deploy issues

## Technical Notes

- **Vercel CLI**: Use `vercel` CLI for initial project setup and linking. Can also be done via Vercel dashboard.
  ```bash
  npm i -g vercel
  vercel login
  vercel link  # Link to existing project or create new
  vercel env pull .env.local  # Pull env vars for local dev
  ```
- **GitHub Integration**: Vercel GitHub app handles auto-deployment. No GitHub Actions needed for deploy itself — Actions are for CI checks only.
- **Branch protection**: Configure in GitHub repo settings:
  - Require status checks: `ci` workflow + Vercel deployment
  - Require PR reviews: 1 approval
  - No direct pushes to main
- **DNS Setup** (for kaven.sh):
  ```
  Type  Name  Value
  A     @     76.76.21.21 (Vercel)
  CNAME www   cname.vercel-dns.com
  ```
- **vercel.json** (optional, for fine-tuning):
  ```json
  {
    "framework": "nextjs",
    "regions": ["iad1"],
    "headers": [
      {
        "source": "/fonts/(.*)",
        "headers": [
          { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
        ]
      }
    ]
  }
  ```
- **GitHub Actions workflow structure**:
  ```yaml
  name: CI
  on:
    push: { branches: [main] }
    pull_request: { branches: [main] }
  jobs:
    ci:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with: { node-version: '20', cache: 'npm' }
        - run: npm ci
        - run: npm run lint
        - run: npm run build
  ```
- **Cost**: Vercel Hobby plan (free) supports 1 project, custom domain, preview deployments. Pro ($20/mo) needed for team features and analytics. Start with Hobby, upgrade when needed.
