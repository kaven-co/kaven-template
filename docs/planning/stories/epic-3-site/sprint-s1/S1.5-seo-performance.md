## Acceptance Criteria

### SEO Meta Tags
- [ ] Every page has unique `<title>` tag following pattern: "Page Name | Kaven â€” SaaS Boilerplate"
- [ ] Every page has unique `<meta name="description">` (150-160 characters, keyword-rich)
- [ ] `<meta name="keywords">` on key pages: "SaaS boilerplate, multi-tenant framework, Next.js starter, Fastify API, TypeScript SaaS"
- [ ] Canonical URLs set on all pages: `<link rel="canonical" href="https://kaven.site/page">`
- [ ] `hreflang` tags present for future i18n readiness (en as default)

### Open Graph
- [ ] `og:title`, `og:description`, `og:image`, `og:url`, `og:type` on all pages
- [ ] Dynamic OG images generated with `@vercel/og` (ImageResponse API):
  - Homepage: Kaven logo + tagline
  - Pricing: "Kaven Pricing â€” Starting at $99"
  - Features: "22+ Enterprise Features"
  - Docs: "Kaven Documentation â€” [Page Title]"
- [ ] OG image dimensions: 1200x630px
- [ ] `og:site_name` set to "Kaven"

### Twitter Cards
- [ ] `twitter:card` = `summary_large_image` on all pages
- [ ] `twitter:title`, `twitter:description`, `twitter:image` matching OG equivalents
- [ ] `twitter:site` set to Kaven Twitter handle

### Technical SEO
- [ ] `robots.txt` at root: allow all crawlers, reference sitemap
- [ ] `sitemap.xml` auto-generated from Next.js routes (using `next-sitemap` or App Router metadata API)
- [ ] Sitemap includes: /, /pricing, /features, /docs/*, all public doc pages
- [ ] Sitemap excludes: /dashboard/*, /billing/*, other authenticated routes
- [ ] Structured data (JSON-LD) on relevant pages:
  - Homepage: `Organization` schema (name, url, logo, sameAs for social)
  - Pricing: `Product` schema for each tier (name, description, offers with price)
  - FAQ: `FAQPage` schema (questions and answers from pricing FAQ)
- [ ] JSON-LD validates without errors on Google Rich Results Test
- [ ] No broken links (internal link audit)

### Core Web Vitals
- [ ] **LCP (Largest Contentful Paint)** < 2.5s on all pages (mobile 4G throttled)
- [ ] **FID (First Input Delay)** < 100ms on all pages
- [ ] **CLS (Cumulative Layout Shift)** < 0.1 on all pages
- [ ] **TTFB (Time to First Byte)** < 600ms (Vercel edge)
- [ ] **FCP (First Contentful Paint)** < 1.8s

### Image Optimization
- [ ] All images use `next/image` with explicit `width`, `height`, and `sizes` attributes
- [ ] Images served in WebP/AVIF format (Next.js automatic)
- [ ] No images above the fold without `priority` prop
- [ ] Lazy loading on all below-fold images (default `next/image` behavior)

### Font Optimization
- [ ] Fonts loaded via `next/font` â€” zero external requests, zero layout shift
- [ ] Font display: swap â€” text visible immediately
- [ ] Only load font weights actually used (400, 500, 600, 700)

### Bundle Optimization
- [ ] No unused JavaScript in critical path (tree-shaking verified)
- [ ] Framer Motion tree-shaken: only import used components (`motion`, `AnimatePresence`)
- [ ] Dynamic imports for heavy components: Shiki, code previews, FAQ accordion
- [ ] Lighthouse performance score >= 95 on desktop, >= 90 on mobile (all pages)
- [ ] Total JS bundle < 200KB gzipped for landing page

## Technical Notes

- **OG Image generation**: Use `@vercel/og` with `ImageResponse` in route handlers:
  ```
  app/api/og/route.tsx        # Default OG image
  app/api/og/pricing/route.tsx # Pricing-specific OG image
  app/api/og/docs/route.tsx    # Docs-specific OG image
  ```
- **Metadata API**: Use Next.js App Router `generateMetadata()` in each page for dynamic meta tags. Shared defaults in `lib/metadata.ts`.
- **Sitemap**: Use Next.js `app/sitemap.ts` (built-in metadata API) or `next-sitemap` package.
- **JSON-LD**: Create reusable components in `components/seo/`:
  ```typescript
  // components/seo/JsonLd.tsx
  export function OrganizationJsonLd() { ... }
  export function ProductJsonLd({ tier }: { tier: PricingTier }) { ... }
  export function FAQJsonLd({ faqs }: { faqs: FAQ[] }) { ... }
  ```
- **Performance monitoring**: Add `web-vitals` library to report CWV metrics to console in development and to analytics endpoint in production.
- **Bundle analysis**: Run `npx @next/bundle-analyzer` to verify bundle sizes. Add as dev dependency.
