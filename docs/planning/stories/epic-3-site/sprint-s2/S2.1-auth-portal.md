# S2.1 — Auth Portal

> **Epic:** 3 (Site — Customer Portal) | **Sprint:** S2 | **Agent:** @dev
> **Priority:** P0
> **Depends on:** M1.2

---

## Contexto

O customer portal precisa de autenticacao para que compradores acessem suas licencas, billing, e modulos. O fluxo de auth usa GitHub OAuth (o publico-alvo sao desenvolvedores que ja tem GitHub) via o marketplace API. O kaven-marketplace ja tera o endpoint `/auth/github` que faz o OAuth dance e retorna um session token. O kaven-site precisa consumir esse fluxo, armazenar a sessao em HTTP-only cookies (seguro contra XSS), e proteger as rotas do portal.

A dependencia em M1.2 (marketplace auth endpoints) pode ser desbloqueada com mock responses durante desenvolvimento, mas a integracao real e necessaria antes do merge.

---

## User Story

**Como** cliente do Kaven que comprou uma licenca,
**Quero** fazer login com minha conta GitHub de forma rapida e segura,
**Para que** eu acesse meu dashboard de licencas, billing, e modulos sem criar mais uma conta.

## Acceptance Criteria

### OAuth Flow
- [ ] "Sign in with GitHub" button visible in site header (when not authenticated)
- [ ] Click redirects to marketplace API: `GET /auth/github?redirect_uri={site_url}/auth/callback`
- [ ] Marketplace API handles GitHub OAuth dance (authorization code flow)
- [ ] Callback route `/auth/callback` receives auth token from marketplace API
- [ ] Token stored in HTTP-only, Secure, SameSite=Lax cookie named `kaven_session`
- [ ] Cookie expiration: 7 days (refreshable)
- [ ] After successful login: redirect to `/dashboard` (or to the page user was trying to access)

### Session Management
- [ ] Auth context provider (`AuthProvider`) wraps the application
- [ ] `useAuth()` hook exposes: `user`, `isAuthenticated`, `isLoading`, `login()`, `logout()`
- [ ] User object contains: `id`, `email`, `name`, `avatarUrl`, `githubUsername`
- [ ] Session validated on app load by calling marketplace API: `GET /auth/me` with cookie
- [ ] If session invalid/expired: clear cookie, set `isAuthenticated = false`
- [ ] Session refresh: on each API call, if 401 received, clear session and redirect to login

### Protected Routes
- [ ] Middleware (`middleware.ts`) checks `kaven_session` cookie for protected paths
- [ ] Protected paths: `/dashboard`, `/dashboard/*`, `/billing`, `/billing/*`, `/licenses`, `/licenses/*`
- [ ] Unauthenticated access to protected route: redirect to `/login?redirect={attempted_path}`
- [ ] After login: redirect back to the originally attempted path

### UI Components
- [ ] Header shows "Sign in with GitHub" button (with GitHub icon) when not authenticated
- [ ] Header shows user avatar (from GitHub), display name, and dropdown menu when authenticated
- [ ] Dropdown menu items: "Dashboard", "Billing", "Licenses", divider, "Sign out"
- [ ] "Sign out" calls marketplace API: `POST /auth/logout`, clears cookie, redirects to homepage
- [ ] Login page (`/login`) with: GitHub sign-in button, brief explanation, privacy policy link
- [ ] Loading skeleton while auth state is being determined (avoid flash of wrong state)

### Security
- [ ] CSRF protection: state parameter in OAuth flow, validated on callback
- [ ] Cookie flags: `HttpOnly`, `Secure`, `SameSite=Lax`, `Path=/`
- [ ] No auth tokens stored in localStorage or sessionStorage
- [ ] No auth tokens exposed in URLs after callback (clean URL after processing)
- [ ] Rate limiting awareness: handle 429 responses from marketplace API gracefully

## Technical Notes

- **OAuth flow sequence**:
  ```
  1. User clicks "Sign in with GitHub"
  2. Browser → GET /auth/github?redirect_uri=https://kaven.site/auth/callback&state={csrf_token}
  3. Marketplace → GitHub OAuth authorize URL
  4. GitHub → User authorizes → GitHub callback to marketplace
  5. Marketplace → Creates session → Redirects to kaven.site/auth/callback?token={jwt}&state={csrf_token}
  6. kaven-site /auth/callback → Validates state, sets HTTP-only cookie, redirects to /dashboard
  ```
- **Auth context implementation**:
  ```typescript
  // lib/auth/auth-context.tsx
  interface AuthUser {
    id: string;
    email: string;
    name: string;
    avatarUrl: string;
    githubUsername: string;
  }
  interface AuthContextType {
    user: AuthUser | null;
    isAuthenticated: boolean;
    isLoading: boolean;
    login: () => void;     // Redirects to OAuth flow
    logout: () => Promise<void>;
  }
  ```
- **Middleware**: Next.js `middleware.ts` at project root. Checks cookie existence (not validation — that's API's job). Lightweight to avoid latency.
  ```typescript
  // middleware.ts
  import { NextResponse } from 'next/server';
  import type { NextRequest } from 'next/server';
  export function middleware(request: NextRequest) {
    const session = request.cookies.get('kaven_session');
    if (!session) {
      const loginUrl = new URL('/login', request.url);
      loginUrl.searchParams.set('redirect', request.nextUrl.pathname);
      return NextResponse.redirect(loginUrl);
    }
    return NextResponse.next();
  }
  export const config = {
    matcher: ['/dashboard/:path*', '/billing/:path*', '/licenses/:path*'],
  };
  ```
- **API proxy**: Create Next.js API route (`app/api/auth/[...path]/route.ts`) to proxy auth requests to marketplace API. This avoids CORS issues and keeps the marketplace API URL server-side only.
- **File structure**:
  ```
  app/
  ├── login/page.tsx                    # Login page
  ├── auth/callback/page.tsx            # OAuth callback handler (client component)
  ├── api/auth/[...path]/route.ts       # Proxy to marketplace auth API
  middleware.ts                          # Route protection
  lib/auth/
  ├── auth-context.tsx                  # AuthProvider + useAuth hook
  ├── auth-api.ts                       # API calls (login, logout, getMe)
  └── csrf.ts                           # CSRF token generation/validation
  components/auth/
  ├── LoginButton.tsx                   # "Sign in with GitHub" button
  ├── UserMenu.tsx                      # Authenticated user dropdown
  └── AuthGuard.tsx                     # Client-side auth check wrapper
  ```
- **Dev mode**: When `NEXT_PUBLIC_MARKETPLACE_API_URL` is not set, mock auth with a test user object stored in cookie. Enables frontend development without marketplace running.
