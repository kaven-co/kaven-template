## Contexto

Apos a compra, o cliente precisa de um lugar central para gerenciar suas licencas. O dashboard mostra: quais licencas estao ativas, qual tier, quando expiram, e como instalar o framework. A experiencia precisa ser simples e orientada a acao — o cliente deve conseguir ir do dashboard ao `kaven init my-project` em menos de 2 minutos.

Os dados vem do marketplace API (`GET /licenses`) que retorna as licencas associadas ao usuario autenticado. A chave de licenca e parcialmente mascarada por seguranca, com opcao de revelar.

---

## User Story

**Como** cliente que comprou uma licenca do Kaven,
**Quero** ver minhas licencas ativas, tier, e instrucoes de instalacao no dashboard,
**Para que** eu saiba exatamente o que tenho e como comecar a usar o framework imediatamente.

## Technical Notes

- **Data fetching**: Use React Server Components where possible. License data fetched server-side for initial render, client-side for refresh.
  ```typescript
  // lib/api/licenses.ts
  interface License {
    id: string;
    key: string;          // Full key (masked on display)
    maskedKey: string;     // Pre-masked: KAVEN-PRO-****-X2F9
    tier: 'starter' | 'complete' | 'pro';
    status: 'active' | 'expiring' | 'expired';
    activatedAt: string;
    expiresAt: string;
    projectsUsed: number;
    projectsAllowed: number;
    tenantsUsed: number;
    tenantsAllowed: number | null;  // null = unlimited
    usage?: {
      totalInstalls: number;
      lastActivity: string;
    };
  }
  ```
- **License key masking**: API returns both `key` (full) and `maskedKey`. Display `maskedKey` by default. "Reveal" toggles to `key` with 10-second auto-revert timer.
- **Clipboard**: Use `navigator.clipboard.writeText()` with toast notification on success.
- **SWR or React Query**: Use `swr` for client-side data fetching with stale-while-revalidate. Lightweight and Next.js-native.
- **File structure**:
  ```
  app/dashboard/
  ├── page.tsx                          # Dashboard page (server component wrapper)
  ├── loading.tsx                       # Loading skeleton
  ├── error.tsx                         # Error boundary
  components/dashboard/
  ├── DashboardHeader.tsx               # Welcome message + stats
  ├── LicenseCard.tsx                   # Individual license card
  ├── LicenseGrid.tsx                   # Grid of license cards
  ├── LicenseKeyDisplay.tsx             # Masked key with reveal/copy
  ├── InstallationGuide.tsx             # Step-by-step setup instructions
  ├── QuickLinks.tsx                    # Navigation cards
  └── EmptyState.tsx                    # No licenses CTA
  lib/api/
  └── licenses.ts                       # License API client
  ```
- **Status color mapping**:
  - Active: `text-green-600 bg-green-50` (light) / `text-green-400 bg-green-900/20` (dark)
  - Expiring: `text-yellow-600 bg-yellow-50` / `text-yellow-400 bg-yellow-900/20`
  - Expired: `text-red-600 bg-red-50` / `text-red-400 bg-red-900/20`
