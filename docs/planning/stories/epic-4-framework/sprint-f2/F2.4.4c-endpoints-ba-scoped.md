## Contexto

Esta é a story que **entrega valor de produto** do épico F2.4: endpoints novos BA-scoped e `EntitlementService` scope-aware. Depende de F2.4.4a (helpers + flag) e F2.4.4b (webhook não polui mais campo legado).

---

## Escopo

### 1. EntitlementService scope-aware

- Aceitar `scope: 'TENANT' | 'BILLING_ACCOUNT'` no parâmetro de verificação.
- Lookup de Features respeita o `FeatureScope` recém-adicionado (F2.4.1).
- Backwards compatible: calls sem `scope` explícito default `TENANT` (mantém F2.1/F2.2 funcionais).

### 2. Endpoints BA-scoped (leitura)

- `GET /api/v1/billing-accounts/:baId` — detalhes da BA (owner, stripeCustomerId mascarado, created_at).
- `GET /api/v1/billing-accounts/:baId/subscriptions` — subs da BA.
- `GET /api/v1/billing-accounts/:baId/invoices` — invoices da BA.

Todos usam `canActOnBillingAccount` do F2.4.4a.

### 3. Payment service migration (`subscription.service`, `invoice.service`)

- Leitura de subs/invoices passa a usar `WHERE billing_account_id = ?` (não `tenant_id`).
- Durante transição, se `ALLOW_LEGACY_TENANT_STRIPE_ID=true`, fallback para `tenant_id` lookup só em **read** (writes sempre BA-scoped).
- `stripeCustomerId` read: **apenas** de `billingAccount`. Nunca de `Tenant` (fallback só via flag explícita).

### 4. Testes

- Integration tests para cada endpoint BA-scoped.
- AuthZ tests: cross-BA access bloqueado, SUPER_ADMIN bypass, owner path.
- EntitlementService: scope-aware tests (quota shared entre tenants da mesma BA).

## Acceptance Criteria

### AC1 — EntitlementService
- [ ] Aceita parâmetro `scope` opcional; default `TENANT`.
- [ ] Quando `scope === 'BILLING_ACCOUNT'`, quota é avaliada sobre BA + todos seus tenants.
- [ ] Tests: quota `MAX_TENANTS` compartilhada entre tenants da mesma BA (novo test case).

### AC2 — Endpoints
- [ ] 3 endpoints GET BA-scoped implementados.
- [ ] `canActOnBillingAccount` aplicado em cada um.
- [ ] Tests: 200 happy path, 403 cross-BA, 404 BA não-existente, 401 unauth.

### AC3 — Payment service
- [ ] `subscription.service` e `invoice.service` migrados para BA-scoped reads.
- [ ] Fallback `tenant_id` atrás da flag `ALLOW_LEGACY_TENANT_STRIPE_ID`.
- [ ] `stripeCustomerId` lido exclusivamente de `billingAccount`.

### AC4 — Quality gates
- [ ] Tests + typecheck + lint verdes.
- [ ] `pnpm test --filter api` inclui ≥ 10 novos testes para endpoints BA-scoped.

### AC5 — QA gate
- [ ] architecture-strategist review: god-method não reintroduzido.
- [ ] security-sentinel review: AuthZ convergente entre endpoints.
- [ ] data-integrity-guardian review: no race conditions em BA reads.
