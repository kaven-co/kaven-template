# F2.1 — MAX_TENANTS Plan Enforcement

> **Epic:** 4 (Framework — Pricing Enforcement) | **Sprint:** F2 | **Agent:** @dev
> **Priority:** P0
> **Depends on:** nenhuma

---

## Contexto

O kaven-framework já tem o sistema de `PlanFeature` com `requireFeature()` e `PlanFeature.limitValue` (-1 = ilimitado). Exemplos ativos no seed: `MAX_EMPLOYEES`, `MAX_CAMPAIGNS`, `MAX_PROPERTIES`. O `feature-guard.middleware.ts` já suporta o tipo `QUOTA` — retorna 403 com sugestão de upgrade quando o limite é excedido.

O que **não existe** é `MAX_TENANTS`. Os 61 capabilities mapeados em `capabilities.seed.ts` não incluem nenhum relacionado a tenants. A infra está pronta; falta apenas criar a feature e plugar o enforcement no endpoint de criação de tenant.

**Limites aprovados (decisão final 2026-04-17):**
- Starter: **3 tenants**
- Builder: **10 tenants**
- Pro: **ilimitado** (-1)

*Benchmark: Linear (2→5→∞), Railway (50→100→∞), padrão de mercado 3→10→∞.*

---

## User Story

**Como** tenant admin tentando criar um novo tenant além do limite do meu plano,
**Quero** receber uma resposta clara de bloqueio com informação de upgrade,
**Para que** eu entenda o limite do meu plano e tenha um caminho direto para expandir.

- [ ] `requireFeature('MAX_TENANTS')` adicionado no endpoint de criação de tenant

    "availableUpgrades": [...],

  }

- [ ] Todos os testes existentes passando (sem regressões)

## Technical Notes

**Arquivos de referência para o padrão existente:**
- `packages/database/prisma/seeds/capabilities.seed.ts` — exemplos: `MAX_EMPLOYEES`, `MAX_CAMPAIGNS`
- `apps/api/src/middleware/feature-guard.middleware.ts` — `requireFeature()` com suporte QUOTA
- `apps/api/src/services/entitlement.service.ts` — `getActiveSubscription()` e check de limites

**Pattern a seguir para adicionar a feature:**
```typescript
// Em capabilities.seed.ts — adicionar junto aos outros MAX_*
{
  code: 'MAX_TENANTS',
  name: 'Maximum Tenants',
  description: 'Maximum number of tenants per project',
  type: 'QUOTA',
  category: 'LIMITS',
  planFeatures: {
    create: [
      { plan: { connect: { name: 'STARTER' } }, limitValue: 3 },
      { plan: { connect: { name: 'BUILDER' } }, limitValue: 10 },
      { plan: { connect: { name: 'PRO' } }, limitValue: -1 },
    ]
  }
}
```

**Endpoint de criação de tenant** — localizar em `apps/api/src/modules/tenants/` e adicionar:
```typescript
await requireFeature(request, reply, 'MAX_TENANTS');
```

**Resposta de erro já implementada** pelo `feature-guard.middleware.ts` para QUOTA — não precisa de código extra.
