## Contexto

Auditoria técnica de 2026-04-20 identificou 3 branches remotas órfãs no repositório `kaven-framework`:

- `feat/f2.4.4b-webhook-hardening`
- `feat/f2.4.5-drop-legacy-stripe-columns`
- `fix/kaven-payments-module-injection-schema`

Branches órfãs acumulam ao longo do tempo e criam ruído na lista de branches remotas — dificultando navegação, aumentando o risco de rebase acidental sobre branch errada e confundindo novos colaboradores sobre o estado real do desenvolvimento.

**Antes de deletar:** verificar que nenhuma branch contém commits não mergeados relevantes que não estejam na `main`.

---

## Escopo

### 1. Auditoria de commits não mergeados

Para cada branch:

```bash
# Verificar se há commits na branch que não estão na main
git log origin/main..origin/feat/f2.4.4b-webhook-hardening --oneline
git log origin/main..origin/feat/f2.4.5-drop-legacy-stripe-columns --oneline
git log origin/main..origin/fix/kaven-payments-module-injection-schema --oneline
```

- Se a saída for vazia: branch pode ser deletada com segurança.
- Se houver commits únicos: revisar se são relevantes ou se foram superseded por outro PR. Documentar a decisão no PR.

### 2. Deleção das branches confirmadas como seguras

```bash
git push origin --delete feat/f2.4.4b-webhook-hardening
git push origin --delete feat/f2.4.5-drop-legacy-stripe-columns
git push origin --delete fix/kaven-payments-module-injection-schema
```

### 3. Verificação pós-deleção

```bash
# Confirmar que as branches foram removidas do remote
git branch -r | grep -E "f2.4.4b|f2.4.5|payments-module-injection"
# Esperado: zero resultados
```

### 4. Prune local

```bash
git fetch --prune
```

### 5. Auditoria de outras branches órfãs (opcional)

- Listar todas as branches remotas com mais de 30 dias sem commit:
  ```bash
  git for-each-ref --sort=committerdate refs/remotes/ \
    --format='%(refname:short) %(committerdate:relative)' \
    | grep -v "main\|HEAD"
  ```
- Se houver branches antigas não identificadas no finding original, documentar e deixar para story de manutenção futura — **não deletar** sem aprovação explícita.

## Acceptance Criteria

### AC1 — Auditoria documentada
- [ ] Para cada uma das 3 branches: output do `git log origin/main..<branch>` presente no PR description.
- [ ] Decisão de deleção (ou preservação) documentada por branch.

### AC2 — Branches deletadas
- [ ] `git branch -r` não lista `feat/f2.4.4b-webhook-hardening`, `feat/f2.4.5-drop-legacy-stripe-columns`, nem `fix/kaven-payments-module-injection-schema`.
- [ ] `git push origin --delete` executado sem erros para cada branch confirmada como segura.

### AC3 — Nenhum commit perdido
- [ ] Se alguma branch tinha commits únicos: eles foram avaliados e descartados conscientemente (documentado) ou preservados em outra branch/PR.

### AC4 — Local limpo
- [ ] `git fetch --prune` executado após deleção.
- [ ] Nenhuma referência local às branches deletadas.

---

## Out of scope

- Deleção de branches além das 3 identificadas sem aprovação explícita.
- Auditoria de branches nos outros repos do ecossistema (marketplace, cli, template).
- Mudanças de código ou configuração.
- Arquivamento de PRs relacionados (operação no GitHub, não no git local).
