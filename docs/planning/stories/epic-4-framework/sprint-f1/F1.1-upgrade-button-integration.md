# F1.1 — Upgrade Button Integration

> **Epic:** 4 (Framework — Upgrade Flow & RC1) | **Sprint:** F1 | **Agent:** @dev
> **Priority:** P0
> **Depends on:** M2.5

---

## Contexto

Este e o ultimo tech debt item pendente do kaven-framework (FE-M2: Upgrade Flow Not Integrated). O botao "Upgrade" na pagina de planos do Tenant App existe na UI mas nao dispara pagamento real. A decisao de gateway foi tomada: Paddle para o marketplace/licencas. Esta story integra o botao com o Paddle checkout via marketplace API.

No kaven-framework, o Tenant App (Next.js) tem uma pagina de planos (`/settings/plans` ou `/pricing`) que mostra os tiers disponiveis com features. O "Upgrade" button precisa criar uma checkout session no marketplace e abrir o Paddle overlay ou redirect para checkout URL. Ao completar, o webhook handler (F1.2) atualiza o plano no backend.

---

## User Story

**Como** tenant admin usando o Kaven,
**Quero** clicar "Upgrade" na pagina de planos e completar o pagamento inline,
**Para que** meu plano seja atualizado imediatamente sem sair da aplicacao.

## Acceptance Criteria

### Paddle.js Integration
- [ ] Paddle.js SDK loaded in Tenant App via `next/script` with `strategy="afterInteractive"`
- [ ] Paddle initialized with `NEXT_PUBLIC_PADDLE_SELLER_ID` and `NEXT_PUBLIC_PADDLE_ENVIRONMENT`
- [ ] Paddle environment configurable: `sandbox` for development, `production` for live
- [ ] Paddle.js version pinned to avoid breaking changes

### Upgrade Button
- [ ] "Upgrade" button on each plan card (except current plan) calls upgrade handler
- [ ] Current plan card shows "Current Plan" badge instead of upgrade button
- [ ] Downgrade button shows "Downgrade" with different styling (less prominent)
- [ ] Button shows loading spinner while checkout session is being created
- [ ] Button disabled during loading to prevent double-clicks

### Checkout Flow
- [ ] Click "Upgrade" → `POST /api/checkout/session` (Tenant App API route)
- [ ] API route proxies to marketplace API: `POST /checkout/session` with payload:
  ```json
  {
    "tierId": "pro",
    "billingCycle": "monthly",
    "customerEmail": "tenant-admin@example.com",
    "metadata": {
      "tenantId": "current-tenant-uuid",
      "currentPlan": "starter",
      "source": "tenant-app"
    }
  }
  ```
- [ ] On success: receive Paddle checkout URL or transaction ID
- [ ] If using Paddle overlay: `Paddle.Checkout.open({ transactionId })` opens inline checkout
- [ ] If using redirect: `window.location.href = checkoutUrl`
- [ ] Paddle overlay preferred for better UX (user stays on page)

### Paddle Event Handling
- [ ] Listen to Paddle.js events on the client side:
  - `checkout.completed` → show success toast ("Plan upgraded successfully!"), refresh plan data
  - `checkout.closed` → show info toast ("Checkout cancelled"), re-enable button
  - `checkout.error` → show error toast with Paddle error message, re-enable button
- [ ] On `checkout.completed`: call `POST /api/plan/refresh` to force-refresh the tenant's plan from backend
- [ ] Plan data refresh shows new tier, updated features, and confirmation message
- [ ] Success state includes confetti animation or highlight effect on new plan card

### Error Handling
- [ ] Network error creating checkout session: show "Unable to connect. Please try again." with retry button
- [ ] Marketplace API error (4xx): show specific error message from API response
- [ ] Paddle loading failure: show fallback message "Payment system unavailable. Please try again later." with link to support
- [ ] If user's email is not verified: show "Please verify your email first" with link to profile settings

### Environment Variables
- [ ] `NEXT_PUBLIC_PADDLE_SELLER_ID` — Paddle seller ID
- [ ] `NEXT_PUBLIC_PADDLE_ENVIRONMENT` — "sandbox" or "production"
- [ ] `MARKETPLACE_API_URL` — server-side only, marketplace API base URL
- [ ] `MARKETPLACE_API_KEY` — server-side only, API key for marketplace auth

## Technical Notes

- **Paddle.js integration pattern**:
  ```typescript
  // lib/paddle.ts
  import Script from 'next/script';

  export function PaddleScript() {
    return (
      <Script
        src="https://cdn.paddle.com/paddle/v2/paddle.js"
        strategy="afterInteractive"
        onLoad={() => {
          window.Paddle.Initialize({
            token: process.env.NEXT_PUBLIC_PADDLE_SELLER_ID!,
            environment: process.env.NEXT_PUBLIC_PADDLE_ENVIRONMENT as 'sandbox' | 'production',
          });
        }}
      />
    );
  }
  ```
- **Checkout session creation** (API route to keep marketplace URL server-side):
  ```typescript
  // app/api/checkout/session/route.ts
  export async function POST(request: Request) {
    const body = await request.json();
    const response = await fetch(`${MARKETPLACE_API_URL}/checkout/session`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${MARKETPLACE_API_KEY}`,
      },
      body: JSON.stringify(body),
    });
    return Response.json(await response.json());
  }
  ```
