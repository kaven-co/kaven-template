## Contexto

O kaven-framework tem codigo de pagamento (billing service, payment service, subscription service, webhook handler, Prisma models) integrado diretamente no monorepo. Para o marketplace funcionar, esse codigo precisa ser extraido em um modulo instalavel via `kaven module add payments`. Isso valida o sistema de modulos do kaven-cli em producao e cria o primeiro modulo oficial do marketplace.

O modulo usa o formato `module.json` do kaven-cli, que define: arquivos incluidos, injections (markers onde o CLI insere codigo no app.ts, routes/index.ts, etc.), dependencias npm, variaveis de ambiente necessarias, e Prisma models a serem merged no schema.

O padrao de markers (`// [KAVEN_MODULE:payments BEGIN]` ... `// [KAVEN_MODULE:payments END]`) garante idempotencia: instalar duas vezes nao duplica codigo, remover deixa o projeto limpo.

---

## User Story

**Como** desenvolvedor usando o Kaven,
**Quero** instalar o modulo de pagamentos via CLI com um unico comando,
**Para que** eu tenha billing, subscriptions, e webhook handling configurados sem copiar codigo manualmente.

## Technical Notes

- **Module extraction strategy**: Don't move files — copy them into a module package directory, then ensure the framework works both with the code inline (current state) and as an installed module.
- **Module package structure**:
  ```
  kaven-payments/
  ├── module.json                       # Module manifest
  ├── README.md                         # Module documentation
  ├── backend/
  │   ├── routes/
  │   │   ├── billing.ts
  │   │   ├── payments.ts
  │   │   ├── subscriptions.ts
  │   │   └── webhooks/
  │   │       ├── stripe.ts
  │   │       └── paddle.ts
  │   ├── services/
  │   │   ├── billing.service.ts
  │   │   ├── payment.service.ts
  │   │   ├── subscription.service.ts
  │   │   ├── stripe-client.ts
  │   │   └── paddle-client.ts
  │   └── prisma/
  │       └── payments.prisma           # Prisma models for merge
  ├── frontend/
  │   ├── components/
  │   │   ├── billing/
  │   │   └── plans/
  │   └── lib/
  │       └── paddle.ts
  └── tests/
      ├── billing.test.ts
      ├── payments.test.ts
      └── subscriptions.test.ts
  ```
- **kaven-cli module add flow**:
  1. Download/extract module package
  2. Copy files from `backend/` and `frontend/` to target directories
  3. Apply injections from `module.json` using marker pattern
  4. Run `npm install` for new dependencies
  5. Run `npx prisma generate` for schema changes
  6. Print post-install instructions (env vars to set)
- **kaven-cli module remove flow**:
  1. Remove all files listed in `module.json.files`
  2. Remove all code between markers in injected files
  3. Run `npm install` to clean unused dependencies
  4. Print post-remove instructions (env vars to clean up)
- **Version compatibility**: Module should declare minimum kaven-framework version in module.json: `"kavenVersion": ">=1.0.0-rc1"`
