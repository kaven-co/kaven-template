## Contexto

O framework **não tem módulo de suporte/tickets**. Decisão aprovada (2026-04-17): instalar **Chatwoot self-hosted** (licença MIT) no Cloud Run existente (`kaven-prod`, `kaventhecreator@gmail.com`), custo zero, como solução de suporte até a primeira venda.

**Roadmap de suporte:**
| Fase | Solução | Custo |
|------|---------|-------|
| **Agora (pre-revenue)** | Chatwoot self-hosted no Cloud Run | $0 |
| **Pós-primeira venda** | Plain.com Horizon ($269/mês) | $269/mês |

**Por que Chatwoot:**
- Licença MIT — self-hosted sem custo
- Cloud Run kaven-prod já existe e está operacional
- REST API completa para integração futura com o framework
- Suporta email + live chat
- Container oficial disponível (`chatwoot/chatwoot`)

**Infra disponível:**
- Cloud Run: `kaven-prod` (us-east1, `kaventhecreator@gmail.com`)
- Neon DB: `ep-late-leaf-aivgdi2v-pooler` (us-east-1) — criar schema separado `chatwoot`
- Upstash Redis: `evolving-cow-17329.upstash.io:6379` (TLS)
- Cloudflare Zone: `f56c57fd1fea36e1609da92b7268ba79` (kaven.site)
- Resend (email — SMTP relay ou API)

## Technical Notes

**Variáveis de ambiente necessárias no Cloud Run:**
```bash
# App
SECRET_KEY_BASE=<gerar com: openssl rand -hex 64>
FRONTEND_URL=https://chat.kaven.site
FORCE_SSL=true
RAILS_ENV=production

# Database (Neon — schema chatwoot)
DATABASE_URL=postgresql://[user]:[pass]@ep-late-leaf-aivgdi2v-pooler.us-east-1.aws.neon.tech/kaven_framework?schema=chatwoot&sslmode=require

# Redis (Upstash TLS)
REDIS_URL=rediss://:@evolving-cow-17329.upstash.io:6379

# Email (Resend SMTP)
MAILER_SENDER_EMAIL=support@kaven.site
SMTP_ADDRESS=smtp.resend.com
SMTP_PORT=587
SMTP_USERNAME=resend
SMTP_PASSWORD=<RESEND_API_KEY>
SMTP_ENABLE_STARTTLS_AUTO=true

# Storage (usar Cloud Storage GCP ou S3 AWS existente)
ACTIVE_STORAGE_SERVICE=local  # mudar para s3 se necessário
```

**Comando de deploy Cloud Run:**
```bash
gcloud run deploy kaven-chatwoot \
  --image chatwoot/chatwoot:latest \
  --platform managed \
  --region us-east1 \
  --project kaven-prod \
  --port 3000 \
  --memory 1Gi \
  --cpu 1 \
  --min-instances 0 \
  --max-instances 2 \
  --env-vars-file chatwoot.env.yaml \
  --command bundle \
  --args "exec,rails,server,-b,0.0.0.0,-p,3000"
```

**Setup inicial (executar após primeiro deploy):**
```bash
# Rodar via Cloud Run Jobs ou exec no container
bundle exec rails db:chatwoot_prepare
bundle exec rails db:seed
```

**DNS Cloudflare:**
```
chat.kaven.site CNAME → [cloud-run-service-url].a.run.app
```

**Sidekiq (jobs assíncronos):**
- Chatwoot precisa de Sidekiq rodando. Opções:
  1. Segundo Cloud Run service para workers (recomendado)
  2. `DISABLE_SIDEKIQ=false` no mesmo container (não recomendado para prod)
