## Contexto

O `AIOX-KAVEN-INTEGRATION.md` (GAP-7) identifica que **duas fontes de métricas coexistem isoladas**:

1. **Métricas AIOX**: sessões de agents, tempo por phase, stories completadas, QA pass/fail rate — vivem apenas em arquivos de sessão dentro do `.aiox/` runtime (gitignored)
2. **Métricas Kaven**: Prometheus (`localhost:9090`), Grafana (`localhost:3003`), Loki + Promtail — monitoram o runtime da API mas não têm awareness do desenvolvimento (quais features estão sendo construídas, por quais agents)

**Consequência prática:** Não é possível correlacionar:
- "O QA pass rate caiu nesta semana" com "Kai delegou 80% das stories direto para Bolt sem Architecture Review"
- "Tempo médio por story aumentou" com "Sprint com muita refatoração de schema"
- "API latência subiu em produção" com "Feature X foi deployada via Agent Y no dia Z"

Esta story implementa um **exportador de métricas AIOX** que escreve no formato Prometheus, e um **dashboard Grafana** "AIOX Development Health" que correlaciona atividade de agents com métricas do sistema.

## User Story

**As a** developer using AIOX + Kaven in a project,
**I want to** see AIOX development metrics (agent velocity, QA rates, story throughput) in my Grafana dashboard,
**So that** I can correlate development patterns with production performance and improve my agent workflow.

## Technical Notes

### Estrutura do Exportador

```javascript
// .aiox-core/core/orchestration/metrics-exporter.js

const http = require('http');
const fs = require('fs');
const path = require('path');

class AIoxMetricsExporter {
  constructor(options = {}) {
    this.port = options.port || 9091;
    this.aioxRuntimeDir = options.aioxRuntimeDir || path.join(process.cwd(), '.aiox');
    this.metrics = this.initMetrics();
    this.updateInterval = options.updateInterval || 30_000;
  }

  initMetrics() {
    return {
      stories_completed: new Map(),  // key: `${phase}:${squad}:${agent}` → count
      stories_active: new Map(),     // key: squad → count
      story_durations: [],           // { phase, complexity, duration_ms }
      qa_pass: new Map(),            // key: agent → count
      qa_fail: new Map(),            // key: `${agent}:${failure_type}` → count
      agent_sessions: new Map(),     // key: `${agent}:${squad}` → count
      handoffs: new Map(),           // key: `${from}:${to}` → count
    };
  }

  collectFromSessionFiles() {
    const sessionsDir = path.join(this.aioxRuntimeDir, 'sessions');
    if (!fs.existsSync(sessionsDir)) return;

    const files = fs.readdirSync(sessionsDir).filter(f => f.endsWith('.json'));
    for (const file of files) {
      try {
        const session = JSON.parse(
          fs.readFileSync(path.join(sessionsDir, file), 'utf-8')
        );
        this.processSession(session);
      } catch (err) {
        // log e continua — não falha o exportador
        console.error(`[aiox-metrics] Failed to process ${file}:`, err.message);
      }
    }
  }

  formatPrometheus() {
    const lines = [];
    // Formatar cada métrica no text format do Prometheus
    // ...
    return lines.join('\n');
  }

  start() {
    this.collectFromSessionFiles();
    setInterval(() => this.collectFromSessionFiles(), this.updateInterval);

    const server = http.createServer((req, res) => {
      if (req.url === '/metrics') {
        res.writeHead(200, { 'Content-Type': 'text/plain; version=0.0.4' });
        res.end(this.formatPrometheus());
      } else {
        res.writeHead(404);
        res.end('Not found');
      }
    });

    server.listen(this.port, () => {
      console.log(`[aiox-metrics] Exporter running on :${this.port}`);
    });
  }
}

module.exports = { AIoxMetricsExporter };
```
