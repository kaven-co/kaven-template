# D2.1 — Service Token para AI Agents

> **Epic:** 6 (Deploy) | **Sprint:** D2 | **Agent:** @kaven-api-dev
> **Priority:** P1
> **Depends on:** ServiceToken model já existe no DB (commit 8347209e — feat(db): add ServiceToken model and migration for service-to-service auth)

---

## Contexto

O `AIOX-KAVEN-INTEGRATION.md` (GAP-5, Seção 7) define o princípio arquitetural de **API Unificada para Humanos + Agentes**: a mesma Fastify API que serve os usuários humanos (via JWT) deve também servir os AI agents do kaven-squad (via Service Token).

A tabela `ServiceToken` e o enum `CallerType` (incluindo `AGENT`) já foram adicionados ao Prisma schema e migrados (commits `8347209e` e `0a340b05`). O que ainda **não existe** é:

- O middleware `service-token.middleware.ts` que valida o header `X-Service-Token`
- A role `AGENT` no sistema de RBAC
- As rotas `POST /service-tokens` e `DELETE /service-tokens/:id` (admin only)
- O cron de revogação automática por inatividade (7 dias)
- A capability `MAX_AGENT_API_CALLS_HOUR` (nova capability a adicionar às 48 existentes)
- Testes de isolamento: agent de tenant A não acessa dados do tenant B

**Risco de segurança (do spec original):** Sem rate limiting por agent e sem revogação automática, um token comprometido pode exfiltrar dados de todos os tenants que tiver acesso. As mitigações abaixo são obrigatórias, não opcionais.

## User Story

**As a** kaven-squad agent (Kai, Atlas, Bolt, etc.) operating on behalf of a tenant,
**I want to** authenticate against the Kaven API using a Service Token with scoped capabilities,
**So that** I can perform authorized operations on behalf of tenants without using human credentials.

## Acceptance Criteria

### Middleware

- [ ] `apps/api/src/middleware/service-token.middleware.ts` implementado
- [ ] Header `X-Service-Token: <opaque-token>` é aceito pelo middleware de auth
- [ ] Middleware valida o token contra a tabela `ServiceToken` (busca por hash do token)
- [ ] Token revogado (`revokedAt != null`) é rejeitado com `401 Unauthorized`
- [ ] Token expirado (`expiresAt < now()`) é rejeitado com `401 Unauthorized` sem consulta ao banco
- [ ] Token válido popula `request.caller` com `{ type: 'AGENT', agentId, agentSquad, tenantId, capabilities }`
- [ ] Atualiza `lastUsedAt` no banco a cada request autenticado (async, não bloqueia response)
- [ ] Rate limit por agent: respeita capability `MAX_AGENT_API_CALLS_HOUR` do tenant

### Routes

- [ ] `POST /api/service-tokens` cria novo token (requer role `ADMIN`)
- [ ] Body: `{ agentId, agentSquad, capabilities, expiresInDays (max 30) }`
- [ ] Response inclui o token raw **uma única vez** (nunca mais exibido após criação)
- [ ] Token armazenado como **hash SHA-256** no banco (nunca o valor raw)
- [ ] `GET /api/service-tokens` lista tokens do tenant com status (active/revoked/expired)
- [ ] `DELETE /api/service-tokens/:id` revoga token manualmente

### Audit Log

- [ ] `AuditLog` registra `callerType: 'AGENT'` com `metadata.agentId` e `metadata.agentSquad`
- [ ] Requests de agents são distinguíveis nos logs de requests de humanos

### Cron de Revogação

- [ ] Cron job que roda diariamente revoga tokens com `lastUsedAt < now() - 7 days`
- [ ] Tokens nunca usados com `createdAt < now() - 7 days` também são revogados
- [ ] Revogação automática cria `AuditLog` entry com `action: 'SERVICE_TOKEN_AUTO_REVOKED'`

### Segurança e Isolamento

- [ ] Testes de isolamento: agent com token do tenant A não consegue acessar dados do tenant B
- [ ] Capabilities do token são subset das capabilities do tenant — não pode escalar privilégios
- [ ] `MAX_AGENT_API_CALLS_HOUR` adicionado ao catálogo de capabilities

### Middleware Implementation

```typescript
// apps/api/src/middleware/service-token.middleware.ts

import { FastifyRequest, FastifyReply } from 'fastify';
import { createHash } from 'crypto';
import { prisma } from '@kaven/database';

export interface AgentCaller {
  type: 'AGENT';
  agentId: string;
  agentSquad: string;
  tenantId: string;
  capabilities: string[];
  serviceTokenId: string;
}

export async function serviceTokenMiddleware(
  request: FastifyRequest,
  reply: FastifyReply
): Promise<void> {
  const rawToken = request.headers['x-service-token'];
  if (!rawToken || typeof rawToken !== 'string') return;

  // Hash do token para busca no banco (nunca comparar raw)
  const tokenHash = createHash('sha256').update(rawToken).digest('hex');

  // Check expiry sem consulta ao banco (performance)
  // O token hash inclui expiry? Não — verificar só após busca

  const serviceToken = await prisma.serviceToken.findUnique({
    where: { token: tokenHash },
    select: {
      id: true,
      tenantId: true,
      agentId: true,
      agentSquad: true,
      capabilities: true,
      revokedAt: true,
      expiresAt: true,
    },
  });

  if (!serviceToken) {
    return reply.code(401).send({ error: 'Invalid service token' });
  }

  if (serviceToken.revokedAt) {
    return reply.code(401).send({ error: 'Service token revoked' });
  }

  if (serviceToken.expiresAt < new Date()) {
    return reply.code(401).send({ error: 'Service token expired' });
  }

  // Atualizar lastUsedAt de forma assíncrona (fire-and-forget)
  prisma.serviceToken.update({
    where: { id: serviceToken.id },
    data: { lastUsedAt: new Date() },
  }).catch((err) => {
    request.log.error({ err }, 'Failed to update service token lastUsedAt');
  });

  // Popular o request caller
  (request as any).caller = {
    type: 'AGENT',
    agentId: serviceToken.agentId,
    agentSquad: serviceToken.agentSquad,
    tenantId: serviceToken.tenantId,
    capabilities: serviceToken.capabilities,
    serviceTokenId: serviceToken.id,
  } satisfies AgentCaller;
}
```

### Routes

```typescript
// apps/api/src/modules/auth/routes/service-tokens.routes.ts

import { FastifyPluginAsync } from 'fastify';
import { randomBytes, createHash } from 'crypto';

const serviceTokensRoutes: FastifyPluginAsync = async (fastify) => {
  fastify.post('/service-tokens', {
    schema: {
      body: {
        type: 'object',
        required: ['agentId', 'agentSquad', 'capabilities', 'expiresInDays'],
        properties: {
          agentId: { type: 'string' },
          agentSquad: { type: 'string' },
          capabilities: { type: 'array', items: { type: 'string' } },
          expiresInDays: { type: 'number', minimum: 1, maximum: 30 },
        },
      },
    },
    preHandler: [fastify.requireRole('ADMIN')],
  }, async (request, reply) => {
    const { agentId, agentSquad, capabilities, expiresInDays } = request.body as any;
    const { tenantId } = request.user;

    const rawToken = randomBytes(32).toString('hex');
    const tokenHash = createHash('sha256').update(rawToken).digest('hex');

    const expiresAt = new Date();
    expiresAt.setDate(expiresAt.getDate() + expiresInDays);

    const serviceToken = await prisma.serviceToken.create({
      data: {
        tenantId,
        agentId,
        agentSquad,
        token: tokenHash,
        capabilities,
        expiresAt,
      },
    });

    await prisma.auditLog.create({
      data: {
        tenantId,
        action: 'SERVICE_TOKEN_CREATED',
        callerType: 'HUMAN',
        metadata: { agentId, agentSquad, serviceTokenId: serviceToken.id },
      },
    });

    // Token raw retornado apenas uma vez
    return reply.code(201).send({ id: serviceToken.id, token: rawToken });
  });

  fastify.delete('/service-tokens/:id', {
    preHandler: [fastify.requireRole('ADMIN')],
  }, async (request, reply) => {
    const { id } = request.params as { id: string };
    const { tenantId } = request.user;

    await prisma.serviceToken.updateMany({
      where: { id, tenantId },
      data: { revokedAt: new Date() },
    });

    return reply.code(204).send();
  });
};
```

### Cron de Revogação

```typescript
// apps/api/src/jobs/service-token-cleanup.job.ts

export async function revokeInactiveServiceTokens() {
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - 7);

  const revoked = await prisma.serviceToken.updateMany({
    where: {
      revokedAt: null,
      OR: [
        { lastUsedAt: { lt: cutoff } },
        { lastUsedAt: null, createdAt: { lt: cutoff } },
      ],
    },
    data: { revokedAt: new Date() },
  });

  // AuditLog para rastreabilidade
  if (revoked.count > 0) {
    await prisma.auditLog.create({
      data: {
        action: 'SERVICE_TOKEN_AUTO_REVOKED',
        callerType: 'SYSTEM',
        metadata: { count: revoked.count, cutoff: cutoff.toISOString() },
      },
    });
  }
}
```
