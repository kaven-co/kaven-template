> **Epic:** 6 (Deploy & Launch) | **Sprint:** D1 | **Agent:** @devops (Deploy/Gage)

## Contexto

O `kaven-co/kaven-marketplace` é a API Fastify que precisa estar em produção para:
- CLI autenticar (`kaven auth login`)
- CLI listar/instalar módulos (`kaven marketplace browse`)
- kaven-site fazer checkout (Paddle webhooks)

Railway foi escolhido por suportar Fastify nativamente, ter auto-deploy do GitHub, e custar ~$20/mês para o uso esperado.

URL final: `https://marketplace.kaven.site`

## User Story

**Como** DevOps do Kaven,
**Quero** fazer o deploy do kaven-marketplace API no Railway,
**Para que** o CLI e o kaven-site possam se conectar ao marketplace em produção.

## Acceptance Criteria

- [ ] Conta Railway criada/acessada em railway.app
- [ ] Projeto Railway criado linkado ao repo `kaven-co/kaven-marketplace`
- [ ] Todos os env vars configurados (ver Technical Notes)
- [ ] `npx prisma migrate deploy` executado em produção com sucesso
- [ ] Deploy inicial bem-sucedido (build green, app running)
- [ ] Custom domain configurado: `marketplace.kaven.site`
- [ ] Health check verde: `curl https://marketplace.kaven.site/health` → `{"status":"ok",...}`
- [ ] Swagger docs acessível: `https://marketplace.kaven.site/docs`
- [ ] Módulos endpoint respondendo: `curl https://marketplace.kaven.site/modules` → `[]` (array vazio)
- [ ] Logs sem erros críticos no Railway dashboard

## Technical Notes

```bash
# 1. Instalar Railway CLI
npm install -g @railway/cli

# 2. Login
railway login

# 3. Na raiz do repo kaven-marketplace
railway init
railway link  # linkar ao repo GitHub

# 4. Configurar environment variables (via Railway dashboard ou CLI)
railway variables set NODE_ENV=production
railway variables set PORT=3001
railway variables set APP_URL=https://marketplace.kaven.site
railway variables set FRONTEND_URL=https://kaven.site
railway variables set DATABASE_URL="postgresql://..."  # da D1.1
railway variables set REDIS_URL="rediss://..."          # da D1.2
railway variables set JWT_SECRET="..."                  # da D1.5
railway variables set JWT_ACCESS_EXPIRY=15m
railway variables set JWT_REFRESH_EXPIRY=7d
railway variables set GITHUB_CLIENT_ID="..."            # da D1.3
railway variables set GITHUB_CLIENT_SECRET="..."        # da D1.3
railway variables set GITHUB_CALLBACK_URL=https://marketplace.kaven.site/auth/github/callback
railway variables set AWS_REGION=us-east-1
railway variables set AWS_ACCESS_KEY_ID="..."           # IAM user kaven-platform-api
railway variables set AWS_SECRET_ACCESS_KEY="..."       # IAM user kaven-platform-api
railway variables set S3_BUCKET_NAME=kaven-marketplace-artifacts
railway variables set PADDLE_ENVIRONMENT=production
railway variables set PADDLE_API_KEY="..."              # da D1.4
railway variables set PADDLE_WEBHOOK_SECRET="..."       # da D1.4
railway variables set PADDLE_PRICE_STARTER="..."        # da D1.4
railway variables set PADDLE_PRICE_COMPLETE="..."       # da D1.4
railway variables set PADDLE_PRICE_PRO="..."            # da D1.4
railway variables set CORS_ORIGINS=https://kaven.site,https://marketplace.kaven.site
railway variables set S3_ALLOWED_DOMAINS=s3.amazonaws.com,s3.us-east-1.amazonaws.com
railway variables set SIGNING_PUBLIC_KEY="..."          # da D1.5
railway variables set DOCS_ENABLED=true

# 5. Configurar build command no railway.toml ou Railway dashboard:
# Build: npm ci && npx prisma generate && npx prisma migrate deploy && npm run build
# Start: node dist/server.js

# 6. Deploy
railway up

# 7. Configurar custom domain
railway domain add marketplace.kaven.site
```

**Validação pós-deploy:**
```bash
# Health check
curl https://marketplace.kaven.site/health
# Esperado: {"status":"ok","version":"0.3.0","database":"connected","redis":"connected"}

# Módulos (deve ser array vazio antes do seed)
curl https://marketplace.kaven.site/modules
# Esperado: []

# Docs
curl https://marketplace.kaven.site/docs
# Deve retornar HTML do Swagger UI
```

**railway.toml** (criar se não existir):
```toml
[build]
command = "npm ci && npx prisma generate && npx prisma migrate deploy && npm run build"

[deploy]
startCommand = "node dist/server.js"
healthcheckPath = "/health"
healthcheckTimeout = 30
restartPolicyType = "on-failure"
```
