# D1.3 — Create GitHub OAuth App — Marketplace Authentication

> **Epic:** 6 (Deploy & Launch) | **Sprint:** D1 | **Agent:** @devops (Deploy/Gage)
> **Priority:** P0 — CLI usa GitHub OAuth para login no marketplace
> **Depends on:** Nenhum (mas precisa da URL do Railway para o callback — pode criar placeholder)

---

## Contexto

O kaven-marketplace usa GitHub OAuth para autenticar desenvolvedores. O fluxo é:
1. CLI abre browser: `https://marketplace.kaven.site/auth/github`
2. GitHub redireciona para callback
3. Marketplace troca code por access_token
4. JWT é gerado e retornado ao CLI

O OAuth App precisa ser criado na org `kaven-co` ou na conta pessoal.

## User Story

**Como** DevOps do Kaven,
**Quero** criar um GitHub OAuth App para o marketplace,
**Para que** os desenvolvedores possam autenticar no marketplace via GitHub.

## Acceptance Criteria

- [ ] GitHub OAuth App criado em github.com/settings/developers (ou org kaven-co)
- [ ] App Name: `Kaven CLI` (ou `Kaven Marketplace`)
- [ ] Homepage URL: `https://kaven.site`
- [ ] Authorization callback URL: `https://marketplace.kaven.site/auth/github/callback`
- [ ] `GITHUB_CLIENT_ID` copiado e salvo
- [ ] `GITHUB_CLIENT_SECRET` gerado e salvo no password manager
- [ ] Env vars adicionados ao Railway (D1.6)

## Technical Notes

**Passos:**
1. Acessar: https://github.com/settings/developers
2. "New OAuth App"
3. Preencher:
   - Application name: `Kaven CLI`
   - Homepage URL: `https://kaven.site`
   - Application description: `Kaven SaaS Boilerplate — Developer Authentication`
   - Authorization callback URL: `https://marketplace.kaven.site/auth/github/callback`
4. "Register application"
5. Copiar `Client ID`
6. "Generate a new client secret" → copiar

**Env vars para o marketplace:**
```bash
GITHUB_CLIENT_ID=Ov23li...
GITHUB_CLIENT_SECRET=xxx...
GITHUB_CALLBACK_URL=https://marketplace.kaven.site/auth/github/callback
```

**Nota:** Se Railway ainda não estiver deployado (D1.6), criar o app com URL placeholder e atualizar depois.
