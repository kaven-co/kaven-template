## Contexto

**Decisão de provider (histórico):**
- **Planejado originalmente:** AWS SES (já com domínio verificado e sandbox exit aprovado em Feb 26)
- **Motivo da troca:** Conta AWS nova sem histórico de billing — complexidade operacional elevada
- **Provider escolhido:** **Resend** (resend.com) — integração simples, free tier 100/dia suficiente para early stage

**Status atual:**
- ✅ Resend configurado e ativo
- ✅ Free tier: 100 emails/dia (suficiente para early adopters)
- ✅ Domínio `kaven.site` verificado (DKIM/SPF/DMARC herdados da config AWS SES anterior)

## User Story

**Como** DevOps do Kaven,
**Quero** emails transacionais funcionando via Resend,
**Para que** welcome, reset de senha e notificações cheguem aos usuários.

## Technical Notes (Resend — provider real)

**Dashboard:** https://resend.com/overview

**Env vars:**
```bash
RESEND_API_KEY=re_xxx
EMAIL_FROM=noreply@kaven.site
EMAIL_REPLY_TO=support@kaven.site
```

**Integração no framework:**
- Provider configurado em `apps/api/src/modules/platform/` (email module)
- Templates: welcome.html, verify-email.html, reset-password.html, invoice-paid.html
- Fallback: SMTP (se necessário)
