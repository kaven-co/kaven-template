## Technical Notes

**Verificar status via CLI:**
```bash
aws ses get-account --region us-east-1 \
  --query '{SendingEnabled:SendingEnabled,MaxSendRate:SendingQuota.MaxSendRate,Max24HourSend:SendingQuota.Max24HourSend}'

# Esperado após aprovação:
# {
#   "SendingEnabled": true,
#   "MaxSendRate": 14.0,
#   "Max24HourSend": 50000.0
# }
```

**Configurar SNS → Marketplace webhook:**
```bash
# Criar SNS Topic
aws sns create-topic --name kaven-ses-notifications --region us-east-1

# Criar subscription (webhook do marketplace)
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:ACCOUNT_ID:kaven-ses-notifications \
  --protocol https \
  --notification-endpoint https://marketplace.kaven.site/webhooks/ses \
  --region us-east-1

# No SES, configurar notification set para o domínio kaven.site:
# SES → Identities → kaven.site → Notifications → Edit
# Bounce: arn:aws:sns:us-east-1:ACCOUNT_ID:kaven-ses-notifications
# Complaint: arn:aws:sns:us-east-1:ACCOUNT_ID:kaven-ses-notifications
```

**Teste de envio:**
```bash
aws ses send-email \
  --from noreply@kaven.site \
  --to your-real-email@gmail.com \
  --subject "Kaven SES Production Test" \
  --text "Se você recebeu este email, SES produção está funcionando!" \
  --region us-east-1
```

**Env vars para o framework:**
```bash
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=xxx  # IAM user kaven-platform-api
AWS_SECRET_ACCESS_KEY=xxx
SES_FROM_EMAIL=noreply@kaven.site
SES_REPLY_TO=support@kaven.site
SNS_BOUNCE_TOPIC_ARN=arn:aws:sns:us-east-1:xxx:kaven-ses-notifications
```
