# D1.5 — Generate Production Secrets — JWT + Ed25519 Keypair

> **Epic:** 6 (Deploy & Launch) | **Sprint:** D1 | **Agent:** @devops (Deploy/Gage)
> **Priority:** P0 — Sem secrets, nenhuma API funciona em produção
> **Depends on:** Nenhum

---

## Contexto

O Kaven precisa de vários secrets de produção:
- JWT secrets (para marketplace e framework — diferentes para isolamento)
- Refresh token secret (para o framework)
- Ed25519 keypair (para assinatura digital de módulos — CLI verifica assinatura no download)

Os secrets **nunca** devem ser commitados em nenhum repo.

## User Story

**Como** DevOps do Kaven,
**Quero** gerar todos os secrets de produção necessários,
**Para que** os sistemas de autenticação e verificação de módulos funcionem com segurança em produção.

## Acceptance Criteria

- [ ] `JWT_SECRET` gerado para o marketplace (mínimo 32 bytes, base64)
- [ ] `JWT_SECRET` gerado para o framework (diferente do marketplace)
- [ ] `REFRESH_TOKEN_SECRET` gerado para o framework
- [ ] Ed25519 `SIGNING_PRIVATE_KEY` gerado (para assinar módulos no publish)
- [ ] Ed25519 `SIGNING_PUBLIC_KEY` gerado (distribuído no CLI para verificação)
- [ ] Todos os secrets salvos no password manager (1Password/Bitwarden)
- [ ] **NUNCA** commitados em nenhum repositório
- [ ] `SIGNING_PUBLIC_KEY` preparado para ser hardcoded no CLI (é público)

## Technical Notes

```bash
# JWT Secrets (executar no terminal — NUNCA commitar output)
echo "JWT_SECRET (marketplace):"
openssl rand -base64 32

echo "JWT_SECRET (framework):"
openssl rand -base64 32

echo "REFRESH_TOKEN_SECRET (framework):"
openssl rand -base64 32

# Ed25519 Keypair para assinatura de módulos
node -e "
const crypto = require('crypto');
const { publicKey, privateKey } = crypto.generateKeyPairSync('ed25519');
const privB64 = privateKey.export({type:'pkcs8',format:'pem'}).toString('base64');
const pubB64 = publicKey.export({type:'spki',format:'pem'}).toString('base64');
console.log('SIGNING_PRIVATE_KEY=' + privB64);
console.log('');
console.log('SIGNING_PUBLIC_KEY=' + pubB64);
"
```

**Distribuição dos secrets:**
```
marketplace/.env:
  JWT_SECRET=<gerado acima>
  SIGNING_PRIVATE_KEY=<Ed25519 private>
  SIGNING_PUBLIC_KEY=<Ed25519 public>

framework/.env:
  JWT_SECRET=<gerado acima — DIFERENTE do marketplace>
  REFRESH_TOKEN_SECRET=<gerado acima>

kaven-cli (hardcoded — é público):
  // src/lib/signature-verifier.ts
  const PUBLIC_KEY = "<Ed25519 public key>";
```

**IMPORTANTE:** A `SIGNING_PUBLIC_KEY` vai ser hardcoded no CLI (é pública por design) — precisa ser adicionada em `kaven-cli/src/lib/signature-verifier.ts` na story D1.10.
