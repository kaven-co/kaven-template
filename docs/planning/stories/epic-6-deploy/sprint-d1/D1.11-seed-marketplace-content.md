## Contexto

Para que o marketplace pareça "real" no launch, precisamos de mais módulos além do `payments`. Existem 4 módulos base que fazem sentido para o launch:
- `payments` — já existente (D1.10)
- `auth-social` — OAuth providers (Google, GitHub, Discord)
- `storage` — File upload + AWS S3 integration
- `analytics` — Usage analytics + telemetry

Estes módulos podem ser "previews" — registros no DB sem arquivo real no S3 (coming soon) — ou implementações reais extraídas do framework.

## User Story

**Como** dev do Kaven,
**Quero** adicionar módulos base ao marketplace,
**Para que** os usuários vejam um catálogo útil ao fazer `kaven marketplace browse`.

## Acceptance Criteria

- [ ] Módulo `payments` já publicado (D1.10) visível no marketplace ✅
- [ ] Módulo `auth-social` registrado no marketplace (mesmo que como "coming_soon")
- [ ] Módulo `storage` registrado no marketplace (mesmo que como "coming_soon")
- [ ] Módulo `analytics` registrado no marketplace (mesmo que como "coming_soon")

## Technical Notes

**Opção 1 — Seed Script (recomendado para coming_soon):**
```typescript
// scripts/seed-marketplace-modules.ts
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

const modules = [
  {
    id: 'auth-social',
    name: 'Social Auth',
    description: 'OAuth2 providers: Google, GitHub, Discord, LinkedIn',
    version: '1.0.0',
    tier: 'complete',
    category: 'authentication',
    status: 'coming_soon',
    author: 'kaven-co',
  },
  {
    id: 'storage',
    name: 'File Storage',
    description: 'Multi-provider file upload: AWS S3, Cloudflare R2, local',
    version: '1.0.0',
    tier: 'starter',
    category: 'storage',
    status: 'coming_soon',
    author: 'kaven-co',
  },
  {
    id: 'analytics',
    name: 'Usage Analytics',
    description: 'Tenant usage tracking, charts, export — privacy-first',
    version: '1.0.0',
    tier: 'pro',
    category: 'analytics',
    status: 'coming_soon',
    author: 'kaven-co',
  },
];

async function seed() {
  for (const mod of modules) {
    await prisma.module.upsert({
      where: { id: mod.id },
      create: mod,
      update: mod,
    });
  }
  console.log('Marketplace seeded with', modules.length, 'modules');
}

seed().finally(() => prisma.$disconnect());
```

**Executar em produção:**
```bash
DATABASE_URL=<production_marketplace_url> npx ts-node scripts/seed-marketplace-modules.ts
```

**Opção 2 — Via API direta:**
```bash
# Se a API tem endpoint de admin para criar módulos
curl -X POST https://marketplace.kaven.site/admin/modules \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"id":"auth-social","name":"Social Auth",...}'
```

**Verificar após seed:**
```bash
kaven marketplace browse
# Deve listar: payments (available), auth-social (coming soon),
#              storage (coming soon), analytics (coming soon)
```
