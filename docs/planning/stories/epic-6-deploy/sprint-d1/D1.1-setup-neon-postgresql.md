# D1.1 — Setup Neon PostgreSQL — 2 Production Databases

> **Epic:** 6 (Deploy & Launch) | **Sprint:** D1 | **Agent:** @devops (Deploy/Gage)
> **Priority:** P0 — BLOCKER (toda a infra depende do banco de dados)
> **Depends on:** Nenhum — pode começar agora

---

## Contexto

O Kaven precisa de 2 databases PostgreSQL em produção:
- `kaven_marketplace` — para a API de marketplace (kaven-co/kaven-marketplace)
- `kaven_framework` — para o framework (dogfooding via kaven-site)

Neon foi escolhido por ser serverless, ter connection pooling built-in, backup automático (7 dias), e free tier generoso ($0 → $25/mês Pro).

## User Story

**Como** DevOps do Kaven,
**Quero** criar 2 databases PostgreSQL em produção no Neon,
**Para que** o Marketplace API e o framework tenham backends de dados funcionais em produção.

## Acceptance Criteria

- [ ] Projeto Neon criado em neon.tech (região us-east-1 para alinhamento com AWS SES + S3)
- [ ] Database `kaven_marketplace` criada
- [ ] Database `kaven_framework` criada
- [ ] Connection strings testadas: `psql $DATABASE_URL` retorna `psql (16.x) connected`
- [ ] SSL mode = `require` configurado em ambas as connections
- [ ] Connection pooling ativo (Neon PgBouncer built-in)
- [ ] Backup automático 7 dias confirmado no painel
- [ ] Connection strings salvas em password manager (1Password/Bitwarden)
- [ ] `.env.production.example` atualizado com as variáveis

## Technical Notes

```bash
# Testar conexão marketplace
psql "postgresql://user:pass@xxx.neon.tech/kaven_marketplace?sslmode=require" -c "SELECT version();"

# Testar conexão framework
psql "postgresql://user:pass@xxx.neon.tech/kaven_framework?sslmode=require" -c "SELECT version();"
```

**Env vars para salvar:**
```bash
# .env (kaven-marketplace)
DATABASE_URL=postgresql://user:pass@xxx.neon.tech/kaven_marketplace?sslmode=require

# .env (kaven-framework)
DATABASE_URL=postgresql://user:pass@xxx.neon.tech/kaven_framework?sslmode=require
```

**Neon free tier specs:**
- Compute: 0.25 CU (suficiente para launch)
- Storage: 3 GB (suficiente)
- Connections: 100 pooled
- Backup: automático 7 dias

## Definition of Done

- [ ] Ambas as databases criadas e acessíveis via psql
- [ ] Connection strings testadas (SELECT 1 passa)
- [ ] Secrets salvos no password manager
- [ ] `.env.production.example` documentado com ambas as variáveis
