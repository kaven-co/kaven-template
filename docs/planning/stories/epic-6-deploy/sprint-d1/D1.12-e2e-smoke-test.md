## Contexto

Esta é a prova final. Se este smoke test passar 100%, a plataforma Kaven está pronta para o launch público em março 2026. O teste valida o fluxo completo end-to-end:

1. Instalar CLI → bootstrap projeto → autenticar → instalar módulo → rodar → signup → email

Nenhuma story anterior pode ter pendências antes de executar este teste.

**Launch readiness gate:** Todos os checks devem ser verde

## User Story

**Como** QA do Kaven,
**Quero** executar o smoke test E2E completo do sistema em produção,
**Para que** possamos confirmar que o Kaven está funcionando end-to-end antes do launch público.

## Acceptance Criteria

### Phase 1 — CLI Bootstrap
- [ ] `npm install -g kaven-cli@latest` instala sem erros
- [ ] `kaven doctor` mostra todos os checks verdes (Node version, Git, pnpm)
- [ ] `kaven init kaven-smoke-test` executa sem erros
- [ ] Diretório `kaven-smoke-test/` criado com estrutura completa do framework
- [ ] `.env` criado com valores substituídos (sem `{{PLACEHOLDERS}}` literais)

### Phase 2 — Authentication
- [ ] `kaven auth login` abre browser automaticamente
- [ ] GitHub OAuth flow completa sem erros
- [ ] JWT salvo localmente (token válido)
- [ ] `kaven auth whoami` mostra usuário GitHub correto

### Phase 3 — Marketplace
- [ ] `kaven marketplace browse` lista módulos (mínimo: payments)
- [ ] `kaven module install payments` executa sem erros
  - [ ] Download do S3 com sucesso
  - [ ] Verificação de assinatura Ed25519 passa (`✓ Signature verified`)
  - [ ] Extração + merge do schema Prisma
  - [ ] Markers `[KAVEN_MODULE:payments BEGIN/END]` adicionados nos arquivos

### Phase 4 — Application Runtime
- [ ] `pnpm install` sem erros (todas as dependências instaladas)
- [ ] `pnpm db:migrate` sem erros (schema aplicado no banco local)
- [ ] `pnpm dev` inicia sem erros (API + Admin + Tenant)
- [ ] `http://localhost:3001` (Tenant) carrega a landing page
- [ ] `http://localhost:3000` (Admin) carrega o painel de admin

### Phase 5 — User Flow
- [ ] Signup em `http://localhost:3001/signup` funciona
- [ ] Email de boas-vindas recebido via AWS SES produção (em < 2 minutos)
- [ ] Email de verificação tem link funcional
- [ ] Clicar no link → conta ativada → redirect para dashboard
- [ ] Login em `http://localhost:3000` (admin) mostra o tenant criado
- [ ] Plan enforcement: features corretas para o tier free/starter

### Phase 6 — Production API
- [ ] `curl https://marketplace.kaven.site/health` → `{"status":"ok",...}`
- [ ] `curl https://marketplace.kaven.site/modules` → lista com módulo payments
- [ ] `curl https://marketplace.kaven.site/docs` → Swagger UI acessível
- [ ] Zero erros 500 nos logs do Railway durante todo o teste

## Technical Notes

**Script de smoke test completo:**
```bash
#!/bin/bash
# kaven-e2e-smoke-test.sh
set -euo pipefail

echo "Kaven E2E Smoke Test — $(date)"
echo "================================="

# Phase 1: CLI Bootstrap
echo "Phase 1: CLI Bootstrap"
npm install -g kaven-cli@latest
kaven doctor
kaven init kaven-smoke-test
cd kaven-smoke-test

# Verificar estrutura criada
test -f ".env" || (echo "FAIL: .env nao criado" && exit 1)
test -f "apps/api/package.json" || (echo "FAIL: apps/api/package.json nao encontrado" && exit 1)
test -f "packages/database/prisma/schema.prisma" || (echo "FAIL: schema.prisma nao encontrado" && exit 1)
echo "PASS: Phase 1 — CLI Bootstrap OK"

# Phase 2: Auth
echo "Phase 2: Authentication"
kaven auth login  # Interativo — abre browser
kaven auth whoami
echo "PASS: Phase 2 — Auth OK"

# Phase 3: Marketplace
echo "Phase 3: Marketplace"
kaven marketplace browse
kaven module install payments
test -f "packages/database/prisma/payments.prisma" || (echo "FAIL: payments.prisma nao mergeado" && exit 1)
echo "PASS: Phase 3 — Marketplace OK"

# Phase 4: App Runtime
echo "Phase 4: Application Runtime"
pnpm install
# Configurar .env com DATABASE_URL real para teste
pnpm db:migrate
pnpm dev &
DEV_PID=$!
sleep 10  # aguardar inicialização

curl -f http://localhost:3001 > /dev/null || (echo "FAIL: Tenant app nao iniciou" && kill $DEV_PID && exit 1)
curl -f http://localhost:3000 > /dev/null || (echo "FAIL: Admin app nao iniciou" && kill $DEV_PID && exit 1)
echo "PASS: Phase 4 — App Runtime OK"

# Phase 5: Manual check (interativo)
echo ""
echo "Phase 5: Manual User Flow Check"
echo "1. Abra http://localhost:3001/signup"
echo "2. Complete o signup com email real"
echo "3. Verifique se email chegou (< 2min)"
echo "4. Clique no link de verificacao"
echo "5. Faca login em http://localhost:3000 (admin)"
echo ""
read -p "Phase 5 passou? (y/n) " phase5_ok
[[ "$phase5_ok" == "y" ]] || (echo "FAIL: Phase 5 falhou" && kill $DEV_PID && exit 1)

# Phase 6: Production API
echo "Phase 6: Production API"
curl -f https://marketplace.kaven.site/health
curl -f https://marketplace.kaven.site/modules
echo "PASS: Phase 6 — Production API OK"

kill $DEV_PID 2>/dev/null
echo ""
echo "ALL PHASES PASSED — LAUNCH READY!"
echo "Deploy target: https://kaven.site"
```
