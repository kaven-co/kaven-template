## Contexto

O Kaven usa Paddle como Merchant of Record (MoR) — Paddle gerencia impostos, chargebacks, e compliance fiscal globalmente. Os 3 tiers são:
- **Starter:** $149 (launch price: $99) — 1 projeto, 10 tenants
- **Complete:** $399 (launch price: $279) — 1 projeto, unlimited tenants
- **Pro:** $799 (launch price: $549) — 5 projetos, marketplace access

Páginas legais já estão live (pré-requisito Paddle): kaven.site/terms + kaven.site/privacy + kaven.site/refund ✅

## User Story

**Como** DevOps do Kaven,
**Quero** configurar os 3 produtos e preços no Paddle produção e registrar o webhook,
**Para que** o sistema de pagamentos esteja funcional no launch.

## Acceptance Criteria

- [ ] Conta Paddle verificada e em modo produção (não sandbox)
- [ ] Produto "Kaven Starter" criado com preço $99 (launch) / $149 (regular)
- [ ] Produto "Kaven Complete" criado com preço $279 (launch) / $399 (regular)
- [ ] Produto "Kaven Pro" criado com preço $549 (launch) / $799 (regular)
- [ ] Webhook configurado: `https://marketplace.kaven.site/webhooks/paddle`
- [ ] Eventos selecionados no webhook: `subscription.created`, `subscription.updated`, `subscription.canceled`, `transaction.completed`, `transaction.payment_failed`
- [ ] `PADDLE_API_KEY` copiado e salvo
- [ ] `PADDLE_WEBHOOK_SECRET` copiado e salvo
- [ ] `PADDLE_PRICE_STARTER`, `PADDLE_PRICE_COMPLETE`, `PADDLE_PRICE_PRO` copiados (IDs dos preços)
- [ ] Checkout testado em sandbox antes de ativar produção

## Technical Notes

**Dashboard:** https://vendors.paddle.com

**Estrutura de produtos:**
```
Kaven Starter
  └── Price: $99/one-time (launch) → depois $149

Kaven Complete
  └── Price: $279/one-time (launch) → depois $399

Kaven Pro
  └── Price: $549/one-time (launch) → depois $799
```

**Env vars para o marketplace:**
```bash
PADDLE_ENVIRONMENT=production
PADDLE_API_KEY=xxx
PADDLE_WEBHOOK_SECRET=xxx
PADDLE_PRICE_STARTER=pri_xxx   # Price ID do produto Starter
PADDLE_PRICE_COMPLETE=pri_xxx  # Price ID do produto Complete
PADDLE_PRICE_PRO=pri_xxx       # Price ID do produto Pro
```

**Webhook events a selecionar:**
- `subscription.created` — nova assinatura
- `subscription.updated` — upgrade/downgrade
- `subscription.canceled` — cancelamento
- `transaction.completed` — pagamento confirmado
- `transaction.payment_failed` — falha no pagamento

**IMPORTANTE:** Verificar que `PADDLE_ENVIRONMENT=production` (não sandbox) antes do launch.
