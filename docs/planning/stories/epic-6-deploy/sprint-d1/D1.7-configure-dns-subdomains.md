## Contexto

O Kaven usa 3 subdomains no Cloudflare (DNS do kaven.site):
- `marketplace.kaven.site` → Railway (Fastify API)
- `admin.kaven.site` → Vercel (Next.js admin app)
- `tenant.kaven.site` → Vercel (Next.js tenant app)

O domínio raiz `kaven.site` já aponta para Vercel (kaven-site landing page).

DNS records SES (DKIM/SPF/DMARC) já foram publicados e estão propagados ✅

## User Story

**Como** DevOps do Kaven,
**Quero** configurar os CNAMEs corretos no Cloudflare,
**Para que** cada subdomain aponte para o serviço correto em produção.

## Acceptance Criteria

- [ ] CNAME `marketplace` → Railway domain (com Cloudflare Proxy ✅ ativado)
- [ ] CNAME `admin` → `cname.vercel-dns.com` (DNS Only, sem proxy — Vercel requer)
- [ ] CNAME `tenant` → `cname.vercel-dns.com` (DNS Only, sem proxy — Vercel requer)
- [ ] `https://marketplace.kaven.site/health` responde (pode levar ~30min para propagar)
- [ ] `https://admin.kaven.site` carrega o admin Next.js app
- [ ] `https://tenant.kaven.site` carrega o tenant Next.js app
- [ ] SSL certificates emitidos automaticamente pelo Cloudflare/Vercel
- [ ] No Vercel: custom domains adicionados em cada projeto (admin + tenant)

## Technical Notes

**Cloudflare DNS Records a configurar:**

| Tipo | Nome | Valor | Proxy |
|------|------|-------|-------|
| CNAME | `marketplace` | `xxx.railway.app` | ✅ Proxied |
| CNAME | `admin` | `cname.vercel-dns.com` | ❌ DNS Only |
| CNAME | `tenant` | `cname.vercel-dns.com` | ❌ DNS Only |

**Por que Proxy OFF para Vercel?**
Vercel valida domain ownership via DNS — o Cloudflare Proxy interfere na verificação SSL do Vercel. Sempre usar "DNS Only" para subdomains apontando para Vercel.

**Vercel — adicionar custom domains:**
```
# Projeto admin no Vercel
vercel domains add admin.kaven.site

# Projeto tenant no Vercel
vercel domains add tenant.kaven.site
```

**Verificar propagação DNS:**
```bash
# Testar após configurar
dig marketplace.kaven.site CNAME
dig admin.kaven.site CNAME
dig tenant.kaven.site CNAME

# Ou via online
# https://dnschecker.org/
```

**Records SES já configurados (não mexer):**
- TXT `_dmarc` — política DMARC
- TXT `kaven.site` — SPF record
- CNAME `mail._domainkey` — DKIM record

## Definition of Done

- [ ] 3 CNAME records configurados no Cloudflare
- [ ] Custom domains adicionados no Vercel para admin + tenant
- [ ] DNS propagado (dig retorna valores corretos)
- [ ] Todos os 3 subdomains carregam via HTTPS

## Time Estimate

- **Total: 2h** (pode levar mais se DNS demorar propagar)
