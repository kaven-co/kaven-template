# D1.10 — Publish Payments Module — marketplace.kaven.site

> **Epic:** 6 (Deploy & Launch) | **Sprint:** D1 | **Agent:** @dev (Bolt)
> **Priority:** P1 — Primeiro módulo no marketplace, prova que o sistema funciona
> **Depends on:** D1.6 (Marketplace API deployed), D1.5 (Ed25519 keys para signing)

---

## Contexto

Para provar que o sistema de marketplace funciona end-to-end, precisamos publicar o primeiro módulo real. O módulo `payments` é perfeito para isso porque:
- Já existe no framework (apps/api/src/modules/billing/ + checkout/)
- É um módulo valioso que justifica o marketplace
- Tem schema Prisma, rotas, e serviços definidos
- Exercita toda a pipeline: extract → module.json → kaven module publish → S3 → Ed25519 sign

## User Story

**Como** dev do Kaven,
**Quero** extrair o módulo payments do framework e publicá-lo no marketplace,
**Para que** usuários possam instalar via `kaven module install payments`.

## Acceptance Criteria

- [ ] Diretório `modules/payments/` criado na raiz do kaven-framework
- [ ] `module.json` criado com metadata completo (ver Technical Notes)
- [ ] Código do módulo extraído: prisma schema extension, routes, services, tests
- [ ] `kaven module publish --dry-run` passa sem erros
- [ ] `kaven module publish` faz upload para S3 e assina com Ed25519
- [ ] Módulo aparece em `kaven marketplace browse` como "payments"
- [ ] `curl https://marketplace.kaven.site/modules` retorna o módulo payments
- [ ] `kaven module install payments` em projeto de teste funciona (download + verify + merge)
- [ ] Assinatura Ed25519 verificada com sucesso pelo CLI

## Technical Notes

**Estrutura do módulo payments:**
```
kaven-framework/modules/payments/
├── module.json                    ← Metadata
├── prisma/
│   └── payments.prisma            ← Schema extension (Stripe, Paddle tables)
├── src/
│   ├── routes/
│   │   ├── checkout.routes.ts
│   │   └── billing.routes.ts
│   ├── services/
│   │   ├── stripe.service.ts
│   │   └── paddle.service.ts
│   └── index.ts                   ← Module entry point
├── tests/
│   └── payments.spec.ts
└── README.md                      ← Como instalar + configurar
```

**module.json:**
```json
{
  "id": "payments",
  "name": "Kaven Payments",
  "version": "1.0.0",
  "description": "Multi-provider payment integration — Stripe + Paddle + PIX",
  "tier": "pro",
  "category": "payments",
  "author": "kaven-co",
  "license": "commercial",
  "kaven": ">=1.0.0-rc1",
  "dependencies": [],
  "scripts": {
    "postinstall": "npx prisma db push"
  },
  "env": [
    "STRIPE_SECRET_KEY",
    "STRIPE_WEBHOOK_SECRET",
    "PADDLE_API_KEY",
    "PADDLE_WEBHOOK_SECRET"
  ],
  "files": [
    "prisma/",
    "src/",
    "tests/",
    "README.md"
  ]
}
```

**Publicar:**
```bash
cd kaven-framework/modules/payments

# Dry run — validar sem publicar
kaven module publish --dry-run
# Deve mostrar: ✓ module.json válido, ✓ arquivos encontrados, ✓ signing key OK

# Publicar
kaven module publish
# Deve mostrar: ✓ uploading... ✓ signed with Ed25519 ✓ published to marketplace
```

**Verificar no marketplace:**
```bash
# Listar módulos
curl https://marketplace.kaven.site/modules | jq '.[] | {id, name, version}'

# Instalar em projeto de teste
cd /tmp/test-project
kaven module install payments
```

**Checar compatibilidade de placeholders no code:**
O módulo publicado tem `[KAVEN_MODULE:payments BEGIN]` e `[KAVEN_MODULE:payments END]` markers para idempotency do CLI.
