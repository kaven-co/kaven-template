# M2.7 — Rate Limiting & Throttling

> **Epic:** 1 (Marketplace) | **Sprint:** M2 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** (none)

---

## Contexto

A public-facing API without rate limiting is an invitation for abuse. Brute-force attacks on auth endpoints, scraping of module listings, and download abuse can degrade the service for legitimate users or run up infrastructure costs.

This story implements granular rate limiting with different thresholds per endpoint category. The device code polling endpoint has a particularly strict limit (1 request per 5 seconds) to comply with RFC 8628, which specifies an `interval` field that clients must respect.

Redis is used as the rate limit store for distributed correctness (multiple API instances share the same counters). The webhook endpoint is excluded from rate limiting because Paddle needs reliable delivery without artificial barriers.

## User Story

**Como** operador do marketplace, **Quero** que o consumo de API seja limitado por endpoint, **Para que** abuso não degrade o serviço para usuários legítimos.

---

## Acceptance Criteria

- [ ] `@fastify/rate-limit` plugin registered globally with Redis store
- [ ] Auth endpoints (`/auth/*`): 10 requests/minute per IP
- [ ] Device code polling (`/auth/device/token`): 1 request/5 seconds per device_code
- [ ] Module listing endpoints (`/modules`, `/modules/:slug`): 60 requests/minute per user (or IP if unauthenticated)
- [ ] Download endpoints (`/download-tokens`, `/artifacts/*`): 50 requests/hour per user
- [ ] Admin endpoints (`/admin/*`): 100 requests/minute per user
- [ ] Webhook endpoints (`/webhooks/*`): no rate limit
- [ ] Rate limit exceeded returns HTTP 429 Too Many Requests
- [ ] 429 response includes `Retry-After` header (seconds until limit resets)
- [ ] 429 response body: `{ error: "TOO_MANY_REQUESTS", retryAfter: number }`
- [ ] Rate limit headers included on every response: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
- [ ] Redis connection failure falls back to in-memory rate limiting (degraded but not broken)
- [ ] Device code polling 429 includes `error: "slow_down"` per RFC 8628 spec
- [ ] Rate limits are configurable via environment variables

## Technical Notes

### Plugin Registration

```typescript
// src/plugins/rate-limit.ts

import rateLimit from '@fastify/rate-limit';
import Redis from 'ioredis';

export async function registerRateLimit(fastify: FastifyInstance): Promise<void> {
  const redisClient = new Redis({
    host: process.env.REDIS_HOST || 'localhost',
    port: parseInt(process.env.REDIS_PORT || '6379'),
    password: process.env.REDIS_PASSWORD,
    keyPrefix: 'kaven:ratelimit:',
    lazyConnect: true,
    retryStrategy: (times) => {
      if (times > 3) return null; // Stop retrying after 3 attempts
      return Math.min(times * 200, 2000);
    },
  });

  let redisAvailable = false;
  try {
    await redisClient.connect();
    redisAvailable = true;
  } catch (err) {
    console.warn('Redis unavailable for rate limiting, falling back to in-memory store');
  }

  await fastify.register(rateLimit, {
    global: true,
    max: 100,                    // Default: 100 requests/minute
    timeWindow: '1 minute',
    ...(redisAvailable && {
      redis: redisClient,
    }),
    keyGenerator: (request) => {
      // Use userId if authenticated, otherwise IP
      return request.user?.id || request.ip;
    },
    errorResponseBuilder: (request, context) => {
      return {
        error: 'TOO_MANY_REQUESTS',
        message: `Rate limit exceeded. Try again in ${context.after}`,
        retryAfter: Math.ceil(context.ttl / 1000),
      };
    },
    addHeadersOnExceeding: {
      'x-ratelimit-limit': true,
      'x-ratelimit-remaining': true,
      'x-ratelimit-reset': true,
    },
    addHeaders: {
      'x-ratelimit-limit': true,
      'x-ratelimit-remaining': true,
      'x-ratelimit-reset': true,
      'retry-after': true,
    },
  });
}
```

### Per-Route Rate Limit Overrides

```typescript
// src/routes/auth.route.ts

// Auth endpoints: 10/minute per IP
fastify.post('/auth/login', {
  config: {
    rateLimit: {
      max: 10,
      timeWindow: '1 minute',
      keyGenerator: (request) => request.ip, // Always by IP for auth
    },
  },
  handler: loginHandler,
});

// Device code polling: 1 request/5 seconds per device_code (RFC 8628)
fastify.post('/auth/device/token', {
  config: {
    rateLimit: {
      max: 1,
      timeWindow: '5 seconds',
      keyGenerator: (request) => {
        const { device_code } = request.body as any;
        return `device:${device_code}`;
      },
      errorResponseBuilder: (request, context) => ({
        error: 'slow_down',
        message: 'Polling too fast. Increase interval per RFC 8628.',
        interval: 5,
        retryAfter: Math.ceil(context.ttl / 1000),
      }),
    },
  },
  handler: deviceTokenHandler,
});
```

### Rate Limit Configuration Loader

```typescript
// src/config/rate-limit.config.ts

export interface RateLimitConfig {
  auth: { max: number; timeWindow: string };
  devicePoll: { max: number; timeWindow: string };
  modules: { max: number; timeWindow: string };
  downloads: { max: number; timeWindow: string };
  admin: { max: number; timeWindow: string };
}

export function loadRateLimitConfig(): RateLimitConfig {
  return {
    auth: {
      max: parseInt(process.env.RATE_LIMIT_AUTH || '10'),
      timeWindow: '1 minute',
    },
    devicePoll: {
      max: 1,
      timeWindow: `${process.env.RATE_LIMIT_DEVICE_POLL || '5'} seconds`,
    },
    modules: {
      max: parseInt(process.env.RATE_LIMIT_MODULES || '60'),
      timeWindow: '1 minute',
    },
    downloads: {
      max: parseInt(process.env.RATE_LIMIT_DOWNLOADS || '50'),
      timeWindow: '1 hour',
    },
    admin: {
      max: parseInt(process.env.RATE_LIMIT_ADMIN || '100'),
      timeWindow: '1 minute',
    },
  };
}
```
