# M3.4 — OpenAPI Documentation

> **Epic:** 1 (Marketplace) | **Sprint:** M3 | **Agent:** @dev
> **Priority:** P2
> **Depends on:** M1.6

---

## Contexto

O kaven-marketplace ja usa `@fastify/swagger` e `@fastify/swagger-ui` como dependencias, e o type provider `fastify-type-provider-zod` para validacao de schemas. Porem, a documentacao OpenAPI nao esta configurada — os endpoints nao tem tags, descricoes, exemplos de autenticacao nem grouping logico. Uma documentacao OpenAPI bem feita e essencial para: (1) desenvolvedores integrando via kaven-cli entenderem a API, (2) parceiros e third-party developers construirem integracoes, (3) QA validar contratos de API automaticamente, (4) gerar SDKs client no futuro. A spec deve ser auto-gerada a partir dos Zod schemas ja existentes, sem duplicacao manual.

## User Story

**Como** desenvolvedor integrando com o marketplace (via CLI ou diretamente),
**Quero** acessar documentacao interativa e completa da API em `/docs`,
**Para que** eu entenda todos os endpoints, seus parametros, respostas e requisitos de autenticacao sem ler codigo-fonte.

## Technical Notes

### Swagger Configuration

```typescript
// src/plugins/swagger.ts
import fastifySwagger from '@fastify/swagger';
import fastifySwaggerUi from '@fastify/swagger-ui';
import { version } from '../../package.json';

export async function registerSwagger(fastify: FastifyInstance) {
  await fastify.register(fastifySwagger, {
    openapi: {
      openapi: '3.1.0',
      info: {
        title: 'Kaven Marketplace API',
        description: 'API for the Kaven module marketplace — module discovery, licensing, downloads, and analytics.',
        version,
        contact: {
          name: 'Kaven Support',
          url: 'https://kaven.dev/support',
          email: 'support@kaven.dev',
        },
        license: {
          name: 'Proprietary',
          url: 'https://kaven.dev/license',
        },
      },
      servers: [
        { url: 'http://localhost:3001', description: 'Development' },
        { url: 'https://api.marketplace.kaven.dev', description: 'Production' },
      ],
      components: {
        securitySchemes: {
          bearerAuth: {
            type: 'http',
            scheme: 'bearer',
            bearerFormat: 'JWT',
            description: 'JWT token obtained via device code flow or login endpoint',
          },
        },
      },
      tags: [
        { name: 'Auth', description: 'Authentication — device code flow, login, token refresh' },
        { name: 'Modules', description: 'Module CRUD — create, read, update, delete marketplace modules' },
        { name: 'Releases', description: 'Release management — publish new versions, list releases' },
        { name: 'Licenses', description: 'License management — create, validate, revoke licenses' },
        { name: 'Downloads', description: 'Module downloads — generate signed URLs, track downloads' },
        { name: 'Search', description: 'Module search and discovery — full-text search, category filters' },
        { name: 'Installations', description: 'Installation tracking — record installs/uninstalls' },
        { name: 'Analytics', description: 'Marketplace analytics — install trends, popular modules, churn' },
        { name: 'Admin', description: 'Admin endpoints — audit logs, user management' },
        { name: 'Webhooks', description: 'Paddle webhook processing — payment events' },
        { name: 'Health', description: 'Health and readiness probes' },
      ],
    },
  });

  if (process.env.DOCS_ENABLED !== 'false') {
    await fastify.register(fastifySwaggerUi, {
      routePrefix: '/docs',
      uiConfig: {
        docExpansion: 'list',
        deepLinking: true,
        filter: true,
        persistAuthorization: true,
      },
    });
  }
}
```

### Tagging Routes

```typescript
// In route registration, add schema with tags:
fastify.get('/modules', {
  schema: {
    tags: ['Modules'],
    summary: 'List all published modules',
    description: 'Returns a paginated list of all published modules. Supports filtering by category and tier.',
    querystring: ModuleListQuerySchema,
    response: {
      200: ModuleListResponseSchema,
    },
    security: [], // Public endpoint
  },
}, handler);

fastify.post('/modules', {
  schema: {
    tags: ['Modules'],
    summary: 'Create a new module',
    description: 'Creates a new module in the marketplace. Requires PUBLISHER role.',
    body: CreateModuleSchema,
    response: {
      201: ModuleResponseSchema,
    },
    security: [{ bearerAuth: [] }],
  },
}, handler);
```

### Adding Descriptions to Zod Schemas

```typescript
// Enhance existing schemas with .describe()
const CreateModuleSchema = z.object({
  name: z.string().min(2).max(50).describe('Module name — must be unique, lowercase, alphanumeric + hyphens'),
  slug: z.string().regex(/^[a-z0-9-]+$/).describe('URL-friendly slug — auto-generated from name if omitted').optional(),
  description: z.string().min(10).max(500).describe('Short description shown in search results'),
  category: z.enum(['auth', 'billing', ...]).describe('Module category for filtering and facets'),
  tier: z.enum(['STARTER', 'COMPLETE', 'PRO']).describe('Minimum plan tier required to install this module'),
  repositoryUrl: z.string().url().describe('Git repository URL (private OK — used for source verification)').optional(),
});
```

### Spec Export

The `@fastify/swagger` plugin automatically exposes:
- `GET /docs/json` — OpenAPI spec as JSON
- `GET /docs/yaml` — OpenAPI spec as YAML

These are registered by default when swagger-ui is enabled. If docs are disabled in production, manually register the JSON/YAML routes if needed for CI validation.

### CI Validation

```bash
# In CI pipeline — validate spec is valid OpenAPI 3.1
npx @redocly/cli lint http://localhost:3001/docs/json
# or
npx swagger-cli validate http://localhost:3001/docs/json
```
