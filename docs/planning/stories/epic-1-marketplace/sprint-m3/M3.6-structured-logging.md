> **Priority:** P1

---

## Contexto

Fastify ja usa Pino como logger nativo, mas a configuracao atual e basica — sem correlation IDs, sem redacao de dados sensiveis, sem formatacao adequada para diferentes ambientes. Em producao, logs estruturados (JSON) sao essenciais para log aggregation (CloudWatch, Datadog, ELK). Em desenvolvimento, logs pretty-printed aceleram debugging. Correlation IDs permitem rastrear uma request completa atraves de multiplos servicos e logs. Redacao de dados sensiveis evita vazamento de tokens e senhas nos logs. Este e o fundamento de observabilidade — sem logging estruturado, debugging em producao e um pesadelo.

## User Story

**Como** engenheiro de plataforma operando o marketplace em producao,
**Quero** logs estruturados com correlation IDs e redacao de dados sensiveis,
**Para que** eu possa debugar problemas rapidamente, rastrear requests end-to-end e garantir que dados sensiveis nunca aparecem nos logs.

## Acceptance Criteria

### Correlation IDs
- [ ] Toda request recebe um `correlationId` unico
- [ ] Se header `X-Request-ID` presente na request, usar esse valor como `correlationId`
- [ ] Se ausente, gerar UUID v4 automaticamente
- [ ] `correlationId` adicionado a todos os logs gerados durante a request
- [ ] `correlationId` retornado no response header `X-Request-ID`
- [ ] `correlationId` disponivel via `request.correlationId` para uso em handlers

### Log Format
- [ ] Em producao (`NODE_ENV=production`): JSON puro, uma linha por log entry (Pino default)
- [ ] Em desenvolvimento (`NODE_ENV=development`): pretty-print com cores via `pino-pretty`
- [ ] Em teste (`NODE_ENV=test`): logs silenciados (`level: 'silent'`) ou `level: 'error'`

### Log Levels
- [ ] `error`: sempre logado — erros de aplicacao, exceptions nao tratadas, falhas de dependencia
- [ ] `warn`: sempre logado — deprecation warnings, rate limit approaching, retry attempts
- [ ] `info`: producao + dev — requests/responses, business events, startup/shutdown
- [ ] `debug`: somente dev — query details, payload inspection, step-by-step flow
- [ ] Log level configuravel via `LOG_LEVEL` env var (default: `info` em prod, `debug` em dev)

### Sensitive Data Redaction
- [ ] JWT tokens mascarados em logs: `Bearer eyJ...` → `Bearer [REDACTED]`
- [ ] Passwords nunca logados (body com `password` field tem valor substituido por `[REDACTED]`)
- [ ] License keys mascarados: `KAVEN-XXXX-XXXX-XXXX` → `KAVEN-****-****-XXXX` (ultimos 4 chars)
- [ ] Headers `Authorization` e `Cookie` redactados automaticamente
- [ ] Implementado via Pino `redact` paths configuration

### Request/Response Logging
- [ ] Cada request loga: `method`, `url`, `correlationId`, `userId` (se autenticado), `ip`
- [ ] Cada response loga: `method`, `url`, `statusCode`, `responseTime` (ms), `correlationId`
- [ ] Erros (4xx/5xx) logam body de resposta e stack trace (se 5xx)
- [ ] Request body NAO logado por default (pode conter dados sensiveis) — somente em `debug` level
- [ ] Query params logados (redactando tokens se presentes)

### Error Logging
- [ ] Erros 5xx logam: stack trace completo, request context (method, url, params, user), `correlationId`
- [ ] Erros 4xx logam: mensagem de erro, request context (sem stack trace)
- [ ] Unhandled rejections e uncaught exceptions capturados e logados antes de crash

### Output
- [ ] Logs enviados para stdout (container-friendly, para log aggregation)
- [ ] Nenhuma escrita em arquivo — log rotation e responsabilidade da infra (Docker, CloudWatch, etc.)

## Technical Notes

### Pino Configuration

```typescript
// src/plugins/logger.ts
import pino from 'pino';

export function createLogger() {
  const isDev = process.env.NODE_ENV === 'development';
  const isTest = process.env.NODE_ENV === 'test';

  return pino({
    level: process.env.LOG_LEVEL ?? (isDev ? 'debug' : isTest ? 'silent' : 'info'),
    redact: {
      paths: [
        'req.headers.authorization',
        'req.headers.cookie',
        'req.body.password',
        'req.body.currentPassword',
        'req.body.newPassword',
        'req.body.licenseKey',
      ],
      censor: '[REDACTED]',
    },
    serializers: {
      req(request) {
        return {
          method: request.method,
          url: request.url,
          correlationId: request.correlationId,
          userId: request.user?.id,
          ip: request.ip,
        };
      },
      res(reply) {
        return {
          statusCode: reply.statusCode,
        };
      },
    },
    ...(isDev && {
      transport: {
        target: 'pino-pretty',
        options: {
          colorize: true,
          translateTime: 'HH:MM:ss.l',
          ignore: 'pid,hostname',
          singleLine: false,
        },
      },
    }),
  });
}
```

### Correlation ID Plugin

```typescript
// src/plugins/correlation-id.ts
import fp from 'fastify-plugin';
import { randomUUID } from 'crypto';

export const correlationIdPlugin = fp(async (fastify) => {
  fastify.decorateRequest('correlationId', '');

  fastify.addHook('onRequest', async (request) => {
    request.correlationId = (request.headers['x-request-id'] as string) ?? randomUUID();
    request.log = request.log.child({ correlationId: request.correlationId });
  });

  fastify.addHook('onSend', async (request, reply) => {
    reply.header('X-Request-ID', request.correlationId);
  });
});
```

### Fastify Integration

```typescript
// In src/app.ts
const app = Fastify({
  logger: createLogger(),
  requestIdHeader: 'x-request-id',
  genReqId: () => randomUUID(),
});

await app.register(correlationIdPlugin);
```

### Request/Response Logging Hook

```typescript
// src/plugins/request-logger.ts
import fp from 'fastify-plugin';

export const requestLoggerPlugin = fp(async (fastify) => {
  fastify.addHook('onRequest', async (request) => {
    request.log.info({ req: request }, 'incoming request');
  });

  fastify.addHook('onResponse', async (request, reply) => {
    request.log.info(
      {
        res: reply,
        responseTime: reply.elapsedTime,
        correlationId: request.correlationId,
      },
      'request completed'
    );
  });

  fastify.addHook('onError', async (request, reply, error) => {
    const level = reply.statusCode >= 500 ? 'error' : 'warn';
    request.log[level](
      {
        err: error,
        req: request,
        correlationId: request.correlationId,
      },
      `request error: ${error.message}`
    );
  });
});
```

### License Key Redaction (Custom Serializer)

```typescript
// For license keys in metadata/response bodies, add custom redaction:
function redactLicenseKey(key: string): string {
  if (!key || key.length < 8) return '[REDACTED]';
  // KAVEN-XXXX-XXXX-XXXX → KAVEN-****-****-XXXX
  const parts = key.split('-');
  if (parts.length < 4) return '[REDACTED]';
  return parts.map((p, i) => (i === 0 || i === parts.length - 1) ? p : '****').join('-');
}
```
