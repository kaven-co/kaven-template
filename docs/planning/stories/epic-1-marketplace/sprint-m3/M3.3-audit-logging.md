# M3.3 — Audit Logging

> **Epic:** 1 (Marketplace) | **Sprint:** M3 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** —

---

## Contexto

Um marketplace que processa licencas, downloads e pagamentos precisa de um trail de auditoria robusto. Audit logging e essencial para: (1) compliance com GDPR/LGPD — saber quem fez o que e quando, (2) debugging de incidentes — rastrear a sequencia de eventos que levou a um problema, (3) seguranca — detectar acessos suspeitos ou acoes maliciosas, (4) suporte ao cliente — entender o que aconteceu na conta de um usuario. O kaven-framework ja tem audit logging para o SaaS base; o marketplace precisa do seu proprio, focado em acoes de marketplace (licencas, modulos, releases, downloads, webhooks).

## User Story

**Como** administrador do marketplace,
**Quero** que todas as acoes sensíveis sejam automaticamente registradas em um log de auditoria pesquisavel,
**Para que** eu possa investigar incidentes, garantir compliance e fornecer suporte informado aos usuarios.

## Acceptance Criteria

- [ ] Novo modelo `AuditLog` criado com campos: `id`, `userId` (nullable — webhooks nao tem user), `action` (string enum), `resource` (string), `resourceId` (string), `metadata` (Json), `ipAddress` (string), `userAgent` (string), `createdAt`
- [ ] Actions suportadas: `license.create`, `license.revoke`, `license.validate`, `module.create`, `module.update`, `module.delete`, `release.publish`, `download.complete`, `user.login`, `user.logout`, `webhook.received`, `webhook.processed`
- [ ] Middleware Fastify `auditLog` que intercepta endpoints sensíveis e registra automaticamente
- [ ] Middleware extrai `userId` do JWT, `ipAddress` do `request.ip`, `userAgent` do header `User-Agent`
- [ ] `metadata` captura informacoes contextuais relevantes (ex: `{ licenseId, moduleSlug, reason }`)
- [ ] Endpoint `GET /admin/audit-logs` retorna logs paginados com filtros: `action`, `userId`, `resource`, `resourceId`, `dateFrom`, `dateTo`
- [ ] Endpoint protegido por role `ADMIN`
- [ ] Retencao de 90 dias: cron job ou scheduled task limpa registros mais antigos
- [ ] GDPR-ready: funcao `anonymizeUserAuditLogs(userId)` substitui `userId` por hash e limpa `ipAddress`/`userAgent`
- [ ] Audit logging nao bloqueia a request principal (fire-and-forget com error handling)
- [ ] Logs sao imutaveis — nenhum endpoint de update ou delete exposto

## Technical Notes

### Prisma Model

```prisma
model AuditLog {
  id         String   @id @default(cuid())
  userId     String?
  action     String   /// @index
  resource   String   /// e.g., "license", "module", "release"
  resourceId String
  metadata   Json     @default("{}")
  ipAddress  String   @default("")
  userAgent  String   @default("")
  createdAt  DateTime @default(now())

  @@index([action, createdAt])
  @@index([userId, createdAt])
  @@index([resource, resourceId])
  @@index([createdAt])
}
```

### Middleware Pattern

```typescript
// src/plugins/audit-log.ts
import fp from 'fastify-plugin';

export const auditLogPlugin = fp(async (fastify) => {
  fastify.decorateRequest('auditAction', null);
  fastify.decorateRequest('auditResource', null);
  fastify.decorateRequest('auditResourceId', null);
  fastify.decorateRequest('auditMetadata', {});

  fastify.addHook('onResponse', async (request, reply) => {
    if (!request.auditAction) return;
    if (reply.statusCode >= 400) return; // Don't log failed requests

    // Fire-and-forget — don't await
    fastify.prisma.auditLog.create({
      data: {
        userId: request.user?.id ?? null,
        action: request.auditAction,
        resource: request.auditResource,
        resourceId: request.auditResourceId,
        metadata: request.auditMetadata,
        ipAddress: request.ip,
        userAgent: request.headers['user-agent'] ?? '',
      },
    }).catch((err) => {
      request.log.error({ err }, 'Failed to create audit log');
    });
  });
});
```

### Route-Level Usage

```typescript
// In a route handler:
fastify.post('/licenses', async (request, reply) => {
  const license = await licenseService.create(request.body);

  // Tag request for audit logging
  request.auditAction = 'license.create';
  request.auditResource = 'license';
  request.auditResourceId = license.id;
  request.auditMetadata = { moduleSlug: request.body.moduleSlug, tier: request.body.tier };

  return reply.status(201).send(license);
});
```

### Retention Cleanup

```typescript
// src/jobs/audit-log-cleanup.ts
// Run daily via cron (node-cron) or external scheduler
export async function cleanupOldAuditLogs(prisma: PrismaClient) {
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - 90);

  const { count } = await prisma.auditLog.deleteMany({
    where: { createdAt: { lt: cutoff } },
  });

  console.log(`Cleaned up ${count} audit logs older than 90 days`);
  return count;
}
```

### GDPR Anonymization

```typescript
export async function anonymizeUserAuditLogs(prisma: PrismaClient, userId: string) {
  const hashedId = crypto.createHash('sha256').update(userId).digest('hex').slice(0, 16);

  await prisma.auditLog.updateMany({
    where: { userId },
    data: {
      userId: `anon_${hashedId}`,
      ipAddress: '0.0.0.0',
      userAgent: '[redacted]',
      metadata: {}, // Clear potentially identifying metadata
    },
  });
}
```

### Edge Cases
- Webhook endpoints have no authenticated user — `userId` is null
- Very high traffic: audit log inserts are fire-and-forget to not slow down requests
- Metadata should not contain sensitive data (no passwords, tokens, full credit card numbers)
- Concurrent audit log writes should not cause deadlocks (no transactions needed, inserts only)
