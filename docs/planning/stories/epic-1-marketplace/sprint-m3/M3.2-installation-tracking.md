# M3.2 — Installation Tracking

> **Epic:** 1 (Marketplace) | **Sprint:** M3 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** M2.2

---

## Contexto

Para entender o sucesso do marketplace, precisamos rastrear cada instalacao e desinstalacao de modulos. Esses dados alimentam: (1) analytics internas para o time Kaven entender quais modulos sao populares, (2) o campo `installCount` no Module para sorting por popularidade (M3.1), e (3) dashboards administrativos para tomada de decisao sobre features e pricing. O kaven-cli ja envia eventos de install/uninstall — o marketplace precisa recebe-los e agregar os dados. Sem tracking, voamos cegos sobre adocao e churn de modulos.

## User Story

**Como** administrador do marketplace,
**Quero** rastrear instalacoes e desinstalacoes de modulos com analytics agregadas,
**Para que** eu entenda quais modulos sao populares, identifique tendencias de churn e tome decisoes informadas sobre o marketplace.

---

## Acceptance Criteria

- [ ] Novo modelo `Installation` criado: `id`, `userId`, `moduleId`, `releaseId`, `version` (string), `projectId` (optional string), `installedAt` (DateTime), `uninstalledAt` (DateTime nullable)
- [ ] Endpoint `POST /installations` registra nova instalacao — body: `{ moduleId, releaseId, version, projectId? }`
- [ ] Endpoint `PATCH /installations/:id/uninstall` marca `uninstalledAt` com timestamp atual
- [ ] Ao registrar instalacao, incrementa `Module.installCount` atomicamente
- [ ] Ao registrar desinstalacao, decrementa `Module.installCount` atomicamente (nunca abaixo de 0)
- [ ] Endpoint `GET /analytics/installs?period=7d|30d|90d` retorna instalacoes por dia no periodo (admin only)
- [ ] Endpoint `GET /analytics/modules/popular?limit=10` retorna top N modulos por `installCount` (admin only)
- [ ] Endpoint `GET /analytics/modules/:slug/stats` retorna: `totalInstalls`, `activeInstalls` (sem `uninstalledAt`), `churnRate` (uninstalled/total * 100), `installsByDay` (ultimos 30 dias)
- [ ] Todos os endpoints analytics protegidos por role `ADMIN`
- [ ] `POST /installations` protegido por autenticacao (Bearer token)
- [ ] Prevenir instalacao duplicada: mesmo `userId` + `moduleId` + `projectId` sem `uninstalledAt` retorna 409 Conflict
- [ ] Validacao: `moduleId` e `releaseId` devem existir, senao retorna 404

## Technical Notes

### Prisma Model

```prisma
model Installation {
  id            String    @id @default(cuid())
  userId        String
  moduleId      String
  releaseId     String
  version       String
  projectId     String?
  installedAt   DateTime  @default(now())
  uninstalledAt DateTime?

  user    User    @relation(fields: [userId], references: [id])
  module  Module  @relation(fields: [moduleId], references: [id])
  release Release @relation(fields: [releaseId], references: [id])

  @@unique([userId, moduleId, projectId, uninstalledAt], name: "unique_active_installation")
  @@index([moduleId, installedAt])
  @@index([userId, installedAt])
  @@index([installedAt])
}
```

### Atomic installCount Update

```typescript
// Use Prisma transaction to ensure atomicity
await prisma.$transaction([
  prisma.installation.create({ data: { userId, moduleId, releaseId, version, projectId } }),
  prisma.module.update({
    where: { id: moduleId },
    data: { installCount: { increment: 1 } },
  }),
]);
```

### Analytics: Installs Per Day

```typescript
// Group by day using Prisma groupBy or raw query
const installsByDay = await prisma.$queryRaw`
  SELECT
    DATE(i."installedAt") AS date,
    COUNT(*)::int AS installs
  FROM "Installation" i
  WHERE i."installedAt" >= NOW() - INTERVAL '${period}'
  GROUP BY DATE(i."installedAt")
  ORDER BY date ASC
`;
```

### Analytics: Churn Rate

```typescript
const stats = await prisma.$queryRaw`
  SELECT
    COUNT(*)::int AS total_installs,
    COUNT(*) FILTER (WHERE "uninstalledAt" IS NULL)::int AS active_installs,
    ROUND(
      COUNT(*) FILTER (WHERE "uninstalledAt" IS NOT NULL)::numeric /
      GREATEST(COUNT(*)::numeric, 1) * 100, 2
    ) AS churn_rate
  FROM "Installation"
  WHERE "moduleId" = ${moduleId}
`;
```

### Edge Cases
- Desinstalacao de instalacao que ja tem `uninstalledAt` retorna 400
- Instalacao com `releaseId` de outro modulo retorna 400
- Analytics com periodo invalido retorna 400
- Modulo com 0 installs nao aparece no popular (a menos que tenha sido instalado e depois todo mundo desinstalou — nesse caso `installCount` seria 0)
- Race condition em installCount: usar transacao Prisma
