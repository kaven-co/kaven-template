# M3.8 — Sprint M3 Test Suite

> **Epic:** 1 (Marketplace) | **Sprint:** M3 | **Agent:** @qa
> **Priority:** P0
> **Depends on:** M3.1, M3.2, M3.3, M3.4, M3.5, M3.6, M3.7

---

## Contexto

Sprint M3 introduz search, analytics, audit logging, security hardening, structured logging e health probes. Cada um desses sistemas precisa de cobertura de testes robusta antes do marketplace ir para producao. Esta story consolida os testes de integracao end-to-end e testes de regressao que cortam transversalmente multiplas stories do sprint. Testes unitarios individuais sao responsabilidade de cada story — esta story foca nos testes que validam a interacao entre sistemas e cenarios que nenhuma story individual cobre sozinha. O target e 80%+ de cobertura total do projeto com no minimo 30 novos test cases.

## User Story

**Como** QA lead do marketplace,
**Quero** uma suite de testes abrangente que cubra todas as features do Sprint M3 incluindo interacoes entre sistemas,
**Para que** tenhamos confianca de que o marketplace esta pronto para producao com regressoes detectadas automaticamente.

## Acceptance Criteria

### Search Tests (M3.1)
- [ ] Full-text search retorna resultados relevantes para query "payments"
- [ ] Search com query que nao corresponde a nenhum modulo retorna array vazio e total 0
- [ ] Paginacao funcional: page 1 retorna primeiros N, page 2 retorna proximos N
- [ ] Filtro por category funcional: somente modulos da categoria retornados
- [ ] Filtro por tier funcional: somente modulos do tier retornados
- [ ] Sort by popularity retorna modulo com mais installs primeiro
- [ ] Sort by recent retorna modulo mais recente primeiro
- [ ] Sort by relevance retorna match mais relevante primeiro
- [ ] Category facets retornam contagens corretas
- [ ] Query com caracteres especiais nao causa SQL injection

### Installation Tracking Tests (M3.2)
- [ ] POST /installations registra instalacao e incrementa installCount
- [ ] Instalacao duplicada (mesmo user + module + project sem uninstall) retorna 409
- [ ] PATCH /installations/:id/uninstall marca uninstalledAt e decrementa installCount
- [ ] Desinstalacao de instalacao ja desinstalada retorna 400
- [ ] GET /analytics/installs retorna contagem correta por dia
- [ ] GET /analytics/modules/popular retorna top N ordenado corretamente
- [ ] GET /analytics/modules/:slug/stats calcula churn rate corretamente
- [ ] Analytics endpoints bloqueados para non-admin users (403)

### Audit Log Tests (M3.3)
- [ ] Criar licenca gera audit log com action "license.create"
- [ ] Revogar licenca gera audit log com action "license.revoke"
- [ ] Publicar release gera audit log com action "release.publish"
- [ ] Audit log captura userId, ipAddress, userAgent corretamente
- [ ] GET /admin/audit-logs retorna logs paginados
- [ ] Filtro por action funcional
- [ ] Filtro por userId funcional
- [ ] Filtro por date range funcional
- [ ] Cleanup job remove logs com mais de 90 dias
- [ ] Cleanup job nao remove logs com menos de 90 dias
- [ ] Anonymize funcao substitui userId e limpa ipAddress/userAgent
- [ ] Audit log nao bloqueia response principal (fire-and-forget)

### Security Tests (M3.5)
- [ ] Request de origem nao permitida nao recebe CORS headers (em producao mode)
- [ ] Request de origem permitida recebe CORS headers corretos
- [ ] Response headers incluem: X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security
- [ ] Tentativa de SQL injection via query param retorna resultado vazio (nao erro SQL)
- [ ] Tentativa de SQL injection via body field e sanitizada pelo Prisma
- [ ] Tentativa de XSS em campo de texto e escapada na response
- [ ] SSRF: URL com IP privado rejeitada com 400
- [ ] SSRF: URL com protocolo file:// rejeitada com 400
- [ ] SSRF: URL de dominio nao permitido rejeitada com 400
- [ ] Body maior que limite configurado retorna 413
- [ ] JWT com secret invalido retorna 401
- [ ] Env validation falha se JWT_SECRET < 32 chars

### Logging Tests (M3.6)
- [ ] Request sem X-Request-ID recebe correlationId gerado automaticamente
- [ ] Request com X-Request-ID usa o valor fornecido como correlationId
- [ ] Response header X-Request-ID presente com correlationId correto
- [ ] Logs de request contem correlationId
- [ ] Authorization header redactado nos logs (valor = [REDACTED])
- [ ] Password field redactado nos logs (valor = [REDACTED])

### Health Probe Tests (M3.7)
- [ ] GET /health retorna 200 com version, timestamp, uptime
- [ ] GET /health responde em < 10ms (sem I/O)
- [ ] GET /ready com todas dependencias ok retorna status "ok" e 200
- [ ] GET /ready com DB down retorna status "unhealthy" e 503
- [ ] GET /ready com Redis down retorna status "degraded" e 200
- [ ] GET /ready com S3 down retorna status "degraded" e 200
- [ ] GET /ready com Redis nao configurado retorna check "skipped"
- [ ] GET /ready com DB timeout (> 3s) retorna status "unhealthy"

### OpenAPI Spec Tests (M3.4)
- [ ] GET /docs/json retorna spec JSON valida
- [ ] Spec valida contra OpenAPI 3.1 schema
- [ ] Todas as routes registradas estao documentadas na spec
- [ ] Security scheme bearerAuth definido na spec
- [ ] Tags corretas para cada grupo de endpoints

### Cross-Cutting Integration Tests
- [ ] Fluxo completo: search → install → audit log created → analytics updated → uninstall → audit log created → analytics updated
- [ ] Correlation ID propagado de request ate audit log metadata
- [ ] Security headers presentes em todas as responses (including /health, /ready, /search)
- [ ] Error responses incluem correlationId no body ou header

### Security Test Patterns

```typescript
// tests/security/injection.test.ts
describe('SQL Injection Prevention', () => {
  it('should not execute injected SQL in search query', async () => {
    const response = await app.inject({
      method: 'GET',
      url: '/search?q=payments%27%3B%20DROP%20TABLE%20%22Module%22%3B--',
    });
    expect(response.statusCode).toBe(200);
    // Verify modules table still exists
    const count = await prisma.module.count();
    expect(count).toBeGreaterThanOrEqual(0);
  });

  it('should not execute injected SQL in body fields', async () => {
    const response = await app.inject({
      method: 'POST',
      url: '/modules',
      headers: { authorization: `Bearer ${adminToken}` },
      payload: {
        name: "test'; DROP TABLE \"Module\";--",
        description: 'Test module',
        category: 'devtools',
        tier: 'STARTER',
      },
    });
    // Should either create module with literal string name or reject with validation error
    expect([201, 400, 422]).toContain(response.statusCode);
  });
});
```

### Cross-Cutting Integration Test

```typescript
// tests/integration/m3-flow.integration.test.ts
describe('M3 Full Flow Integration', () => {
  it('search → install → audit → analytics → uninstall → audit → analytics', async () => {
    // 1. Search for module
    const searchRes = await app.inject({
      method: 'GET',
      url: '/search?q=payments',
      headers: { 'x-request-id': 'test-correlation-123' },
    });
    expect(searchRes.statusCode).toBe(200);
    expect(searchRes.headers['x-request-id']).toBe('test-correlation-123');
    const { data } = searchRes.json();
    expect(data.length).toBeGreaterThan(0);
    const moduleId = data[0].id;

    // 2. Install module
    const installRes = await app.inject({
      method: 'POST',
      url: '/installations',
      headers: { authorization: `Bearer ${userToken}` },
      payload: { moduleId, releaseId: testReleaseId, version: '1.0.0' },
    });
    expect(installRes.statusCode).toBe(201);
    const installationId = installRes.json().id;

    // 3. Verify audit log created
    const auditRes = await app.inject({
      method: 'GET',
      url: '/admin/audit-logs?action=installation.create',
      headers: { authorization: `Bearer ${adminToken}` },
    });
    expect(auditRes.json().data.length).toBeGreaterThan(0);

    // 4. Verify analytics updated
    const popularRes = await app.inject({
      method: 'GET',
      url: '/analytics/modules/popular',
      headers: { authorization: `Bearer ${adminToken}` },
    });
    expect(popularRes.statusCode).toBe(200);

    // 5. Uninstall
    const uninstallRes = await app.inject({
      method: 'PATCH',
      url: `/installations/${installationId}/uninstall`,
      headers: { authorization: `Bearer ${userToken}` },
    });
    expect(uninstallRes.statusCode).toBe(200);

    // 6. Verify churn reflected in stats
    const statsRes = await app.inject({
      method: 'GET',
      url: `/analytics/modules/payments/stats`,
      headers: { authorization: `Bearer ${adminToken}` },
    });
    const stats = statsRes.json();
    expect(stats.churnRate).toBeGreaterThan(0);
  });
});
```
