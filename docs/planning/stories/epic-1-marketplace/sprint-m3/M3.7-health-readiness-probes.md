# M3.7 — Health & Readiness Probes

> **Epic:** 1 (Marketplace) | **Sprint:** M3 | **Agent:** @devops
> **Priority:** P1
> **Depends on:** —

---

## Contexto

O marketplace sera deployado em containers (Docker/Kubernetes). Kubernetes (e qualquer orquestrador de containers) depende de health probes para decidir se um container esta vivo (liveness) e pronto para receber trafico (readiness). Sem probes adequados: (1) containers travados nao sao reiniciados, (2) containers que perderam conexao com o banco continuam recebendo trafico e retornando 500, (3) deploys blue/green nao sabem quando o novo container esta pronto. O endpoint `GET /health` ja existe de forma basica — precisamos expandi-lo e adicionar `GET /ready` com checks de dependencias reais (DB, Redis, S3).

## User Story

**Como** engenheiro de plataforma fazendo deploy do marketplace em Kubernetes,
**Quero** endpoints de liveness e readiness que verificam dependencias reais,
**Para que** o orquestrador reinicie containers travados, remova do load balancer containers com dependencias down, e valide que novos deploys estao prontos antes de receber trafico.

## Acceptance Criteria

### Liveness Probe (`GET /health`)
- [ ] Retorna 200 com body: `{ status: "ok", version: "x.y.z", timestamp: "ISO-8601", uptime: 12345 }`
- [ ] `version` lido de `package.json`
- [ ] `uptime` em segundos (process.uptime())
- [ ] Nao faz nenhum check de dependencia (proposito: verificar se o processo esta vivo)
- [ ] Tempo de resposta < 5ms (sem I/O)
- [ ] Sem autenticacao requerida

### Readiness Probe (`GET /ready`)
- [ ] Verifica tres dependencias: PostgreSQL (DB), Redis, S3
- [ ] Cada check tem timeout de 3 segundos
- [ ] Checks executam em paralelo (Promise.allSettled)
- [ ] Response shape:

```json
{
  "status": "ok" | "degraded" | "unhealthy",
  "checks": {
    "db": { "status": "ok", "responseTime": 12 },
    "redis": { "status": "ok", "responseTime": 5 },
    "s3": { "status": "ok", "responseTime": 45 }
  },
  "version": "x.y.z",
  "uptime": 12345,
  "timestamp": "2026-02-16T10:00:00Z"
}
```

- [ ] Regras de status:
  - **ok** (200): todas as dependencias ok
  - **degraded** (200): dependencia non-critical down (Redis ou S3)
  - **unhealthy** (503): dependencia critical down (DB)
- [ ] DB e **critical**: se DB down → status `unhealthy`, HTTP 503
- [ ] Redis e **non-critical**: se Redis down → status `degraded`, HTTP 200
- [ ] S3 e **non-critical**: se S3 down → status `degraded`, HTTP 200
- [ ] Se Redis nao configurado (REDIS_URL ausente), check retorna `{ status: "skipped" }`
- [ ] Se S3 nao configurado (S3_BUCKET ausente), check retorna `{ status: "skipped" }`
- [ ] Sem autenticacao requerida
- [ ] Response time do endpoint < 4s no pior caso (3s timeout + overhead)

### Kubernetes Compatibility
- [ ] Liveness probe: `GET /health` — restart on failure
- [ ] Readiness probe: `GET /ready` — remove from LB on 503
- [ ] Startup probe (optional): `GET /ready` with higher timeout — para cold starts

## Technical Notes

### DB Health Check

```typescript
async function checkDatabase(prisma: PrismaClient): Promise<HealthCheck> {
  const start = Date.now();
  try {
    await Promise.race([
      prisma.$queryRaw`SELECT 1`,
      new Promise((_, reject) => setTimeout(() => reject(new Error('DB timeout')), 3000)),
    ]);
    return { status: 'ok', responseTime: Date.now() - start };
  } catch (error) {
    return { status: 'error', responseTime: Date.now() - start, error: (error as Error).message };
  }
}
```

### Redis Health Check

```typescript
async function checkRedis(redis: Redis | null): Promise<HealthCheck> {
  if (!redis) return { status: 'skipped' };

  const start = Date.now();
  try {
    await Promise.race([
      redis.ping(),
      new Promise((_, reject) => setTimeout(() => reject(new Error('Redis timeout')), 3000)),
    ]);
    return { status: 'ok', responseTime: Date.now() - start };
  } catch (error) {
    return { status: 'error', responseTime: Date.now() - start, error: (error as Error).message };
  }
}
```

### S3 Health Check

```typescript
async function checkS3(s3Client: S3Client | null, bucket: string | undefined): Promise<HealthCheck> {
  if (!s3Client || !bucket) return { status: 'skipped' };

  const start = Date.now();
  try {
    await Promise.race([
      s3Client.send(new HeadBucketCommand({ Bucket: bucket })),
      new Promise((_, reject) => setTimeout(() => reject(new Error('S3 timeout')), 3000)),
    ]);
    return { status: 'ok', responseTime: Date.now() - start };
  } catch (error) {
    return { status: 'error', responseTime: Date.now() - start, error: (error as Error).message };
  }
}
```

### Readiness Route

```typescript
// src/routes/health.ts
fastify.get('/ready', {
  schema: {
    tags: ['Health'],
    summary: 'Readiness probe — checks all dependencies',
    response: {
      200: ReadinessResponseSchema,
      503: ReadinessResponseSchema,
    },
  },
}, async (request, reply) => {
  const [dbCheck, redisCheck, s3Check] = await Promise.allSettled([
    checkDatabase(fastify.prisma),
    checkRedis(fastify.redis),
    checkS3(fastify.s3, fastify.config.S3_BUCKET),
  ]);

  const checks = {
    db: dbCheck.status === 'fulfilled' ? dbCheck.value : { status: 'error', error: 'Check failed' },
    redis: redisCheck.status === 'fulfilled' ? redisCheck.value : { status: 'error', error: 'Check failed' },
    s3: s3Check.status === 'fulfilled' ? s3Check.value : { status: 'error', error: 'Check failed' },
  };

  // DB is critical
  const dbDown = checks.db.status === 'error';
  // Redis and S3 are non-critical
  const nonCriticalDown = checks.redis.status === 'error' || checks.s3.status === 'error';

  let status: 'ok' | 'degraded' | 'unhealthy';
  let statusCode: number;

  if (dbDown) {
    status = 'unhealthy';
    statusCode = 503;
  } else if (nonCriticalDown) {
    status = 'degraded';
    statusCode = 200;
  } else {
    status = 'ok';
    statusCode = 200;
  }

  return reply.status(statusCode).send({
    status,
    checks,
    version: fastify.config.version,
    uptime: Math.floor(process.uptime()),
    timestamp: new Date().toISOString(),
  });
});
```

### Types

```typescript
interface HealthCheck {
  status: 'ok' | 'error' | 'skipped';
  responseTime?: number;
  error?: string;
}

interface ReadinessResponse {
  status: 'ok' | 'degraded' | 'unhealthy';
  checks: {
    db: HealthCheck;
    redis: HealthCheck;
    s3: HealthCheck;
  };
  version: string;
  uptime: number;
  timestamp: string;
}
```
