# M3.1 — Module Search & Discovery

> **Epic:** 1 (Marketplace) | **Sprint:** M3 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** M1.6

---

## Contexto

Com o CRUD de modulos completo (M1) e o sistema de licenciamento e pagamentos funcional (M2), o marketplace precisa agora de uma experiencia de busca robusta. Desenvolvedores precisam encontrar modulos rapidamente por nome, descricao, categoria e tier de plano. PostgreSQL full-text search com tsvector oferece busca nativa de alta performance sem dependencias externas (Elasticsearch, Meilisearch), mantendo a stack simples e o custo operacional baixo. A busca e o ponto de entrada principal do marketplace — se nao funcionar bem, nenhum modulo sera descoberto.

## User Story

**Como** desenvolvedor usando o kaven-cli ou o marketplace web,
**Quero** buscar modulos por palavras-chave, categoria e tier de plano,
**Para que** eu encontre rapidamente o modulo certo para minha necessidade sem navegar manualmente por listas.

---

## Acceptance Criteria

- [ ] Endpoint `GET /search` aceita query params: `q` (texto livre), `category` (enum), `tier` (STARTER|COMPLETE|PRO), `sort` (popular|recent|relevance), `page`, `limit`
- [ ] Campo `search_vector` adicionado ao modelo `Module` como coluna gerada (`tsvector`) baseada em `name` e `description`
- [ ] Indice GIN criado sobre `search_vector` para performance de busca
- [ ] Busca full-text usa `plainto_tsquery` para queries simples e `websearch_to_tsquery` para queries avancadas (com operadores AND/OR/NOT)
- [ ] Resultados incluem `relevanceScore` (ts_rank) e `highlightedName`/`highlightedDescription` (ts_headline)
- [ ] Resposta inclui `facets.categories` com contagem de resultados por categoria
- [ ] Paginacao funcional: `page` (default 1), `limit` (default 20, max 100), resposta inclui `total`, `page`, `limit`, `totalPages`
- [ ] Busca sem `q` retorna todos os modulos (listagem) respeitando filtros de `category` e `tier`
- [ ] Busca com `q` vazio ou somente espacos retorna todos os modulos
- [ ] Sort `popular` ordena por `installCount` desc, `recent` por `createdAt` desc, `relevance` por `ts_rank` desc (default quando `q` presente)
- [ ] Resposta com 0 resultados retorna 200 com array vazio e `total: 0`
- [ ] Tempo de resposta < 100ms para queries tipicas (com indice GIN)

## Technical Notes

### Migration: Add search_vector

```sql
-- Migration: add_search_vector_to_module
ALTER TABLE "Module" ADD COLUMN "search_vector" tsvector
  GENERATED ALWAYS AS (
    setweight(to_tsvector('english', coalesce("name", '')), 'A') ||
    setweight(to_tsvector('english', coalesce("description", '')), 'B')
  ) STORED;

CREATE INDEX "Module_search_vector_idx" ON "Module" USING GIN ("search_vector");
```

### Search Query Pattern

```typescript
// Use Prisma.$queryRaw for full-text search (Prisma doesn't natively support tsvector)
const results = await prisma.$queryRaw`
  SELECT
    m.*,
    ts_rank(m.search_vector, websearch_to_tsquery('english', ${query})) AS relevance_score,
    ts_headline('english', m.name, websearch_to_tsquery('english', ${query}),
      'StartSel=<mark>, StopSel=</mark>, MaxWords=50, MinWords=10') AS highlighted_name,
    ts_headline('english', m.description, websearch_to_tsquery('english', ${query}),
      'StartSel=<mark>, StopSel=</mark>, MaxWords=80, MinWords=20') AS highlighted_description
  FROM "Module" m
  WHERE m.search_vector @@ websearch_to_tsquery('english', ${query})
    AND m."status" = 'PUBLISHED'
    ${category ? Prisma.sql`AND m."category" = ${category}` : Prisma.empty}
    ${tier ? Prisma.sql`AND m."tier" = ${tier}` : Prisma.empty}
  ORDER BY ${sort === 'relevance' ? Prisma.sql`relevance_score DESC` : sort === 'popular' ? Prisma.sql`m."installCount" DESC` : Prisma.sql`m."createdAt" DESC`}
  LIMIT ${limit} OFFSET ${(page - 1) * limit}
`;
```

### Category Facets

```typescript
// Separate query for facets (run in parallel with main search)
const facets = await prisma.$queryRaw`
  SELECT m."category", COUNT(*)::int AS count
  FROM "Module" m
  WHERE m.search_vector @@ websearch_to_tsquery('english', ${query})
    AND m."status" = 'PUBLISHED'
  GROUP BY m."category"
  ORDER BY count DESC
`;
```

### Zod Schema

```typescript
const SearchQuerySchema = z.object({
  q: z.string().optional().default(''),
  category: z.enum(['auth', 'billing', 'analytics', 'communication', 'storage', 'ui', 'integration', 'devtools']).optional(),
  tier: z.enum(['STARTER', 'COMPLETE', 'PRO']).optional(),
  sort: z.enum(['popular', 'recent', 'relevance']).optional().default('relevance'),
  page: z.coerce.number().int().min(1).optional().default(1),
  limit: z.coerce.number().int().min(1).max(100).optional().default(20),
});
```

### Edge Cases
- Query com caracteres especiais (SQL injection via raw query — use parameterized queries)
- Query muito longa (> 500 chars — truncate)
- Categoria invalida retorna 400
- Modulos com status != PUBLISHED nao aparecem nos resultados
