# M1.7 — Release Management

> **Epic:** 1 (Marketplace) | **Sprint:** M1 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** M1.6 (Module CRUD)

---

## Contexto

Cada módulo pode ter múltiplas releases (versões). O CLI precisa saber qual versão instalar e validar compatibilidade com a versão do Kaven framework. O schema `Release` já existe com `version`, `artifactUrl`, `signature`.

## User Story

**Como** publisher de módulos,
**Quero** publicar novas versões do meu módulo com versionamento semântico,
**Para que** os desenvolvedores possam instalar versões específicas e receber updates.

---

## Acceptance Criteria

- [ ] `POST /modules/:slug/releases` cria nova release (requer auth + é dono ou ADMIN)
- [ ] Validação semver no campo version (1.0.0, 1.2.3-beta.1)
- [ ] Rejeita versions duplicadas para o mesmo módulo (409 Conflict)
- [ ] `GET /modules/:slug/versions` lista todas as versions (ordenadas desc)
- [ ] `GET /modules/:slug/versions/latest` retorna a mais recente
- [ ] `GET /modules/:slug/versions/:version` retorna release específica
- [ ] Response inclui: version, artifactUrl, signature, createdAt, changelog
- [ ] Incrementa `Module.installCount` ao gerar download token (preparação para M2.2)

```typescript
const semverRegex = /^\d+\.\d+\.\d+(-[a-zA-Z0-9.]+)?$/;
const createReleaseSchema = z.object({
  version: z.string().regex(semverRegex, 'Invalid semver format'),
  artifactUrl: z.string().url(),
  signature: z.string().min(1),
  changelog: z.string().optional(),
  minKavenVersion: z.string().optional(),

  checksum: z.string().optional(),
});
```

### ReleaseService:
```typescript
export class ReleaseService {
  async create(moduleSlug: string, data: CreateReleaseDto) {
    const module = await this.moduleService.getBySlug(moduleSlug);
    // Check unique version
    const existing = await prisma.release.findUnique({
      where: { moduleId_version: { moduleId: module.id, version: data.version } }
    });
    if (existing) throw new ConflictError('Version already exists');
    return prisma.release.create({ data: { ...data, moduleId: module.id } });
  }
```
