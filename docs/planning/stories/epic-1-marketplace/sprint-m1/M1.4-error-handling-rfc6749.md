# M1.4 — Error Handling RFC 6749

> **Epic:** 1 (Marketplace) | **Sprint:** M1 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** Nenhum (pode rodar em paralelo)

---

## Contexto

Atualmente os erros são `throw new Error('invalid_grant')` — genéricos, sem formato padronizado, sem HTTP status codes corretos. O Fastify captura e retorna 500 para tudo.

Precisamos de error handling consistente seguindo RFC 6749 para auth e um padrão próprio para API errors.

## User Story

**Como** consumidor da API (CLI ou frontend),
**Quero** receber respostas de erro padronizadas e informativas,
**Para que** eu possa tratar erros programaticamente e mostrar mensagens úteis ao usuário.

---

## Acceptance Criteria

- [ ] Error responses OAuth seguem RFC 6749 formato:
  ```json
  { "error": "invalid_grant", "error_description": "Device code not found" }
  ```
- [ ] Error responses API seguem formato consistente:
  ```json
  { "error": "not_found", "message": "Module not found", "statusCode": 404 }
  ```
- [ ] HTTP status codes corretos para cada tipo de erro (400, 401, 403, 404, 409, 410, 422, 500)
- [ ] Erros não exposam stack traces ou detalhes internos em production
- [ ] Erros de validação Zod retornam 422 com detalhes dos campos
- [ ] Global error handler registrado no Fastify
- [ ] Custom error classes para cada tipo de erro

## Technical Notes

### Novo arquivo — Error classes:
```typescript
// src/errors/index.ts

export class AppError extends Error {
  constructor(
    public statusCode: number,
    public error: string,
    public message: string,
    public details?: unknown
  ) {
    super(message);
  }
}

export class NotFoundError extends AppError {
  constructor(resource = 'Resource') {
    super(404, 'not_found', `${resource} not found`);
  }
}

export class UnauthorizedError extends AppError {
  constructor(description = 'Authentication required') {
    super(401, 'unauthorized', description);
  }
}

export class ForbiddenError extends AppError {
  constructor(description = 'Insufficient permissions') {
    super(403, 'forbidden', description);
  }
}

export class ConflictError extends AppError {
  constructor(description = 'Resource already exists or was already used') {
    super(409, 'conflict', description);
  }
}

export class GoneError extends AppError {
  constructor(description = 'Resource has expired') {
    super(410, 'expired_token', description);
  }
}

export class ValidationError extends AppError {
  constructor(details: unknown) {
    super(422, 'validation_error', 'Invalid input', details);
  }
}

// OAuth-specific errors (RFC 6749 Section 5.2)
export class OAuthError extends Error {
  constructor(
    public statusCode: number,
    public error: string,
    public error_description: string
  ) {
    super(error_description);
  }
}
```

### Global error handler no server.ts:
```typescript
server.setErrorHandler((error, request, reply) => {
  if (error instanceof AppError) {
    return reply.status(error.statusCode).send({
      error: error.error,
      message: error.message,
      ...(error.details && { details: error.details }),
    });
  }

  if (error instanceof OAuthError) {
    return reply.status(error.statusCode).send({
      error: error.error,
      error_description: error.error_description,
    });
  }

  // Zod validation errors
  if (error.validation) {
    return reply.status(422).send({
      error: 'validation_error',
      message: 'Invalid input',
      details: error.validation,
    });
  }

  // Unknown errors — hide details in production
  request.log.error(error);
  return reply.status(500).send({
    error: 'server_error',
    message: process.env.NODE_ENV === 'production'
      ? 'Internal server error'
      : error.message,
  });
});
```
