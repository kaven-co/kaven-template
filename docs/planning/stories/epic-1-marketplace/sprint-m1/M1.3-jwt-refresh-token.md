# M1.3 — JWT Refresh Token Flow

> **Epic:** 1 (Marketplace) | **Sprint:** M1 | **Agent:** @dev
> **Priority:** P1
> **Depends on:** M1.1 (device code gera tokens), M1.2 (OAuth gera tokens)

---

## Contexto

O AuthService já gera `access_token` (1h) e `refresh_token` (7d) no `pollToken()`, mas não existe endpoint para refresh, e o middleware `authenticate` não faz role checking.

## User Story

**Como** desenvolvedor autenticado,
**Quero** que meu token seja renovado automaticamente sem precisar re-autenticar,
**Para que** minha sessão não expire durante uso normal do CLI ou portal.

---

## Acceptance Criteria

- [ ] `POST /auth/refresh` aceita refresh_token e retorna novo access_token + refresh_token
- [ ] Refresh tokens são single-use (invalidados após uso — rotation)
- [ ] Refresh tokens expirados retornam 401
- [ ] Refresh tokens inválidos/reusados invalidam TODOS os tokens do user (security)
- [ ] Middleware `authenticate` extrai role do JWT e popula `request.user`
- [ ] Decorator `requireRole('ADMIN')` restringe endpoints a admins
- [ ] `request.user` contém: `{ id, role, email, githubId }`
- [ ] Tokens incluem `iat` e `jti` (JWT ID) para tracking

## Technical Notes

### Novo modelo Prisma (adicionar ao schema):
```prisma
model RefreshToken {
  id        String   @id @default(uuid())
  token     String   @unique
  userId    String   @map("user_id")
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  expiresAt DateTime @map("expires_at")
  usedAt    DateTime? @map("used_at")
  revokedAt DateTime? @map("revoked_at")
  replacedBy String? @map("replaced_by")
  createdAt DateTime @default(now()) @map("created_at")

  @@index([userId])
  @@index([token])
  @@map("refresh_tokens")
}
```

### Rota:
```typescript
POST /auth/refresh
Body: { refresh_token: string }
→ 200: { access_token, refresh_token, expires_in }
→ 401: { error: "invalid_grant", error_description: "Refresh token expired or invalid" }
```

### Middleware aprimorado:
```typescript
// Em plugins/auth.ts
fastify.decorate('authenticate', async (request, reply) => {
  try {
    const decoded = await request.jwtVerify();
    request.user = {
      id: decoded.sub,
      role: decoded.role,
      email: decoded.email,
      githubId: decoded.githubId,
    };
  } catch (err) {
    reply.status(401).send({ error: 'unauthorized' });
  }
});

fastify.decorate('requireRole', (role: string) => {
  return async (request, reply) => {
    if (request.user?.role !== role) {
      reply.status(403).send({ error: 'forbidden', error_description: `Requires role: ${role}` });
    }
  };
});
```

### Type augmentation:
```typescript
declare module 'fastify' {
  interface FastifyRequest {
    user?: { id: string; role: string; email: string; githubId: string };
  }
  interface FastifyInstance {
    requireRole: (role: string) => (req: FastifyRequest, reply: FastifyReply) => Promise<void>;
  }
}
```
