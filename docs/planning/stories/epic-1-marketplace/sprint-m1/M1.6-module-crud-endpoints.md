# M1.6 — Module CRUD Endpoints

> **Epic:** 1 (Marketplace) | **Sprint:** M1 | **Agent:** @dev
> **Priority:** P0 — funcionalidade core do marketplace
> **Depends on:** M1.5 (migrations)

---

## Contexto

O schema tem `Module` e `Release` models prontos, mas zero endpoints existem. O CLI precisa de `GET /modules` para listar e `GET /modules/:slug` para buscar manifests.

## User Story

**Como** desenvolvedor usando o kaven-cli,
**Quero** buscar módulos disponíveis no marketplace,
**Para que** eu possa descobrir, avaliar e instalar módulos no meu projeto.

---

## Acceptance Criteria

### Listagem
- [ ] `GET /modules` retorna lista paginada de módulos públicos
- [ ] Query params: `?page=1&limit=20&category=payments&sort=newest|popular`
- [ ] Response inclui: slug, name, description, latestVersion, installCount, createdAt
- [ ] Módulos privados (`isPublic=false`) não aparecem na listagem pública
- [ ] Módulos privados aparecem para ADMIN

### Detalhes
- [ ] `GET /modules/:slug` retorna detalhes completos do módulo
- [ ] Inclui: todas as releases com versions, description, compatibility info
- [ ] Inclui: latest release com artifactUrl (se autenticado + licença válida)
- [ ] 404 para slugs inexistentes

### Criação (Admin)
- [ ] `POST /modules` cria novo módulo (requer role ADMIN)
- [ ] Validação: slug único, formato kebab-case, name obrigatório
- [ ] 409 se slug já existe

### Atualização (Admin)
- [ ] `PUT /modules/:slug` atualiza módulo existente (requer ADMIN)
- [ ] Campos atualizáveis: name, description, isPublic

### Deleção (Admin)
- [ ] `DELETE /modules/:slug` soft-delete (marca isPublic=false, ou campo deletedAt)
- [ ] Não deleta releases existentes
