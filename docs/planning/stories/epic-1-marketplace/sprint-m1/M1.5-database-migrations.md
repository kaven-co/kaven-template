# M1.5 — Database Migrations

> **Epic:** 1 (Marketplace) | **Sprint:** M1 | **Agent:** @data-engineer
> **Priority:** P0 — foundation para todos os outros
> **Depends on:** Nenhum

---

## Contexto

O marketplace usa `prisma db push` (dev only) sem migrations formais. Para production, precisamos de migration files versionadas. Também precisamos adicionar o modelo `RefreshToken` (M1.3) e indexes de performance.

## User Story

**Como** engenheiro de dados,
**Quero** que o schema do marketplace tenha migrations versionadas e indexes otimizados,
**Para que** possamos deploy em production com controle de versão do schema.

---

## Acceptance Criteria

- [ ] Migration inicial gerada a partir do schema atual (`prisma migrate dev --name init`)
- [ ] Migration para o modelo RefreshToken
- [ ] Indexes adicionados para queries frequentes:
  - `DeviceCode.userCode` (lookup na rota de approval)
  - `DeviceCode.status` + `expiresAt` (cleanup de expirados)
  - `License.userId` + `status` (lookup de licenças ativas)
  - `Module.slug` (já tem @unique, confirmar index)
  - `Release.moduleId` + `version` (já tem @@unique)
  - `RefreshToken.userId` (lookup de tokens por user)
  - `RefreshToken.token` (lookup na rota de refresh)
- [ ] Script de cleanup de device codes expirados (cron-ready)
- [ ] Seed data atualizado para incluir RefreshToken fixtures
- [ ] `pnpm db:migrate` funciona no CI

### Schema additions:
```prisma
// Adicionar ao schema.prisma

model RefreshToken {
  id         String    @id @default(uuid())
  token      String    @unique
  userId     String    @map("user_id")
  user       User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  expiresAt  DateTime  @map("expires_at")
  usedAt     DateTime? @map("used_at")
  revokedAt  DateTime? @map("revoked_at")
  replacedBy String?   @map("replaced_by")
  createdAt  DateTime  @default(now()) @map("created_at")

  @@index([userId])
  @@index([token])
  @@map("refresh_tokens")
}

// Indexes extras no DeviceCode
model DeviceCode {
  // ... campos existentes ...
  @@index([userCode])            // ADD
  @@index([status, expiresAt])   // ADD
}

// Indexes extras no License
model License {
  // ... campos existentes ...
  @@index([userId, status])      // ADD
}

// User precisa da relation RefreshToken
model User {
  // ... campos existentes ...
  refreshTokens RefreshToken[]   // ADD
}
```

### Cleanup script:
```typescript
// scripts/cleanup-expired-codes.ts
import { PrismaClient } from '@prisma/client';
const prisma = new PrismaClient();

async function cleanup() {
  const deleted = await prisma.deviceCode.deleteMany({
    where: { expiresAt: { lt: new Date() }, status: { in: ['PENDING', 'DENIED'] } }
  });
  console.log(`Cleaned ${deleted.count} expired device codes`);

  const expiredTokens = await prisma.refreshToken.deleteMany({
    where: { expiresAt: { lt: new Date() } }
  });
  console.log(`Cleaned ${expiredTokens.count} expired refresh tokens`);
}

cleanup();
```
