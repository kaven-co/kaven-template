# M1.9 — Sprint M1 Test Suite

> **Epic:** 1 (Marketplace) | **Sprint:** M1 | **Agent:** @qa
> **Priority:** P0 — gate para fechar o sprint
> **Depends on:** M1.1 — M1.8 (tudo do sprint)

---

## Contexto

O marketplace tem apenas 1 teste placeholder (`expect(true).toBe(true)`). Após M1.1-M1.8, precisamos de uma suite completa cobrindo todos os endpoints e serviços. Target: 80%+ coverage.

## User Story

**Como** QA engineer,
**Quero** uma suite de testes abrangente para todas as features do Sprint M1,
**Para que** tenhamos confiança de que a API funciona corretamente antes de avançar.

---

## Acceptance Criteria

### Cobertura por área:

#### Auth — Device Code (M1.1)
- [ ] Criar device code → retorna code + user_code + URI
- [ ] Buscar device code por user_code → retorna detalhes
- [ ] Buscar user_code inexistente → 404
- [ ] Buscar user_code expirado → 410
- [ ] Aprovar device code → status APPROVED
- [ ] Aprovar sem auth → 401
- [ ] Aprovar expirado → 410
- [ ] Aprovar já usado → 409
- [ ] Negar device code → status DENIED
- [ ] Poll token com código aprovado → retorna JWT
- [ ] Poll token pendente → authorization_pending error
- [ ] Poll token expirado → expired_token error

#### Auth — GitHub OAuth (M1.2)
- [ ] GET /auth/github → redirect 302 para GitHub
- [ ] Callback com code válido → cria user + retorna JWT
- [ ] Callback com user existente → login sem duplicar
- [ ] Callback com code inválido → error handling

#### Auth — Refresh Tokens (M1.3)
- [ ] Refresh com token válido → novos tokens
- [ ] Refresh com token expirado → 401
- [ ] Refresh com token já usado → revoga família
- [ ] Token rotation: old token invalidado após uso

#### Error Handling (M1.4)
- [ ] AppError retorna formato correto com status code
- [ ] OAuthError retorna formato RFC 6749
- [ ] Zod validation error → 422 com detalhes
- [ ] Unknown error → 500 sem stack trace em production

#### Modules (M1.6)
- [ ] Listar módulos públicos → pagination funcional
- [ ] Listar com filtros (category, sort) → filtrado corretamente
- [ ] Buscar por slug → detalhes completos
- [ ] Buscar slug inexistente → 404
- [ ] Criar módulo (admin) → 201
- [ ] Criar módulo (user) → 403
- [ ] Criar slug duplicado → 409
- [ ] Atualizar módulo (admin) → 200
- [ ] Soft delete (admin) → não aparece mais na listagem

#### Releases (M1.7)
- [ ] Criar release → versão registrada
- [ ] Criar versão duplicada → 409
- [ ] Versão inválida (não semver) → 422
- [ ] Listar versões → ordenadas desc
- [ ] Buscar latest → retorna mais recente
- [ ] Buscar versão específica → retorna correta

#### Validation (M1.8)
- [ ] Body vazio em POST → 422
- [ ] UUID inválido em params → 422
- [ ] Pagination fora do range → defaults
- [ ] Slug com caracteres inválidos → 422

### Setup de teste:
```typescript
// tests/setup.ts
import Fastify from 'fastify';
import { PrismaClient } from '@prisma/client';

// Use test database
const prisma = new PrismaClient({
  datasources: { db: { url: process.env.TEST_DATABASE_URL } }
});

export async function buildTestApp() {
  const app = Fastify();
  // Register all plugins and routes
  await app.ready();
  return app;
}

export async function cleanDatabase() {
  await prisma.$transaction([
    prisma.refreshToken.deleteMany(),
    prisma.deviceCode.deleteMany(),
    prisma.release.deleteMany(),
    prisma.license.deleteMany(),
    prisma.module.deleteMany(),
    prisma.user.deleteMany(),
  ]);
}
```
