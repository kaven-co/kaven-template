# C1.1 — Real Device Code Flow

> **Epic:** 2 (CLI) | **Sprint:** C1 | **Agent:** @dev
> **Priority:** P0
> **Depends on:** M1.1 (Marketplace Auth — Device Code Grant endpoint)

---

## Contexto

The current `src/commands/auth/login.ts` uses a completely mocked device code flow that returns a hardcoded JWT token (`jwt-mock-token-for-kaven-explorador`). The `AuthService` stores this mock token in `~/.kaven/auth.json` but never contacts any real API. For the CLI to work with the real kaven-marketplace, we need to implement the full OAuth 2.0 Device Authorization Grant (RFC 8628) flow against the marketplace's `/auth/device-code` and `/auth/token` endpoints.

The device code flow is ideal for CLI tools because it doesn't require the CLI to handle secrets or run a local HTTP server — the user authenticates in their browser, and the CLI polls for completion.

## User Story

**As a** developer using kaven-cli,
**I want to** authenticate with my real Kaven account via device code flow,
**So that** I can access marketplace modules, manage licenses, and publish modules under my identity.

## Acceptance Criteria

- [ ] `kaven auth login` sends `POST /auth/device-code` with `{ client_id: "kaven-cli" }` to the marketplace API
- [ ] Response contains `{ device_code, user_code, verification_uri, expires_in, interval }`
- [ ] CLI displays user_code prominently: `"Your code: ABCD-1234"`
- [ ] CLI auto-opens `verification_uri` in default browser using the `open` npm package
- [ ] If browser open fails (SSH session, headless), show URL manually: `"Open this URL: https://auth.kaven.sh/device?code=ABCD-1234"`
- [ ] CLI polls `POST /auth/token` with `{ device_code, grant_type: "urn:ietf:params:oauth:grant-type:device_code" }`
- [ ] Polling uses exponential backoff: 5s initial → 10s → 15s → 20s (capped)
- [ ] Spinner displays countdown: `"Waiting for authorization... (expires in 4:32)"`
- [ ] On `authorization_pending` response: continue polling silently
- [ ] On `slow_down` response: increase interval by 5s
- [ ] On success (200 with tokens): store JWT in `~/.kaven/auth.json`, show `"Logged in as user@email.com"`
- [ ] On `access_denied`: show error `"Authorization denied by user. Try again with 'kaven auth login'."`
- [ ] On `expired_token`: show error `"Device code expired. Run 'kaven auth login' again."`
- [ ] On network error: show `"Network error. Check your connection and try again."` with retry suggestion
- [ ] Max polling duration: 5 minutes (300s), then timeout with expiry message
- [ ] Stored auth.json format: `{ access_token, refresh_token, expires_at, user: { email, githubId, tier } }`
- [ ] File permissions on auth.json: `0600` (owner read/write only)

### Target Implementation Architecture

```typescript
// src/commands/auth/login.ts — TARGET
import open from 'open';
import ora from 'ora';
import { MarketplaceClient } from '../../infrastructure/MarketplaceClient';
import { AuthService } from '../../core/AuthService';

export async function loginCommand() {
  const client = new MarketplaceClient();
  const authService = new AuthService();

  // Step 1: Request device code
  const { device_code, user_code, verification_uri, expires_in, interval } =
    await client.requestDeviceCode();

  // Step 2: Display code and open browser
  console.log(`\n  Your verification code: ${chalk.bold.cyan(user_code)}\n`);

  try {
    await open(verification_uri);
    console.log(chalk.dim('  Browser opened automatically.'));
  } catch {
    console.log(`  Open this URL: ${chalk.underline(verification_uri)}`);
  }

  // Step 3: Poll for token with exponential backoff
  const spinner = ora('Waiting for authorization...').start();
  const token = await pollForToken(client, device_code, expires_in, interval, spinner);

  // Step 4: Store token
  authService.saveTokens(token);
  spinner.succeed(`Logged in as ${chalk.green(token.user.email)}`);
}

async function pollForToken(
  client: MarketplaceClient,
  deviceCode: string,
  expiresIn: number,
  initialInterval: number,
  spinner: ora.Ora
): Promise<AuthTokens> {
  const deadline = Date.now() + expiresIn * 1000;
  let interval = initialInterval;
  const backoffIntervals = [5, 10, 15, 20]; // seconds
  let backoffIndex = 0;

  while (Date.now() < deadline) {
    const remaining = Math.ceil((deadline - Date.now()) / 1000);
    const mm = Math.floor(remaining / 60);
    const ss = String(remaining % 60).padStart(2, '0');
    spinner.text = `Waiting for authorization... (expires in ${mm}:${ss})`;

    await sleep(interval * 1000);

    try {
      const result = await client.pollDeviceToken(deviceCode);

      if (result.status === 'success') return result.tokens;
      if (result.status === 'slow_down') interval += 5;
      if (result.status === 'access_denied') {
        throw new Error('Authorization denied by user. Try again with \'kaven auth login\'.');
      }
      if (result.status === 'expired_token') {
        throw new Error('Device code expired. Run \'kaven auth login\' again.');
      }
      // authorization_pending: continue polling
    } catch (error) {
      if (error.code === 'ECONNREFUSED' || error.code === 'ENOTFOUND') {
        throw new Error('Network error. Check your connection and try again.');
      }
      throw error;
    }

    // Exponential backoff
    if (backoffIndex < backoffIntervals.length - 1) {
      backoffIndex++;
      interval = backoffIntervals[backoffIndex];
    }
  }

  throw new Error('Device code expired. Run \'kaven auth login\' again.');
}
```

### API Endpoints (from M1.1)

| Endpoint | Method | Body | Response |
|----------|--------|------|----------|
| `/auth/device-code` | POST | `{ client_id }` | `{ device_code, user_code, verification_uri, expires_in, interval }` |
| `/auth/token` | POST | `{ device_code, grant_type }` | `{ access_token, refresh_token, user }` or `{ error }` |
