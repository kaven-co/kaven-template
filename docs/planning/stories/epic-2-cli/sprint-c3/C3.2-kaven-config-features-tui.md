# C3.2 — `kaven config features` — Feature Flag TUI

> **Epic:** 2 (CLI) | **Sprint:** C3 | **Agent:** @kaven-api-dev
> **Priority:** P2
> **Depends on:** None

---

## Contexto

O Kaven Framework possui **48 capabilities** (feature flags) que controlam quais funcionalidades estão disponíveis por tenant e por tier de plano. Atualmente (GAP-3 do `AIOX-KAVEN-INTEGRATION.md`), a configuração dessas capabilities é feita **editando manualmente** o arquivo `packages/database/prisma/seeds/capabilities.seed.ts`.

Esse processo é:
- Não documentado para novos usuários do framework
- Propenso a erros (nomes de capabilities errados, tipos inconsistentes)
- Não gerencia a diferença entre capabilities booleanas (35) e numéricas (13)
- Não tem validação de limites ou conflitos entre capabilities

O objetivo desta story é criar um **TUI interativo** (`kaven config features`) que guia o desenvolvedor pela seleção das 48 capabilities, agrupadas por categoria e com defaults por tier (Starter/Complete/Pro/Enterprise), e que gera automaticamente o seed file correto.

## User Story

**As a** developer configuring a new Kaven project,
**I want to** interactively select feature flags via a guided CLI wizard,
**So that** my project's capability seed file is generated correctly without manually editing TypeScript.

---

## Acceptance Criteria

- [ ] `kaven config features` abre TUI interativo com lista das 48 capabilities agrupadas por categoria
- [ ] `kaven config features --tier <starter|complete|pro|enterprise>` pré-seleciona o conjunto padrão do tier e pula para confirmação
- [ ] Capabilities booleanas são exibidas como checkboxes (`[ ] FEATURE_CUSTOM_DOMAIN`)
- [ ] Capabilities numéricas são exibidas como inputs numéricos com valor padrão do tier (`MAX_TEAM_MEMBERS: 5`)
- [ ] Capabilities agrupadas por categoria: Auth, Tenancy, Billing, API Access, Support, Limits
- [ ] Ao confirmar, gera `packages/database/prisma/seeds/capabilities.seed.ts` com as capabilities selecionadas
- [ ] Arquivo gerado é válido TypeScript — `tsc --noEmit` não reporta erros
- [ ] Se o arquivo já existe: pergunta se deve sobrescrever (`--force` para pular confirmação)
- [ ] `--dry-run` mostra o arquivo que seria gerado sem salvar
- [ ] Após geração, exibe instrução: `"Run pnpm db:seed to apply capabilities to your database"`
- [ ] Capabilities de tier superior são marcadas visualmente como locked (mas selecionáveis com warning)

### Command Registration

```typescript
// kaven-cli: src/commands/config/features.ts

import { Command } from 'commander';
import { checkbox, input, confirm } from '@inquirer/prompts';

export function registerConfigFeaturesCommand(config: Command) {
  config
    .command('features')
    .description('Interactive TUI to configure feature flags (capabilities)')
    .option('--tier <tier>', 'Start with tier defaults (starter|complete|pro|enterprise)')
    .option('--force', 'Overwrite existing seed file without confirmation')
    .option('--dry-run', 'Show generated file without saving')
    .action(async (opts) => {
      await configFeaturesCommand(opts);
    });
}
```

### Output — Seed File Gerado

```typescript
// packages/database/prisma/seeds/capabilities.seed.ts
// Generated by kaven config features — 2026-04-01T10:00:00Z
// Tier: complete

import { PrismaClient } from '@prisma/client';

export async function seedCapabilities(prisma: PrismaClient) {
  const capabilities = [
    { key: 'FEATURE_EMAIL_VERIFICATION', type: 'boolean', defaultValue: 'true', description: 'Email verification on signup' },
    { key: 'FEATURE_CUSTOM_DOMAIN', type: 'boolean', defaultValue: 'true', description: 'Custom domain per tenant' },
    { key: 'MAX_TEAM_MEMBERS', type: 'numeric', defaultValue: '25', description: 'Maximum team members per tenant' },
    // ...
  ];

  for (const cap of capabilities) {
    await prisma.capability.upsert({
      where: { key: cap.key },
      update: cap,
      create: cap,
    });
  }
}
```
