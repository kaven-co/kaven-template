# C3.1 — `kaven init` AIOX Environment Bootstrap

> **Epic:** 2 (CLI) | **Sprint:** C3 | **Agent:** @kaven-devops
> **Priority:** P1
> **Depends on:** feat/aiox-bridge (GAP-1 parcial — AIOX Core install via `--with-squad` já implementado)

---

## Contexto

O GAP-1 do `AIOX-KAVEN-INTEGRATION.md` foi parcialmente resolvido na branch `feat/aiox-bridge`: o `kaven init --with-squad` já chama `npx aiox-core@5.0.3 install --quiet` após o scaffold. No entanto, a instalação do AIOX Core por si só não é suficiente.

A **Phase 0.3** do fluxo Greenfield Kaven define um step **obrigatório** após o install do AIOX: o `@devops *environment-bootstrap`. Esse step produz:

- `.aiox/config.yaml` — configuração runtime do AIOX para o projeto específico
- `.aiox/environment-report.json` — relatório do ambiente detectado (stack, versões, estrutura)

Sem esses artefatos, todos os agents de planning (`@analyst`, `@pm`, `@architect`, `@po`) iniciam sessões "às cegas" — sem contexto do ambiente. O `greenfield-handler.js` do AIOX v5 **espera esses arquivos** antes de avançar para as Phases 1-2. Projetos que pulam esse step terão agents tomando decisões arquiteturais sem conhecer a stack real instalada.

Esta story completa o GAP-1 implementando o passo final: chamar o environment-bootstrap dentro do `targetDir` após o AIOX Core ser instalado.

## User Story

**As a** developer that just ran `kaven init my-saas --with-squad`,
**I want to** have the AIOX environment automatically bootstrapped after installation,
**So that** all planning agents have full context about my project environment from the very first session.

---

## Acceptance Criteria

- [ ] Após `kaven init <name> --with-squad`, o step de environment-bootstrap é executado automaticamente dentro do `targetDir`
- [ ] `.aiox/config.yaml` existe no projeto criado com valores corretos do ambiente detectado
- [ ] `.aiox/environment-report.json` existe com stack, versões (Node, pnpm, Docker), e estrutura do projeto
- [ ] Se o environment-bootstrap falhar (ex: Docker não disponível), o `kaven init` exibe warning mas **não falha** — projeto é criado, usuário pode rodar manualmente depois
- [ ] Output do bootstrap é exibido com spinner e mensagem de progresso: `"Bootstrapping AIOX environment..."`
- [ ] Se `--skip-aiox` flag for passado, o step de bootstrap é pulado inteiramente
- [ ] O step de bootstrap ocorre **após** o install do AIOX Core e **após** o git init do projeto
- [ ] Mensagem de sucesso ao final inclui instrução de como rodar o bootstrap manualmente: `kaven aiox bootstrap`
- [ ] `kaven aiox bootstrap` (novo subcommand) permite rodar o bootstrap em projetos existentes

### Implementação do Bootstrap

```typescript
// src/commands/init/aiox-bootstrap.ts

import { execSync } from 'child_process';
import * as fs from 'fs';
import * as path from 'path';
import ora from 'ora';
import chalk from 'chalk';

export async function runEnvironmentBootstrap(
  projectDir: string,
  options: { skipAiox?: boolean }
): Promise<void> {
  if (options.skipAiox) return;

  // Verificar se AIOX Core foi instalado
  const aioxCorePath = path.join(projectDir, '.aiox-core');
  if (!fs.existsSync(aioxCorePath)) {
    console.warn(chalk.yellow('  ⚠ AIOX Core not found — skipping environment bootstrap'));
    return;
  }

  const spinner = ora('Bootstrapping AIOX environment...').start();

  try {
    // Chamar o environment-bootstrap via AIOX CLI
    // O aiox-core expõe: node .aiox-core/bin/aiox.js devops environment-bootstrap
    execSync('node .aiox-core/bin/aiox.js devops environment-bootstrap --quiet', {
      cwd: projectDir,
      stdio: 'pipe',
      timeout: 30000,
    });
    spinner.succeed('AIOX environment bootstrapped');
  } catch (error) {
    spinner.warn('AIOX environment bootstrap failed — run manually: kaven aiox bootstrap');
    // Não falha o init — bootstrap é step pós-criação
  }
}
```

### Novo Subcommand `kaven aiox bootstrap`

```typescript
// src/commands/aiox/index.ts

export function registerAioxCommand(program: Command) {
  const aiox = program
    .command('aiox')
    .description('AIOX integration utilities');

  aiox
    .command('bootstrap')
    .description('Run AIOX environment bootstrap in current project')
    .action(async () => {
      const projectDir = process.cwd();
      await runEnvironmentBootstrap(projectDir, {});
    });
}
```
