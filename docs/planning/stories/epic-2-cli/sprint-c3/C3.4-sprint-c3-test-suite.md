## Contexto

Esta story garante a cobertura de testes completa para as três features implementadas no Sprint C3:

- **C3.1** — AIOX Environment Bootstrap (`kaven init --with-squad` + `kaven aiox bootstrap`)
- **C3.2** — Feature Flag TUI (`kaven config features`)
- **C3.3** — Module Activation Wizard (`kaven module activate/deactivate/list`)

O padrão de testes do kaven-cli usa **Vitest** com **MSW** para mockar chamadas externas e **memfs** para operações de filesystem sem IO real. O objetivo é atingir cobertura de 85%+ nas funcionalidades novas, com ênfase em edge cases de filesystem, erros de rede, e fluxos de rollback.

O Sprint C2 estabeleceu 255 testes e o modelo de qualidade que deve ser seguido: testes de integração que cobrem o happy path + testes unitários para cada módulo auxiliar.

## User Story

**As a** kaven-squad QA engineer,
**I want to** have comprehensive tests for all Sprint C3 features,
**So that** regressions in the AIOX integration, feature flag generation, and module activation are caught before merging.

## Technical Notes

### Setup de Testes — Mocking Pattern

```typescript
// tests/commands/aiox/bootstrap.test.ts
import { vi, describe, it, expect, beforeEach } from 'vitest';
import { vol } from 'memfs';

vi.mock('fs', async () => {
  const memfs = await import('memfs');
  return memfs.fs;
});

vi.mock('child_process', () => ({
  execSync: vi.fn(),
}));

describe('runEnvironmentBootstrap', () => {
  beforeEach(() => {
    vol.reset();
  });

  it('skips bootstrap when .aiox-core is not present', async () => {
    vol.fromJSON({ '/project/package.json': '{}' });
    const { execSync } = await import('child_process');

    await runEnvironmentBootstrap('/project', {});

    expect(execSync).not.toHaveBeenCalled();
  });

  it('calls environment-bootstrap when .aiox-core is present', async () => {
    vol.fromJSON({
      '/project/.aiox-core/bin/aiox.js': '// aiox cli',
    });
    const { execSync } = await import('child_process');

    await runEnvironmentBootstrap('/project', {});

    expect(execSync).toHaveBeenCalledWith(
      'node .aiox-core/bin/aiox.js devops environment-bootstrap --quiet',
      expect.objectContaining({ cwd: '/project' })
    );
  });

  it('warns but does not throw when execSync fails', async () => {
    vol.fromJSON({ '/project/.aiox-core/bin/aiox.js': '' });
    const { execSync } = await import('child_process');
    vi.mocked(execSync).mockImplementationOnce(() => { throw new Error('Bootstrap failed'); });

    // Should not throw
    await expect(runEnvironmentBootstrap('/project', {})).resolves.toBeUndefined();
  });
});
```

### Schema Modifier Test Pattern

```typescript
// tests/lib/schema-modifier.test.ts

describe('schemaModifier', () => {
  const SCHEMA_WITH_BILLING = `
model Invoice {
  id       String @id
  tenantId String
}

model Order {
  id       String @id
  tenantId String
}
`;

  it('deactivates module by commenting out models', () => {
    const result = deactivateModule(SCHEMA_WITH_BILLING, ['Invoice', 'Order']);
    expect(result).toContain('// model Invoice {');
    expect(result).toContain('// model Order {');
    expect(result).not.toContain('\nmodel Invoice');
  });

  it('round-trip preserves original schema', () => {
    const deactivated = deactivateModule(SCHEMA_WITH_BILLING, ['Invoice']);
    const reactivated = activateModule(deactivated, ['Invoice']);
    expect(reactivated).toBe(SCHEMA_WITH_BILLING);
  });
});
```
