# C3.3 — `kaven module activate` — Module Activation Wizard

> **Epic:** 2 (CLI) | **Sprint:** C3 | **Agent:** @kaven-db-engineer
> **Priority:** P2
> **Depends on:** None

---

## Contexto

O Kaven Framework possui **54 models Prisma** divididos em grupos funcionais (Core, Billing, Projects, Notifications, etc.) distribuídos no arquivo `packages/database/prisma/schema.extended.prisma`. Para projetos que não precisam de todos os módulos, o processo atual (GAP-4 do `AIOX-KAVEN-INTEGRATION.md`) é **comentar/descomentar manualmente** os models relevantes nesse arquivo.

Esse processo é perigoso por vários motivos:
- Módulos têm **dependências entre si**: `billing` depende de `tenant`, `projects` depende de `user`, etc.
- Desativar um módulo sem gerenciar dependências quebra silenciosamente o schema Prisma
- Não há validação — o erro só aparece em runtime (`prisma generate` ou `prisma migrate`)
- O processo não é autodocumentado — novo desenvolvedor não sabe quais módulos existem

Esta story implementa o `kaven module activate <module>` — um wizard que lê o `schema.extended.prisma`, resolve dependências, comenta/descomenta models automaticamente, e roda `pnpm db:generate + pnpm db:migrate`.

## User Story

**As a** developer setting up a new Kaven project,
**I want to** activate or deactivate Prisma modules via a CLI command that handles dependencies automatically,
**So that** my database schema only includes what I need, without risking broken migrations.

## Acceptance Criteria

- [ ] `kaven module list` exibe todos os módulos disponíveis com status (active/inactive) e dependências
- [ ] `kaven module activate <module>` ativa o módulo especificado
- [ ] Antes de ativar, verifica pré-requisitos: se o módulo depende de outro inativo, exibe erro claro
- [ ] `kaven module activate billing --with-deps` ativa automaticamente todos os módulos dependentes necessários
- [ ] `kaven module deactivate <module>` desativa o módulo (comenta os models no schema)
- [ ] Antes de desativar, verifica se outros módulos ativos dependem deste — se sim, exibe erro com lista dos dependentes
- [ ] Após ativar/desativar, roda automaticamente `pnpm db:generate` e `pnpm db:migrate` (com spinners)
- [ ] `--skip-migrate` pula as etapas de generate/migrate (útil para CI ou batch operations)
- [ ] `--dry-run` mostra quais models seriam afetados sem modificar o schema
- [ ] Confirmação explícita antes de qualquer mudança no schema (a menos que `--yes`)
- [ ] Output de confirmação lista quantos models foram afetados: `"Module billing activated. 6 models added (Invoice, Order, Subscription, Plan, Payment, Product)."`

### Schema Parsing — Comentar/Descomentar Models

```typescript
// kaven-cli: src/lib/schema-modifier.ts

export function deactivateModule(schemaPath: string, models: string[]): string {
  let schema = fs.readFileSync(schemaPath, 'utf-8');

  for (const modelName of models) {
    // Encontrar o bloco do model e comentá-lo
    const modelRegex = new RegExp(
      `(model ${modelName} \\{[^}]*\\})`,
      'gs'
    );
    schema = schema.replace(modelRegex, (match) =>
      match.split('\n').map(line => `// ${line}`).join('\n')
    );
  }

  return schema;
}

export function activateModule(schemaPath: string, models: string[]): string {
  let schema = fs.readFileSync(schemaPath, 'utf-8');

  for (const modelName of models) {
    // Encontrar o bloco comentado e descomentar
    const commentedModelRegex = new RegExp(
      `((?:// .*\\n)*// model ${modelName} \\{(?:\\n// [^}]*)*\\n// \\})`,
      'g'
    );
    schema = schema.replace(commentedModelRegex, (match) =>
      match.split('\n').map(line => line.replace(/^\/\/ /, '')).join('\n')
    );
  }

  return schema;
}
```

### Command Registration

```typescript
// kaven-cli: src/commands/module/activate.ts

export function registerModuleActivateCommand(moduleCmd: Command) {
  moduleCmd
    .command('activate <module>')
    .description('Activate a Prisma module and run migrations')
    .option('--with-deps', 'Automatically activate required dependencies')
    .option('--skip-migrate', 'Skip db:generate and db:migrate after activation')
    .option('--dry-run', 'Show affected models without modifying schema')
    .option('--yes', 'Skip confirmation prompt')
    .action(async (moduleName, opts) => {
      await activateModuleCommand(moduleName, opts);
    });

  moduleCmd
    .command('deactivate <module>')
    .description('Deactivate a Prisma module (comments models in schema)')
    .option('--skip-migrate', 'Skip db:generate and db:migrate')
    .option('--dry-run', 'Show affected models without modifying schema')
    .option('--yes', 'Skip confirmation prompt')
    .action(async (moduleName, opts) => {
      await deactivateModuleCommand(moduleName, opts);
    });
}
```
