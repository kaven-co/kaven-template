# QA Report: F3.2 — Connection Pool Singleton + Remoção de Dead Code

**Gate Decision:** ❌ FAIL
**Date:** 2026-04-21
**Assigned QA:** Quinn (Guardian)

---

## Executive Summary

A implementação funcional da story F3.2 está tecnicamente correta em relação ao código da aplicação. O Singleton do Prisma foi estabelecido e o código morto removido. No entanto, o **Quality Gate** falhou devido a uma regressão massiva na suíte de testes da API. A suíte foi deixada em estado "vermelho", com erros de mock e inconsistência de schema em testes de segurança.

## Findings

### 1. Singleton Prisma (H-1)
- **Status:** ✅ PASS
- **Evidência:** `grep` confirmou que apenas `apps/api/src/lib/prisma.ts` instancia `new PrismaClient()`. Todos os outros 8 serviços utilizam o singleton.
- **Observação:** O singleton foi estendido para incluir middleware de RLS e logging de queries, o que é excelente para a arquitetura.

### 2. Remoção de Dead Code (H-8)
- **Status:** ✅ PASS
- **Evidência:** Arquivo `apps/api/src/middleware/prisma-soft-delete.ts` foi deletado e não há referências remanescentes na codebase.

### 3. Testes Unitários
- **Status:** ❌ FAIL
- **Problema:** Erro `TypeError: prisma.$on is not a function`.
- **Causa Raiz:** Os mocks nos arquivos `*.spec.ts` (ex: `tasks.service.spec.ts`) mockam a classe `PrismaClient` do `@prisma/client` de forma incompleta. Como o novo singleton em `lib/prisma.ts` executa `prisma.$on` e `prisma.$use` imediatamente ao ser importado, os testes quebram.
- **Recomendação:** Atualizar a estratégia de mock para incluir os métodos de infraestrutura ou mockar diretamente o módulo `lib/prisma.ts`.

### 4. Testes de Segurança (E2E)
- **Status:** ❌ FAIL
- **Problema:** `The column Tenant.stripe_customer_id does not exist`.
- **Causa Raiz:** Fixtures em `test/security/fixtures/security.fixtures.ts` tentam usar o campo `stripe_customer_id` no modelo `Tenant`, mas este campo foi movido para `BillingAccount` em refatorações anteriores (F2.4).
- **Recomendação:** Atualizar os fixtures para refletir o schema atual.

---

## QA Fix Request

Solicito as seguintes correções antes da aprovação:

1.  **Fix Unit Mocks:** Atualizar os mocks do Prisma em todos os arquivos `.spec.ts` da API para suportar o singleton.
2.  **Fix Security Fixtures:** Atualizar `security.fixtures.ts` removendo a referência a `stripe_customer_id` no `Tenant`.
3.  **Test Run:** Garantir que `pnpm test --filter api` retorne VERDE.

### 5. Git Workflow & Atomicity
- **Status:** ⚠️ CONCERNS
- **Finding:** O último commit (`2ef968b9`) incluiu 63 arquivos, a maioria irrelevante para a F3.2 (docs de outras stories, guias de ID visual).
- **Risco:** Dificulta o rollback e polui o histórico da branch.
- **Recomendação:** Reforçar o uso de commits atômicos. O @dev não deve usar `git add .` indiscriminadamente.
