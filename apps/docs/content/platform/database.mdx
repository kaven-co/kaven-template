# Database (Split-Schema & Prisma 7)

O Kaven Framework utiliza uma arquitetura de **Split-Schema** potencializada pelo **Prisma 7**. Esta abordagem permite que a plataforma evolua o "core" (262 models) sem interferir nas suas customizações.

## 🏗️ Como funciona

O banco de dados é uma composição dinâmica de dois arquivos:

1.  **`schema.base.prisma`**: Localizado em `packages/database/prisma/`. Contém os modelos fundamentais de Multi-tenancy, Auth, Billing e Security. **Não deve ser editado.**
2.  **`schema.extended.prisma`**: Onde você adiciona seus modelos de negócio.
3.  **Merge Inteligente**: O sistema une os arquivos em um `schema.prisma` final otimizado para Prisma 7.

## 🔄 Fluxo de Trabalho com Kaven CLI

### 1. Adicionando uma nova tabela

Adicione seu modelo em `packages/database/prisma/schema.extended.prisma`:

```prisma
model MyProduct {
  id        String   @id @default(uuid())
  tenantId  String
  tenant    Tenant   @relation(fields: [tenantId], references: [id])
  name      String
  createdAt DateTime @default(now())
}
```

### 2. Sincronizando o Framework

Use a Kaven CLI para realizar o merge e gerar o client:

```bash
kaven db generate
```

### 3. Migrations no Prisma 7

O Kaven utiliza adapters serverless para máxima performance. Execute suas migrations via:

```bash
pnpm --filter database migrate:dev
```

> [!IMPORTANT]
> No Prisma 7, certifique-se de que seu `prisma.config.ts` está configurado corretamente para o seu ambiente (Neon, local ou CI).

## 💡 Vantagens

- **Atualizações Seguras**: Quando lançarmos uma nova versão do Kaven com melhorias no `schema.base.prisma`, você só precisa atualizar esse arquivo e rodar o `generate`. Suas tabelas no `extended` estarão intactas.
- **Isolamento de Responsabilidade**: Fica claro o que pertence ao framework e o que pertence ao seu produto.
- **Automação**: O CLI cuida de toda a concatenação e validação sintática básica.

> [!TIP]
> Use relações entre os modelos do `base` e do `extended` normalmente. O Prisma reconhecerá as relações após o merge dos arquivos.

## ⚠️ Limitações e Cuidados

- **Nomes Duplicados**: Não defina modelos com o mesmo nome em ambos os arquivos, pois o merge falhará.
- **Enums**: Assim como os modelos, enums também podem ser estendidos no arquivo `extended`.
