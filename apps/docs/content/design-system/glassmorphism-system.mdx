# Sistema Glassmorphism Adaptativo

> Sistema de design glassmorphism que adapta automaticamente entre os temas "Frosted Ice" (Light Mode) e "Deep Tech" (Dark Mode), garantindo uma estética premium em qualquer condição de iluminação.

## Visão Geral

O Sistema Glassmorphism Adaptativo do Kaven UI v2 foi projetado para criar interfaces SaaS de alta qualidade que funcionam perfeitamente em ambos os modos de tema. Ao invés de usar uma única abordagem de glassmorphism, o sistema adapta completamente sua aparência baseado no tema ativo.

**Principais Características:**

- ✅ Dois estilos distintos: "Frosted Ice" (Light) e "Deep Tech" (Dark)
- ✅ Transições suaves entre temas
- ✅ Variáveis CSS semânticas para fácil customização
- ✅ Sistema de cores adaptativo para ícones e badges
- ✅ Otimizado para legibilidade e contraste
- ✅ Performance otimizada com backdrop-filter

## Filosofia de Design

### Light Mode: "Frosted Ice" ❄️

O tema Light Mode utiliza uma abordagem de **vidro fosco limpo e arejado**, inspirado em gelo translúcido:

**Características Visuais:**

- Fundos brancos com alta opacidade (80-90%)
- Bordas brancas sutis para definição
- Sombras suaves e coloridas
- Texto escuro para máximo contraste
- Sensação de leveza e clareza

**Quando Usar:**

- Ambientes bem iluminados
- Interfaces corporativas
- Dashboards administrativos
- Aplicações de produtividade

### Dark Mode: "Deep Tech" 🌌

O tema Dark Mode utiliza uma abordagem de **vidro tecnológico sofisticado**, inspirado em interfaces futuristas:

**Características Visuais:**

- Fundos escuros translúcidos (50-75% opacidade)
- Bordas com brilho sutil
- Sombras profundas
- Efeito de profundidade em camadas
- Sensação de sofisticação e tecnologia

**Quando Usar:**

- Ambientes com pouca luz
- Interfaces criativas/técnicas
- Aplicações noturnas
- Experiências imersivas

## Variáveis CSS

### Estrutura Base

O sistema utiliza variáveis CSS que são redefinidas automaticamente quando o tema muda:

```css
/* apps/admin/app/globals.css */

:root {
  /* Light Mode - Frosted Ice */
  --glass-gradient-start: rgba(255, 255, 255, 0.8);
  --glass-gradient-end: rgba(255, 255, 255, 0.6);
  --color-glass-border: rgba(255, 255, 255, 0.5);
  --color-glass-highlight: rgba(255, 255, 255, 0.9);
  --glass-shadow: 0 8px 32px rgba(0, 0, 0, 0.08);
  --spotlight-color: rgba(255, 255, 255, 0.1);
}

.dark {
  /* Dark Mode - Deep Tech */
  --glass-gradient-start: rgba(17, 25, 40, 0.75);
  --glass-gradient-end: rgba(17, 25, 40, 0.5);
  --color-glass-border: rgba(255, 255, 255, 0.08);
  --color-glass-highlight: rgba(255, 255, 255, 0.05);
  --glass-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
  --spotlight-color: rgba(255, 255, 255, 0.03);
}
```

### Classe Utilitária `.glass-panel`

```css
.glass-panel {
  background: linear-gradient(
    135deg,
    var(--glass-gradient-start),
    var(--glass-gradient-end)
  );
  backdrop-filter: blur(10px) saturate(180%);
  -webkit-backdrop-filter: blur(10px) saturate(180%);
  border: 1px solid var(--color-glass-border);
  box-shadow: var(--glass-shadow);
  position: relative;
  overflow: hidden;
}

.glass-panel::before {
  content: '';
  position: absolute;
  top: 0;
  left: 0;
  right: 0;
  height: 1px;
  background: var(--color-glass-highlight);
  opacity: 0.5;
}
```

### Uso Prático

```tsx
// Aplicando glassmorphism em um card
<div className="glass-panel rounded-2xl p-6">
  <h3>Conteúdo do Card</h3>
  <p>Texto com contraste adequado</p>
</div>
```

## Sistema de Cores Semântico

### Padrão "Soft BG + Strong Text"

Para Light Mode, usamos fundos suaves com texto forte para garantir contraste:

```tsx
// Azul (Informação)
className = 'bg-blue-100 text-blue-600 dark:bg-blue-500/10 dark:text-blue-500';

// Amarelo (Atenção)
className =
  'bg-yellow-100 text-yellow-700 dark:bg-yellow-500/10 dark:text-yellow-500';

// Verde (Sucesso)
className =
  'bg-emerald-100 text-emerald-700 dark:bg-emerald-500/10 dark:text-emerald-400';

// Vermelho (Erro)
className = 'bg-red-100 text-red-700 dark:bg-red-500/10 dark:text-red-500';
```

### Aplicação em Ícones

```tsx
import { Users } from 'lucide-react';

// Ícone com cores adaptativas
<div className="p-3 rounded-full bg-blue-100 text-blue-600 dark:bg-blue-500/10 dark:text-blue-500">
  <Users className="h-5 w-5" />
</div>;
```

### Aplicação em Badges de Tendência

```tsx
// Badge positivo (crescimento)
<div className="bg-emerald-100 text-emerald-700 dark:bg-emerald-500/10 dark:text-emerald-400">
  +12%
</div>

// Badge negativo (queda)
<div className="bg-red-100 text-red-700 dark:bg-red-500/10 dark:text-red-500">
  -8%
</div>
```

## Guia de Migração

### De Cards Manuais para StatCard

**Antes (Manual):**

```tsx
<div className="relative overflow-hidden rounded-2xl bg-card p-6 shadow-xl border border-border/50">
  <div className="flex items-center gap-4 mb-4">
    <div className="p-3 bg-blue-500/10 rounded-full">
      <Users className="h-6 w-6 text-blue-500" />
    </div>
    <span className="text-sm font-bold uppercase">Novos Cadastros</span>
  </div>
  <div className="flex items-end justify-between">
    <div>
      <h3 className="text-3xl font-bold">156</h3>
      <div className="flex items-center gap-2 text-sm">
        <span className="text-green-500">+12%</span>
        <span className="text-muted-foreground">vs 7 dias</span>
      </div>
    </div>
  </div>
</div>
```

**Depois (StatCard):**

```tsx
<StatCard
  title="Novos Cadastros"
  value={156}
  icon={Users}
  trend={12}
  subtitle="vs 7 dias anteriores"
  variant="outline"
  iconClassName="bg-blue-100 text-blue-600 dark:bg-blue-500/10 dark:text-blue-500"
/>
```

**Benefícios:**

- ✅ 70% menos código
- ✅ Consistência automática
- ✅ Cores adaptativas por padrão
- ✅ Acessibilidade integrada
- ✅ Manutenção centralizada

### Atualizando Cores de Ícones

**Padrão Antigo (Apenas Dark Mode):**

```tsx
iconClassName = 'bg-blue-500/10 text-blue-500';
```

**Padrão Novo (Adaptativo):**

```tsx
iconClassName =
  'bg-blue-100 text-blue-600 dark:bg-blue-500/10 dark:text-blue-500';
```

## Troubleshooting

### Problema: Bordas brancas fortes no Dark Mode

**Sintoma:** Cards no Dark Mode aparecem com bordas brancas muito visíveis, quebrando o efeito glassmorphism.

**Causa:** Variáveis CSS do Light Mode não foram sobrescritas no bloco `.dark`.

**Solução:**

```css
/* Certifique-se de que as variáveis estão dentro do bloco .dark */
.dark {
  --color-glass-border: rgba(255, 255, 255, 0.08); /* NÃO 0.5 */
  --glass-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
}
```

### Problema: Baixo contraste em Light Mode

**Sintoma:** Texto ou ícones difíceis de ler em Light Mode.

**Causa:** Uso de cores "glass" do Dark Mode no Light Mode.

**Solução:**

```tsx
// ❌ ERRADO - Apenas dark mode
className = 'bg-blue-500/10 text-blue-500';

// ✅ CORRETO - Adaptativo
className = 'bg-blue-100 text-blue-600 dark:bg-blue-500/10 dark:text-blue-500';
```

### Problema: Glassmorphism não funciona em Safari

**Sintoma:** Efeito de blur não aparece no Safari.

**Causa:** Falta do prefixo `-webkit-`.

**Solução:**

```css
.glass-panel {
  backdrop-filter: blur(10px) saturate(180%);
  -webkit-backdrop-filter: blur(10px) saturate(180%); /* ← Adicionar */
}
```

### Problema: Performance ruim com muitos cards

**Sintoma:** Lag ao rolar página com muitos elementos glassmorphism.

**Causa:** `backdrop-filter` é computacionalmente caro.

**Solução:**

```css
/* Usar will-change para otimizar */
.glass-panel {
  will-change: transform;
  transform: translateZ(0); /* Force GPU acceleration */
}

/* Ou desabilitar blur em dispositivos móveis */
@media (max-width: 768px) {
  .glass-panel {
    backdrop-filter: none;
    background: var(--glass-gradient-start);
  }
}
```

## Boas Práticas

### ✅ DO

- **Use variáveis CSS** para todos os valores de glassmorphism
- **Teste em ambos os temas** antes de finalizar
- **Mantenha contraste adequado** (mínimo WCAG AA)
- **Use classes utilitárias** como `.glass-panel`
- **Aplique cores semânticas** para ícones e badges

### ❌ DON'T

- **Não use valores hardcoded** de cor/opacidade
- **Não ignore o Light Mode** (foque apenas no Dark)
- **Não abuse de backdrop-filter** (performance)
- **Não misture padrões** (glass no light, solid no dark)
- **Não use cores arbitrárias** sem sistema
